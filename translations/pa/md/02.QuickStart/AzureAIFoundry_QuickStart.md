# **Microsoft Foundry ਵਿੱਚ Phi-3 ਦੀ ਵਰਤੋਂ ਕਰਨਾ**

ਜੈਨੇਰੇਟਿਵ ਏਆਈ ਦੇ ਵਿਕਾਸ ਨਾਲ, ਅਸੀਂ ਉਮੀਦ ਕਰਦੇ ਹਾਂ ਕਿ ਇੱਕ ਸੰਗਠਿਤ ਪਲੇਟਫਾਰਮ ਦੇ ਜ਼ਰੀਏ ਵੱਖ-ਵੱਖ LLM ਅਤੇ SLM, ਉੱਦਯੋਗ ਡੇਟਾ ਇੰਟੀਗ੍ਰੇਸ਼ਨ, ਫਾਈਨ-ਟਿਊਨਿੰਗ/RAG ਓਪਰੇਸ਼ਨ, ਅਤੇ ਵੱਖ-ਵੱਖ ਉੱਦਯੋਗ ਕਾਰੋਬਾਰਾਂ ਦੀ ਮੂਲਿਆੰਕਣ ਨੂੰ ਸੁਚੱਜੇ ਤਰੀਕੇ ਨਾਲ ਸੰਭਾਲਿਆ ਜਾ ਸਕੇ, ਤਾਂ ਕਿ ਜੈਨੇਰੇਟਿਵ ਏਆਈ ਸਮਾਰਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਬਿਹਤਰ ਤਰੀਕੇ ਨਾਲ ਲਾਗੂ ਕੀਤਾ ਜਾ ਸਕੇ। [Microsoft Foundry](https://ai.azure.com) ਇੱਕ ਉੱਦਯੋਗ-ਪੱਧਰੀ ਜੈਨੇਰੇਟਿਵ ਏਆਈ ਐਪਲੀਕੇਸ਼ਨ ਪਲੇਟਫਾਰਮ ਹੈ।

![aistudo](../../../../translated_images/pa/aifoundry_home.f28a8127c96c7d93.webp)

Microsoft Foundry ਨਾਲ, ਤੁਸੀਂ ਵੱਡੇ ਭਾਸ਼ਾ ਮਾਡਲ (LLM) ਦੇ ਪ੍ਰਤੀਕਿਰਿਆਵਾਂ ਦਾ ਮੂਲਿਆੰਕਣ ਕਰ ਸਕਦੇ ਹੋ ਅਤੇ ਪ੍ਰਾਡਮਟ ਫ਼ਲੋ ਨਾਲ ਪ੍ਰਾਰੰਭਕ ਐਪਲੀਕੇਸ਼ਨ ਕੰਪੋਨੈਂਟਸ ਨੂੰ ਪ੍ਰਬੰਧਿਤ ਕਰਕੇ ਵਧੀਆ ਪ੍ਰਦਰਸ਼ਨ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦੇ ਹੋ। ਇਹ ਪਲੇਟਫਾਰਮ ਪ੍ਰਮਾਣਿਕ ਸਿੱਧਾਂਤਾਂ ਤੋਂ ਪੂਰਨ ਉਤਪਾਦਨ ਤੱਕ ਪਰਿਵਰਤਨ ਲਈ ਸਕੇਲੈਬਿਲਿਟੀ ਨੂੰ ਸਮਰਥਨ ਦਿੰਦਾ ਹੈ। ਲੰਬੇ ਸਮੇਂ ਤੱਕ ਸਫਲਤਾ ਲਈ ਲਗਾਤਾਰ ਨਿਗਰਾਨੀ ਅਤੇ ਸੰਸ਼ੋਧਨ ਹੁੰਦਾ ਹੈ।

ਅਸੀਂ Microsoft Foundry 'ਤੇ Phi-3 ਮਾਡਲ ਨੂੰ ਸਧਾਰਣ ਕਦਮਾਂ ਨਾਲ ਤੁਰੰਤ ਤਾਇਨਾਤ ਕਰ ਸਕਦੇ ਹਾਂ, ਫਿਰ Microsoft Foundry ਦੀ ਵਰਤੋਂ ਕਰਕੇ Phi-3 ਸਬੰਧੀ Playground/ਚੈਟ, ਫਾਈਨ-ਟਿਊਨਿੰਗ, ਮੂਲਿਆੰਕਣ ਅਤੇ ਹੋਰ ਸਬੰਧਿਤ ਕਾਰਜਾਂ ਨੂੰ ਪੂਰਾ ਕਰ ਸਕਦੇ ਹਾਂ।

## **1. ਤਿਆਰੀ**

ਜੇ ਤੁਹਾਡੇ ਕੋਲ ਪਹਿਲਾਂ ਹੀ [Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/overview?WT.mc_id=aiml-138114-kinfeylo) ਤੁਹਾਡੇ ਮਸ਼ੀਨ 'ਤੇ ਇੰਸਟਾਲ ਹੈ, ਤਾਂ ਇਸ ਟੈਂਪਲੇਟ ਦੀ ਵਰਤੋਂ ਕਰਨ ਲਈ ਸਿਰਫ ਇੱਕ ਨਵੇਂ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਇਸ ਕਮਾਂਡ ਨੂੰ ਚਲਾਉਣਾ ਕਾਫ਼ੀ ਹੈ।

## ਮੈਨੂਅਲ ਬਣਾਉਣਾ

Microsoft Foundry ਪ੍ਰੋਜੈਕਟ ਅਤੇ ਹੱਬ ਬਣਾਉਣਾ ਤੁਹਾਡੇ ਏਆਈ ਕੰਮ ਨੂੰ ਸੁਚੱਜੇ ਤਰੀਕੇ ਨਾਲ ਆਯੋਜਿਤ ਅਤੇ ਪ੍ਰਬੰਧਿਤ ਕਰਨ ਦਾ ਇੱਕ ਵਧੀਆ ਤਰੀਕਾ ਹੈ। ਸ਼ੁਰੂਆਤ ਕਰਨ ਲਈ ਇਹ ਕਦਮ ਦਰ ਕਦਮ ਗਾਈਡ ਹੈ:

### Microsoft Foundry ਵਿੱਚ ਪ੍ਰੋਜੈਕਟ ਬਣਾਉਣਾ

1. **Microsoft Foundry 'ਤੇ ਜਾਓ**: Microsoft Foundry ਪੋਰਟਲ 'ਚ ਸਾਈਨ ਇਨ ਕਰੋ।
2. **ਪ੍ਰੋਜੈਕਟ ਬਣਾਓ**:
   - ਜੇ ਤੁਸੀਂ ਕਿਸੇ ਪ੍ਰੋਜੈਕਟ ਵਿੱਚ ਹੋ, ਤਾਂ ਸਫ਼ੇ ਦੇ ਉੱਪਰ ਖੱਬੇ ਕੋਨੇ ਵਿੱਚ "Microsoft Foundry" 'ਤੇ ਕਲਿਕ ਕਰਕੇ ਹੋਮ ਪੇਜ 'ਤੇ ਜਾਓ।
   - "+ Create project" ਚੁਣੋ।
   - ਪ੍ਰੋਜੈਕਟ ਲਈ ਇੱਕ ਨਾਮ ਦਰਜ ਕਰੋ।
   - ਜੇ ਤੁਹਾਡੇ ਕੋਲ ਇੱਕ ਹੱਬ ਹੈ, ਤਾਂ ਉਹ ਡਿਫਾਲਟ ਤੌਰ 'ਤੇ ਚੁਣਿਆ ਜਾਵੇਗਾ। ਜੇ ਤੁਹਾਨੂੰ ਇੱਕ ਤੋਂ ਜ਼ਿਆਦਾ ਹੱਬਾਂ ਤੱਕ ਪਹੁੰਚ ਹੈ, ਤਾਂ ਤੁਸੀਂ ਡ੍ਰਾਪਡਾਊਨ ਤੋਂ ਕੋਈ ਹੋਰ ਚੁਣ ਸਕਦੇ ਹੋ। ਜੇ ਤੁਸੀਂ ਨਵਾਂ ਹੱਬ ਬਣਾਉਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ "Create new hub" ਚੁਣੋ ਅਤੇ ਇੱਕ ਨਾਮ ਦਿਓ।
   - "Create" ਤੇ ਕਲਿਕ ਕਰੋ।

### Microsoft Foundry ਵਿੱਚ ਹੱਬ ਬਣਾਉਣਾ

1. **Microsoft Foundry 'ਤੇ ਜਾਓ**: ਆਪਣੇ Azure ਖਾਤੇ ਨਾਲ ਸਾਈਨ ਇਨ ਕਰੋ।
2. **ਹੱਬ ਬਣਾਉਣ ਲਈ**:
   - ਖੱਬੇ ਮੇਨੂ ਤੋਂ Management center ਚੁਣੋ।
   - "All resources" ਚੁਣੋ, ਫਿਰ "+ New project" ਦੇ ਨਾਲ ਵਾਲੇ ਤੀਰ ਤੇ ਕਲਿਕ ਕਰੋ ਅਤੇ "+ New hub" ਚੁਣੋ।
   - "Create a new hub" ਡਾਇਲਾਗ ਵਿਚ, ਆਪਣੇ ਹੱਬ ਲਈ ਇੱਕ ਨਾਮ ਦਿਓ (ਜਿਵੇਂ ਕਿ contoso-hub) ਅਤੇ ਹੋਰ ਖੇਤਰਾਂ ਵਿਚ ਜਰੂਰਤ ਮੁਤਾਬਕ ਸੋਧ ਕਰੋ।
   - "Next" ਚੁਣੋ, ਜਾਣਕਾਰੀ ਦੀ ਸਮੀਖਿਆ ਕਰੋ ਅਤੇ ਫਿਰ "Create" ਚੁਣੋ।

ਵਧੇਰੇ ਵਿਸਥਾਰ ਨਾਲ ਹਦਾਇਤਾਂ ਲਈ, ਤੁਸੀਂ ਅਧਿਕਾਰਿਕ [Microsoft ਦਸਤਾਵੇਜ਼](https://learn.microsoft.com/azure/ai-studio/how-to/create-projects) ਨੂੰ ਵੇਖ ਸਕਦੇ ਹੋ।

ਕਾਮਯਾਬ ਬਣਾਉਂਦੇ ਹੀ, ਤੁਸੀਂ [ai.azure.com](https://ai.azure.com/) ਰਾਹੀਂ ਬਣਾਈ ਗਈ ਸਟੂਡੀਓ ਨੂੰ ਐਕਸੈਸ ਕਰ ਸਕਦੇ ਹੋ।

ਇੱਕ AI Foundry 'ਤੇ ਕਈ ਪ੍ਰੋਜੈਕਟ ਹੋ ਸਕਦੇ ਹਨ। ਤਿਆਰੀ ਲਈ AI Foundry ਵਿੱਚ ਇੱਕ ਪ੍ਰੋਜੈਕਟ ਬਣਾਓ।

Microsoft Foundry [QuickStarts](https://learn.microsoft.com/azure/ai-studio/quickstarts/get-started-code) ਬਣਾਓ

## **4. Microsoft Foundry ਤੋਂ ਮਾਡਲ ਨੂੰ ਤਾਇਨਾਤ ਕਰਨਾ**

Azure Model Catalog ਤੋਂ ਮਾਡਲ ਤਾਇਨਾਤ ਕਰਨ ਲਈ, ਤੁਸੀਂ ਇਹ ਕਦਮ ਫਾਲੋ ਕਰ ਸਕਦੇ ਹੋ:

- Microsoft Foundry 'ਚ ਸਾਈਨ ਇਨ ਕਰੋ।
- Microsoft Foundry ਮਾਡਲ ਕੈਟਲਾਗ ਵਿੱਚੋਂ ਆਪਣਾ ਮਾਡਲ ਚੁਣੋ ਜੋ ਤੁਸੀਂ ਤਾਇਨਾਤ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ।
- ਮਾਡਲ ਦੇ ਵਿਸਥਾਰ ਸਫ਼ੇ 'ਤੇ Deploy ਚੁਣੋ ਅਤੇ ਫਿਰ Serverless API with Azure AI Content Safety ਚੁਣੋ।
- ਉਸ ਪ੍ਰੋਜੈਕਟ ਨੂੰ ਚੁਣੋ ਜਿਸ ਵਿੱਚ ਤੁਸੀਂ ਆਪਣੇ ਮਾਡਲ ਤਾਇਨਾਤ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ। Serverless API ਦੀ ਵਰਤੋਂ ਕਰਨ ਲਈ, ਤੁਹਾਡਾ ਵਰਕਸਪੇਸ East US 2 ਜਾਂ Sweden Central ਖੇਤਰ ਨਾਲ ਸੰਬੰਧਤ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ। ਤੁਸੀਂ Deployment ਦੇ ਨਾਮ ਨੂੰ ਕਸਟਮਾਈਜ਼ ਵੀ ਕਰ ਸਕਦੇ ਹੋ।
- ਤਾਇਨਾਤ ਵਿਜ਼ਾਰਡ ਵਿੱਚ, ਕੀਮਤਾਂ ਅਤੇ ਨਿਯਮਾਂ ਬਾਰੇ ਜਾਣਨ ਲਈ Pricing and terms ਚੁਣੋ।
- Deploy 'ਤੇ ਕਲਿਕ ਕਰੋ। ਤਾਇਨਾਤ ਪੂਰੀ ਹੋਣ ਦਾ ਇੰਤਜ਼ਾਰ ਕਰੋ ਅਤੇ ਤੁਸੀਂ ਤਾਇਨਾਤ ਪੇਜ 'ਤੇ ਰੀਡਾਇਰੈਕਟ ਕੀਤਾ ਜਾਵੇਗਾ।
- Open in playground ਚੁਣੋ ਤਾਂ ਜੋ ਮਾਡਲ ਨਾਲ ਇੰਟਰਐਕਟ ਕਰਨਾ ਸ਼ੁਰੂ ਕਰ ਸਕੋ।
- ਤੁਸੀਂ Deployments ਪੇਜ 'ਤੇ ਵਾਪਸ ਜਾ ਸਕਦੇ ਹੋ, ਤਾਇਨਾਤ ਚੁਣੋ, ਅਤੇ ਟਾਰਗੇਟ URL ਅਤੇ ਸੀਕ੍ਰੇਟ ਕੀ ਲਈ ਨੋਟ ਕਰ ਸਕਦੇ ਹੋ, ਜਿਹਨਾਂ ਦੀ ਵਰਤੋਂ ਤੁਸੀਂ ਤਾਇਨਾਤ ਨੂੰ ਕਾਲ ਕਰਨ ਅਤੇ ਕਮਪਲੀਸ਼ਨ ਬਣਾਉਣ ਲਈ ਕਰ ਸਕਦੇ ਹੋ।
- ਤੁਸੀਂ ਸਦਾ Build ਟੈਬ ਵਿੱਚ ਜਾ ਕੇ Components ਸੈਕਸ਼ਨ ਤੋਂ Deployments ਚੁਣ ਕੇ ਐਂਡਪੌਇੰਟ ਦੇ ਵੇਰਵੇ, URL ਅਤੇ ਐਕਸੈਸ ਕੀਜ਼ ਨੂੰ ਵੇਖ ਸਕਦੇ ਹੋ।

> [!NOTE]
> ਕਿਰਪਾ ਕਰਕੇ ਧਿਆਨ ਦਿਓ ਕਿ ਇਹ ਕਦਮ ਕਰਨ ਲਈ ਤੁਹਾਡੇ ਖਾਤੇ ਕੋਲ Resource Group 'ਤੇ Azure AI Developer ਰੋਲ ਦੀ ਮਨਜ਼ੂਰੀ ਹੋਣੀ ਜ਼ਰੂਰੀ ਹੈ।

## **5. Microsoft Foundry ਵਿੱਚ Phi API ਦੀ ਵਰਤੋਂ**

ਤੁਸੀਂ ਬਹੁਤ ਆਸਾਨੀ ਨਾਲ ਬੇਨਤੀ ਪੈਰਾਮੀਟਰ ਅਤੇ ਪ੍ਰਤੀਕਿਰਿਆ ਪੈਰਾਮੀਟਰ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦੇ ਹੋ।
