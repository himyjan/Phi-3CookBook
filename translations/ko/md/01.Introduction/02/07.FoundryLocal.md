## Foundry Local에서 Phi-Family 모델 시작하기

### Foundry Local 소개

Foundry Local은 엔터프라이즈급 AI 기능을 로컬 하드웨어로 직접 제공하는 강력한 온디바이스 AI 추론 솔루션입니다. 이 튜토리얼은 Foundry Local에서 Phi-Family 모델을 설정하고 사용하는 방법을 안내하며, AI 작업의 완전한 제어권을 유지하면서 개인 정보 보호 및 비용 절감을 실현할 수 있도록 도와줍니다.

Foundry Local은 기기에서 AI 모델을 로컬로 실행하여 성능, 개인 정보 보호, 맞춤화 및 비용 효율성을 제공합니다. 직관적인 CLI, SDK, REST API를 통해 기존 워크플로우 및 애플리케이션과 원활하게 통합됩니다.


![arch](../../../../../translated_images/ko/foundry-local-arch.8823e321dd8258d7.webp)

### Foundry Local을 선택해야 하는 이유

Foundry Local의 장점을 이해하면 AI 배포 전략에 관한 현명한 결정을 내리는 데 도움이 됩니다:

- **온디바이스 추론:** 자체 하드웨어에서 모델을 로컬로 실행하여 비용을 절감하고 모든 데이터를 기기에 안전하게 유지할 수 있습니다.

- **모델 맞춤화:** 사전 설정된 모델을 선택하거나 특정 요구 사항과 사용 사례에 맞는 자체 모델을 사용할 수 있습니다.

- **비용 효율성:** 기존 하드웨어를 활용하여 반복적인 클라우드 서비스 비용을 없애 AI를 보다 접근하기 쉽게 만듭니다.

- **원활한 통합:** SDK, API 엔드포인트, CLI를 통해 애플리케이션과 연결하며, 필요에 따라 Microsoft Foundry로 쉽게 확장할 수 있습니다.

> **시작 안내:** 이 튜토리얼은 CLI 및 SDK 인터페이스를 통한 Foundry Local 사용에 중점을 둡니다. 두 가지 방식을 모두 배우며 사용 사례에 맞는 최적의 방법을 선택할 수 있도록 도와드립니다.

## 1부: Foundry Local CLI 설정

### 1단계: 설치

Foundry Local CLI는 로컬에서 AI 모델을 관리하고 실행하는 관문입니다. 먼저 시스템에 설치하는 방법부터 시작해봅시다.

**지원 플랫폼:** Windows 및 macOS

자세한 설치 지침은 [공식 Foundry Local 문서](https://github.com/microsoft/Foundry-Local/blob/main/README.md)를 참조하세요.

### 2단계: 사용 가능한 모델 탐색

Foundry Local CLI가 설치되면 사용 가능한 모델을 확인할 수 있습니다. 아래 명령어로 지원되는 모든 모델을 확인하세요:


```bash
foundry model list
```

### 3단계: Phi Family 모델 이해하기

Phi Family는 다양한 사용 사례와 하드웨어 구성에 최적화된 여러 모델을 제공합니다. Foundry Local에서 사용 가능한 Phi 모델은 다음과 같습니다:

**사용 가능한 Phi 모델:**

- **phi-3.5-mini** - 기본 작업용 컴팩트 모델
- **phi-3-mini-128k** - 더 긴 대화를 위한 확장 컨텍스트 버전
- **phi-3-mini-4k** - 일반 용도의 표준 컨텍스트 모델
- **phi-4** - 향상된 기능을 갖춘 고급 모델
- **phi-4-mini** - Phi-4의 경량 버전
- **phi-4-mini-reasoning** - 복잡한 추론 작업에 특화된 모델

> **하드웨어 호환성:** 각 모델은 시스템 능력에 따라 다른 하드웨어 가속기(CPU, GPU)용으로 구성할 수 있습니다.

### 4단계: 첫 번째 Phi 모델 실행

실습 예제로 시작해 보겠습니다. 단계별로 복잡한 문제를 해결하는데 뛰어난 `phi-4-mini-reasoning` 모델을 실행해 봅니다.


**모델 실행 명령어:**

```bash
foundry model run Phi-4-mini-reasoning-generic-cpu
```

> **첫 실행 설정:** 모델을 처음 실행할 경우 Foundry Local이 자동으로 로컬 기기로 다운로드합니다. 네트워크 속도에 따라 다운로드 시간이 다르니 초기 설정 시 잠시 기다려 주세요.

### 5단계: 실제 문제로 모델 테스트

이제 모델이 단계별 추론을 어떻게 수행하는지 보기 위해 고전 논리 문제로 테스트해 봅시다:

**예제 문제:**

```txt
Please calculate the following step by step: Now there are pheasants and rabbits in the same cage, there are thirty-five heads on top and ninety-four legs on the bottom, how many pheasants and rabbits are there?
```

**예상 동작:** 모델은 이 문제를 논리적 단계로 나누어 꿩은 다리가 2개, 토끼는 4개라는 사실을 이용해 연립방정식을 해결할 것입니다.

## 2부: Foundry Local SDK로 애플리케이션 구축하기

### SDK를 사용해야 하는 이유

CLI는 테스트와 빠른 상호작용에 적합하지만, SDK를 사용하면 Foundry Local을 프로그래밍 방식으로 애플리케이션에 통합할 수 있습니다. 이로 인해 다음과 같은 가능성이 열립니다:

- 맞춤 AI 기반 애플리케이션 구축
- 자동화된 워크플로우 생성
- 기존 시스템에 AI 기능 통합
- 챗봇과 인터랙티브 도구 개발

### 지원 프로그래밍 언어

Foundry Local은 개발 선호에 맞게 여러 프로그래밍 언어용 SDK를 제공합니다:

**📦 사용 가능한 SDK:**

- **C# (.NET):** [SDK 문서 및 예제](https://github.com/microsoft/Foundry-Local/tree/main/sdk/cs)
- **Python:** [SDK 문서 및 예제](https://github.com/microsoft/Foundry-Local/tree/main/sdk/python)
- **JavaScript:** [SDK 문서 및 예제](https://github.com/microsoft/Foundry-Local/tree/main/sdk/js)
- **Rust:** [SDK 문서 및 예제](https://github.com/microsoft/Foundry-Local/tree/main/sdk/rust)

### 다음 단계

1. 개발 환경에 맞는 **선호하는 SDK 선택**
2. <strong>SDK별 문서</strong>를 참조하여 자세한 구현 가이드 확인
3. 복잡한 애플리케이션 개발 전에 **간단한 예제부터 시작**
4. 각 SDK 저장소에 제공된 **샘플 코드 탐색**
