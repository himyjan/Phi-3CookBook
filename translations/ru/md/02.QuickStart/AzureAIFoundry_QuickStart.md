# **Использование Phi-3 в Microsoft Foundry**

С развитием генеративного ИИ мы надеемся использовать единую платформу для управления различными LLM и SLM, интеграции корпоративных данных, операций тонкой настройки/RAG и оценки различных корпоративных бизнесов после интеграции LLM и SLM и т.д., чтобы генеративный ИИ мог лучше реализовывать интеллектуальные приложения. [Microsoft Foundry](https://ai.azure.com) — корпоративная платформа для генеративных ИИ-приложений.

![aistudo](../../../../translated_images/ru/aifoundry_home.f28a8127c96c7d93.webp)

С помощью Microsoft Foundry вы можете оценивать ответы больших языковых моделей (LLM) и оркестровать компоненты приложений с подсказками с помощью prompt flow для лучшей производительности. Платформа обеспечивает масштабируемость для легкого преобразования прототипов в полноценное производство. Непрерывный мониторинг и совершенствование поддерживают долгосрочный успех.

Мы можем быстро развернуть модель Phi-3 в Microsoft Foundry с помощью простых шагов, а затем использовать Microsoft Foundry для выполнения связанных с Phi-3 Playground/Chat, тонкой настройки, оценки и других задач.

## **1. Подготовка**

Если у вас уже установлен [Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/overview?WT.mc_id=aiml-138114-kinfeylo) на вашем компьютере, использование этого шаблона сводится к выполнению этой команды в новой директории.

## Ручное создание

Создание проекта и хаба в Microsoft Foundry — отличный способ организовать и управлять вашей работой с ИИ. Вот пошаговое руководство для начала:

### Создание проекта в Microsoft Foundry

1. **Перейдите в Microsoft Foundry**: Войдите в портал Microsoft Foundry.
2. **Создайте проект**:
   - Если вы находитесь в проекте, выберите "Microsoft Foundry" в левом верхнем углу страницы, чтобы перейти на главную страницу.
   - Выберите "+ Create project".
   - Введите имя для проекта.
   - Если у вас есть хаб, он будет выбран по умолчанию. Если у вас есть доступ к нескольким хабам, вы можете выбрать другой из выпадающего списка. Если хотите создать новый хаб, выберите "Create new hub" и укажите имя.
   - Нажмите "Create".

### Создание хаба в Microsoft Foundry

1. **Перейдите в Microsoft Foundry**: Войдите с вашей учетной записью Azure.
2. **Создайте хаб**:
   - Выберите Центр управления в левом меню.
   - Выберите "Все ресурсы", затем стрелку вниз рядом с "+ New project" и выберите "+ New hub".
   - В диалоговом окне "Create a new hub" введите имя для вашего хаба (например, contoso-hub) и при необходимости измените другие поля.
   - Нажмите "Next", проверьте информацию и затем "Create".

Для более подробных инструкций вы можете обратиться к официальной [документации Microsoft](https://learn.microsoft.com/azure/ai-studio/how-to/create-projects).

После успешного создания вы можете получить доступ в созданную студию через [ai.azure.com](https://ai.azure.com/).

На одном AI Foundry может быть несколько проектов. Создайте проект в AI Foundry, чтобы подготовиться.

Создайте Microsoft Foundry [QuickStarts](https://learn.microsoft.com/azure/ai-studio/quickstarts/get-started-code)


## **2. Развертывание модели Phi в Microsoft Foundry**

Нажмите на опцию Explore проекта, чтобы войти в каталог моделей и выберите Phi-3.

Выберите Phi-3-mini-4k-instruct.

Нажмите 'Deploy' для развертывания модели Phi-3-mini-4k-instruct.

> [!NOTE]
>
> При развертывании вы можете выбрать вычислительную мощность.

## **3. Playground Chat Phi в Microsoft Foundry**

Перейдите на страницу развертывания, выберите Playground и общайтесь с Phi-3 через Microsoft Foundry.

## **4. Развертывание модели из Microsoft Foundry**

Для развертывания модели из каталога моделей Azure выполните следующие шаги:

- Войдите в Microsoft Foundry.
- Выберите модель, которую хотите развернуть, из каталога моделей Microsoft Foundry.
- На странице сведений о модели выберите Deploy, затем выберите Serverless API с Azure AI Content Safety.
- Выберите проект, в котором хотите развернуть модели. Чтобы использовать предложение Serverless API, ваш рабочий пространство должно находиться в регионе East US 2 или Sweden Central. Можно настроить имя развертывания.
- В мастере развертывания выберите Pricing and terms, чтобы ознакомиться с ценами и условиями использования.
- Нажмите Deploy. Дождитесь готовности развертывания и перенаправления на страницу Развертывания.
- Нажмите Open in playground, чтобы начать взаимодействие с моделью.
- Вы можете вернуться на страницу Развертывания, выбрать развертывание и заметить целевой URL конечной точки и секретный ключ, которые можно использовать для вызова развертывания и генерации ответов.
- Вы всегда можете найти детали конечной точки, URL и ключи доступа, перейдя на вкладку Build и выбрав Развертывания в разделе Components.

> [!NOTE]
> Обратите внимание, что для выполнения этих шагов учетная запись должна иметь роль Azure AI Developer с разрешениями на группу ресурсов.

## **5. Использование Phi API в Microsoft Foundry**

Вы можете получить доступ к https://{Your project name}.region.inference.ml.azure.com/swagger.json через Postman GET и, используя ключ, ознакомиться с предоставляемыми интерфейсами.

Вы сможете очень удобно получить параметры запроса, а также параметры ответа.
