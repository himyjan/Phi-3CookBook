# Phi Hardware-Unterstützung

Microsoft Phi wurde für ONNX Runtime optimiert und unterstützt Windows DirectML. Es funktioniert gut auf verschiedenen Hardwaretypen, einschließlich GPUs, CPUs und sogar mobilen Geräten.

## Gerätehardware  
Konkret umfasst die unterstützte Hardware:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPUs, 128 GiB Speicher)

## Mobile SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 oder höher A16/A17 Prozessor

## Phi Hardware-Spezifikation

- Mindestkonfiguration erforderlich.
- Windows: DirectX 12-fähige GPU und mindestens 4 GB kombinierten RAM

CUDA: NVIDIA GPU mit Compute Capability >= 7.02

## Ausführung von onnxruntime auf mehreren GPUs

Derzeit verfügbare Phi ONNX-Modelle sind nur für 1 GPU. Es ist möglich, Multi-GPU für Phi-Modelle zu unterstützen, aber ORT mit 2 GPUs garantiert nicht, dass es eine höhere Durchsatzrate verglichen mit 2 Instanzen von ORT bietet. Bitte sehen Sie für die neuesten Updates [ONNX Runtime](https://onnxruntime.ai/).

Auf der [Build 2024 hat das GenAI ONNX Team](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) bekanntgegeben, dass sie Multi-Instance anstelle von Multi-GPU für Phi-Modelle aktiviert haben.

Derzeit können Sie damit eine Instanz von onnnxruntime oder onnxruntime-genai mit der CUDA_VISIBLE_DEVICES Umgebungsvariablen wie folgt ausführen.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

Erkunden Sie Phi gerne weiter in [Microsoft Foundry](https://ai.azure.com)
