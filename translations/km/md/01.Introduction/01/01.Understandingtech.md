# បច្ចេកវិទ្យាសំខាន់ៗដែលបានกล่าวដូចខាងក្រោម

1. [DirectML](https://learn.microsoft.com/windows/ai/directml/dml?WT.mc_id=aiml-138114-kinfeylo) - API កម្រិតទាបសម្រាប់មេស៊ីន lerning ដែលបានបង្កើតឡើងនៅលើ DirectX 12 ដើម្បីជំនួយសមត្ថភាពពី hardware ។
2. [CUDA](https://blogs.nvidia.com/blog/what-is-cuda-2/) - វេទិកា parallel computing និងម៉ូដែល API ដែលបានអភិវឌ្ឍដោយ Nvidia អនុញ្ញាតឲ្យដំណើរការទូទៅលើ GPU នៅក្រោម graphics processing units។ 
3. [ONNX](https://onnx.ai/) (Open Neural Network Exchange) - ទ្រង់ទ្រាយបើកដែលបានออกแบบសម្រាប់បំណន្នាទ្រង់ទ្រង់ម៉ូដែល machine learning ដែលផ្តល់ភាពអាចធ្វើការជាមួយគ្នា រវាង frameworks ML ផ្សេងៗ។  
4. [GGUF](https://github.com/ggerganov/ggml/blob/master/docs/gguf.md) (Generic Graph Update Format) - ទ្រង់ទ្រាយដែលប្រើសម្រាប់តំណាង និងធ្វើបច្ចុប្បន្នភាពម៉ូដែល machine learning ជាក់ស្តែងសម្រាប់ម៉ូដែលភាសាកំណត់តូចៗដែលអាចរត់បានល្អលើ CPU ជាមួយ quantization 4-8bit។

## DirectML

DirectML គឺជា API កម្រិតទាបដែលអនុញ្ញាតសមត្ថភាព hardware-accelerated machine learning។ វាត្រូវបានបង្កើតនៅលើ DirectX 12 ដើម្បីប្រើប្រាស់ GPU acceleration ហើយវាជា vendor-agnostic មានន័យថាវាមិន​ទាមទារការផ្លាស់ប្តូរកូដសម្រាប់ដំណើរការលើ GPU vendors ផ្សេងៗទេ។ វាត្រូវបានប្រើសម្រាប់ការបណ្តុះបណ្តាលម៉ូដែល និងកិច្ចការ inferencing លើ GPU ចម្បង។

សម្រាប់ hardware support, DirectML ត្រូវបានរចនាឡើងដើម្បីដំណើរការជាមួយ GPU ច្រើនប្រភេទ រួមមាន AMD GPUs រួមបញ្ចូលទាំង integrated និង discrete, Intel integrated GPUs, និង NVIDIA discrete GPUs។ វា​ជា​ផ្នែក​មួយ​នៃ​វេទិកា Windows AI Platform ហើយគាំទ្រនៅលើ Windows 10 & 11 ដែលអាចធ្វើបណ្តុះបណ្តាល និង inferencing ម៉ូដែលលើឧបករណ៍ Windows មួយចំនួន។

មានការអាប់ដេត និងឱកាសជាច្រើនដែលទាក់ទងនឹង DirectML ដូចជា គាំទ្រជាមួយលើសពី ១៥០ ONNX operators និងប្រើដោយ ONNX runtime និង WinML។ វាគាំទ្រដោយ Integrated Hardware Vendors (IHVs) សំខាន់ៗ ដែលរៀបចំបញ្ជារអាជ្ញារ metacommands ផ្សេងៗ។

## CUDA

CUDA មានន័យថា Compute Unified Device Architecture គឺជាវេទិកា parallel computing និងម៉ូដែល API ដែលបង្កើតឡើងដោយ Nvidia។ វាអនុញ្ញាតឲ្យអ្នកអភិវឌ្ឍ software ប្រើ GPU ដែលគាំទ្រ CUDA សម្រាប់ការគណនាទូទៅ - វិធីសាស្ត្រនេះហៅថា GPGPU (General-Purpose computing on Graphics Processing Units)។ CUDA គឺជាគន្លងសំខាន់សម្រាប់ការរុំរើស GPU របស់ Nvidia ហើយត្រូវបានប្រើយ៉ាងទូលំទូលាយនៅវិស័យនានា រួមបញ្ចូល machine learning, គណិតវិទ្យាវិទ្យាសាស្ត្រ និង video processing។

Hardware support សម្រាប់ CUDA កំណត់សម្រាប់ GPUs របស់ Nvidia ជាការពិតព្រោះវា​ជា​បច្ចេកវិទ្យា​ដែលផ្តល់​ដោយ Nvidia។ រាល់ស្ថាបត្យកម្មនីមួយៗគាំទ្រដល់ខ្នាតនៃ CUDA toolkit តាមកំណែ ដែលផ្តល់បណ្ណាល័យ និងឧបករណ៍ចាំបាច់សម្រាប់អ្នកអភិវឌ្ឍសម្រាប់កសាង និងរត់កម្មវិធី CUDA។

## ONNX

ONNX (Open Neural Network Exchange) គឺជាទ្រង់ទ្រាយបើកដែលបានออกแบบដើម្បីតំណាងម៉ូដែល machine learning។ វាផ្តល់នូវការបរិច្ឆេទនៃគំរូ computation graph ដែលអាចពង្រីកបាន ដែលដូចជាការបកស្រាយនៃ operators built-in និងប្រភេទទិន្នន័យស្តង់ដារ ONNX អនុញ្ញាតឲ្យអ្នកអភិវឌ្ឍផ្លាស់ប្តូរម៉ូដែលរវាង frameworks ML ផ្សេងៗ ដែលធ្វើឲ្យមានភាពអាចធ្វើការជាមួយគ្នា ហើយងាយស្រួលក្នុងការបង្កើត និងដាក់កម្មវិធី AI ។

Phi3 mini អាចរត់ជាមួយ ONNX Runtime លើ CPU និង GPU ចម្រង់ឧបករណ៍ផ្សេងៗ រួមមាន server platforms, Windows, Linux និង Mac desktops, និង CPU សម្រាប់ទូរស័ព្ទចល័ត។  
ការកំណត់តម្លែបង្កើតដែលយើងបានបន្ថែមមាន:

- ម៉ូដែល ONNX សម្រាប់ int4 DML៖ បម្លែងទៅ int4 តាមរយៈ AWQ  
- ម៉ូដែល ONNX សម្រាប់ fp16 CUDA  
- ម៉ូដែល ONNX សម្រាប់ int4 CUDA៖ បម្លែងទៅ int4 តាមរយៈ RTN  
- ម៉ូដែល ONNX សម្រាប់ int4 CPU និង Mobile៖ បម្លែងទៅ int4 តាមរយៈ RTN  

## Llama.cpp

Llama.cpp ជាបណ្ណាល័យម៉ាស៊ីនស្នូល code មួយដែលបានសរសេរជា C++។ វាធ្វើការបញ្ជាក់លទ្ធផលលើម៉ូដែលភាសាធំនានា (LLMs) រួមមាន Llama។ វាត្រូវបានអភិវឌ្ឍរួមជាមួយបណ្ណាល័យ ggml (បណ្ណាល័យ tensor សំរាប់គោលបំណងទូទៅ) ដែល Llama.cpp មានគោលបំណងផ្តល់នូវការបញ្ជាក់លទ្ធផលយឺតជាង និងប្រើប្រាស់កម្រិតមេម៉ូរីតិចជាង Python ដើម។ វាគាំទ្រការបង្កើត hardware, quantization ហើយផ្តល់ API ងាយស្រួល និងឧទាហរណ៍បី។ ប្រសិនបើអ្នកចាប់អារម្មណ៍លើការបង្កើត​លទ្ធផល LLM ដែលមានប្រសិទ្ធភាព ខ្ញុំបង្ហាញថា Llama.cpp គួរតែត្រូវបានពិចារណា ពីព្រោះ Phi3 អាចរត់ Llama.cpp បាន។

## GGUF

GGUF (Generic Graph Update Format) គឺជាទ្រង់ទ្រាយដែលប្រើសម្រាប់តំណាង និងធ្វើបច្ចុប្បន្នភាពម៉ូដែល machine learning។ វាពិសេសសម្រាប់ម៉ូដែលភាសា​តូច (SLMs) ដែលអាចរត់បានល្អលើ CPU ជាមួយការបម្លែង quantization 4-8bit។ GGUF មានប្រយោជន៍សម្រាប់ការបង្កើត​ដំណាក់កាលលឿន និងប្រតិបត្តិការម៉ូដែលលើឧបករណ៍កម្រិតជិត ឬក៏ក្នុងកិច្ចការប្រមូលជា batch ដូចជា CI/CD pipeline។
