# សេចក្តីសង្ខេបគម្រោង Phi-3-Vision-128K-Instruct

## ម៉ូដែល

Phi-3-Vision-128K-Instruct គឺជាម៉ូដែលពហុប្រភេទសាមញ្ញ និងទំនើបនៅចំណុចកណ្តាលគម្រោងនេះ។ វា ជាផ្នែកមួយនៃគ្រួសារម៉ូដែល Phi-3 ហើយគាំទ្រជ_LENGTH ប្រវែងពី Context រហូតដល់ 128,000 តួអក្សរ។ ម៉ូដែលនេះត្រូវបានបណ្តុះបណ្តាលលើទិន្នន័យនានាដែលរួមមានទិន្នន័យសំយុទ្ធ និងគេហទំព័រដែលអាចចូលដំណើរការបាននៅសាធារណៈ ដែលបានត្រួតពិនិត្យយ៉ាងម៉ត់ចត់ ដើម្បីផ្តោតលើមាតិកាគុណភាពខ្ពស់ និងទាមទារការពិចារណាខ្ពស់។ ដំណើរការបណ្តុះបណ្តាលរួមមានការត្រួតពិនិត្យដោយគ្រូបង្រៀន និងការបង្រួបបង្រួមការជ្រើសរើសដោយផ្ទាល់ ដើម្បីធានាការបន្តតាមសេចក្តីណែនាំយ៉ាងត្រឹមត្រូវ និងវិធានការសុវត្ថិភាពរឹងមាំ។

## ការបង្កើតទិន្នន័យគំរូមានសារសំខាន់ច្រើនបានដូចជា៖

1. **ការបង្ហាញតេស្ត**៖ ទិន្នន័យគំរូអនុញ្ញាតឱ្យអ្នកបង្ហាញតេស្តកម្មវិធីរបស់អ្នកនៅក្រោមស្ថានភាពផ្សេងៗដោយមិនប៉ះពាល់ទិន្នន័យពិត។ នេះមានសារៈសំខាន់យ៉ាងខ្លាំងនៅក្នុងដំណាក់កាលអភិវឌ្ឍន៍ និងដំណាក់កាលស្រោមភាគ។

2. **ការតម្រឹមប្រសិទ្ធភាព**៖ ជាមួយទិន្នន័យគំរូដែលเลียนបង្កើតបំណុលនិងភាពស្មុគស្មាញឆ្លុះបញ្ចាំងទិន្នន័យពិត អ្នកអាចសម្គាល់បញ្ហាបញ្ចប់ប្រសិទ្ធភាព និងបង្រួមបង្រួមកម្មវិធីរបស់អ្នកឱ្យបានល្អប្រសើរឡើង។

3. **ការបង្កើតគំរូដំបូង**៖ ទិន្នន័យគំរូអាចត្រូវប្រើសម្រាប់បង្កើតគំរូដំបូង និងគំរូចម្លាក់ ដែលអាចជួយយល់ពីតម្រូវការនៃអ្នកប្រើ និងទទួលបានមតិយោបល់។

4. **វិភាគទិន្នន័យ**៖ នៅក្នុងវិទ្យាសាស្ត្រ​ទិន្នន័យ ទិន្នន័យគំរូត្រូវបានប្រើសម្រាប់ការវិភាគទិន្នន័យបឋម ការបណ្តុះបណ្តាលម៉ូដែល និងការបង្ហាញតេស្តគណិតវិទ្យា។

5. **សន្តិសុខ**៖ ការប្រើទិន្នន័យគំរូនៅក្នុងបរិយាកាសអភិវឌ្ឍន៍និងតេស្តអាចជួយការពារការបញ្ចេញទិន្នន័យពិតដ៏សំខាន់ដោយអវិជ្ជមាន។

6. **ការសិក្សា**៖ ប្រសិនបើអ្នកកំពុងស្វែងយល់បច្ចេកវិទ្យាឬឧបករណ៍ថ្មី ការប្រើជាមួយទិន្នន័យគំរូអាចផ្តល់វិធីអនុវត្តវាជាមុន។

ចងចាំថា គុណភាពទិន្នន័យគំរូរបស់អ្នកអាចប៉ះពាល់យ៉ាងខ្លាំងដល់សកម្មភាពទាំងនេះ។ វាគួរតែមានភាពស្រដៀងទៅនឹងទិន្នន័យពិតក្នុងបរិបទរចនាសម្ព័ន្ធ និងភាពខុសគ្នា។

## សំណុំទិន្នន័យ

ឧទាហរណ៍ដែលល្អមួយសម្រាប់សំណុំទិន្នន័យគំរូគឺ [DBQ/Burberry.Product.prices.United.States dataset](https://huggingface.co/datasets/DBQ/Burberry.Product.prices.United.States) (មាននៅលើ Huggingface)។  
សំណុំទិន្នន័យគំរូផលិតផល Burberry រួមជាមួយនឹងមេតាដាតា​ស្តីពីប្រភេទផលិតផល តម្លៃ និងចំណងជើង មានជួរដេកចំនួន ៣,០៤០ ដែលមកពីផលិតផលដាច់ដោយឡែកមួយៗ។ សំណុំទិន្នន័យនេះអាចអោយយើងបញ្ចេញសមត្ថភាពម៉ូដែលក្នុងការយល់ និងបកប្រែទិន្នន័យវិជ្ជមាន ដោយបង្កើតអត្ថបទពិពណ៌នាដែលចាប់យកលម្អិតទាំងអស់នៃរូបភាព និងលក្ខណៈពិសេសរបស់ម៉ាក។

**ចំណាំ៖** អ្នកអាចប្រើសំណុំទិន្នន័យណាមួយដែលរួមមានរូបភាព។

## ការពិចារណាស្មុគស្មាញ

ម៉ូដែលត្រូវការបង្កើតការពិចារណាអំពីតម្លៃ និងឈ្មោះដែលផ្ដោតលើរូបភាពតែមួយ។ វាតម្រូវឱ្យម៉ូដែលមិនត្រឹមត្រូវចាប់យកលក្ខណៈមើលឃើញទេ ប៉ុន្តែត្រូវយល់ដឹងពីន័យរបស់វា ក្នុងបរិបទរបស់តម្លៃផលិតផល និងម៉ាក។ ដោយការសម្របសម្រួលការពិពណ៌នាអត្ថបទត្រឹមត្រូវពីរូបភាព គម្រោងនេះបង្ហាញពីសក្តានុពលនៃការរួមបញ្ចូលទិន្នន័យវិជ្ជមានដើម្បីបង្កើនសមត្ថភាព និងអាចប្រើប្រាស់ម៉ូដែលបានយ៉ាងទូលំទូលាយនៅក្នុងការអនុវត្តពិភពជាក់ស្តែង។

## ស្ថាបត្យកម្ម Phi-3 Vision

ស្ថាបត្យកម្មម៉ូដែលគឺជាកំណែកមួយនៃ Phi-3 ពហុប្រភេទ។ វាបំលែងទាំងអត្ថបទ និងទិន្នន័យរូបភាព ដើម្បីបញ្ចូលទាំងពីរជាលំដាប់រាយបញ្ជីតែមួយ សម្រាប់ការយល់ដឹង និងភារកិច្ចបង្កើត។ ម៉ូដែលប្រើស្រទាប់ embedding ផ្សេងពីរចំពោះអត្ថបទនិងរូបភាព។ តួអក្សរ​អត្ថបទត្រូវបានបម្លែងទៅជាផ្ទុកវ៉ិចទ័រ សម្រាប់រូបភាពត្រូវបានដំណើរការតាមម៉ូដែលឃ្លីប (CLIP vision model) ដើម្បីទាញយកលក្ខណៈ embedding។ embedding រូបនេះបន្ទាប់មកត្រូវបានបញ្ចូលជាមួយ embedding អត្ថបទ ដោយបំលែងទំហំឲ្យគ្នា ដើម្បីធានាថាអាចបញ្ចូលសម្រួលបាន។

## ការរួមបញ្ចូល embedding អត្ថបទ និងរូបភាព

តួអក្សរពិសេសជាកន្លែងបង្ហាញភាគរយ embedding រូបភាព ដែលត្រូវបញ្ជូលទៅក្នុងលំដាប់អត្ថបទ។ នៅពេលដំណើរការបានតែតួអក្សរពិសេសទាំងនោះ ត្រូវបានជំនួសជាមួយ embedding រូបភាព ដែលធ្វើអោយម៉ូដែលអាចដោះស្រាយអត្ថបទ និងរូបភាពជា​ដំណើរតែមួយ។ prompt សម្រាប់សំណុំទិន្នន័យរបស់យើងត្រូវបានទ្រង់ទ្រាយដោយប្រើ token ពិសេស <|image|> ដូចជា៖

```python
text = f"<|user|>\n<|image_1|>What is shown in this image?<|end|><|assistant|>\nProduct: {row['title']}, Category: {row['category3_code']}, Full Price: {row['full_price']}<|end|>"
```
