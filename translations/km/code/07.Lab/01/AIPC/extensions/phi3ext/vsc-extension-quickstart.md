# សូមស្វាគមន៍មកកាន់ការពង្រីក VS Code របស់អ្នក

## តើមានអ្វីនៅក្នុងថតនេះ

* ថតនេះមានឯកសារទាំងអស់ដែលចាំបាច់សម្រាប់ការពង្រីករបស់អ្នក។
* `package.json` - នេះជាឯកសារមានសារៈសំខាន់ដែលអ្នកប្រកាសពីការពង្រីក និងកាម៉ង់របស់អ្នក។
  * ផ្លាដា ឧទាហរណ៍ចុះបញ្ជីកាម៉ង់មួយ ហើយកំណត់ចំណងជើងនិងឈ្មោះកាម៉ង់។ ជាមួយព័ត៌មាននេះ VS Code អាចបង្ហាញកាម៉ង់នៅក្នុងប៉ាឡែតកាម៉ង់។ វាមិនចាំបាច់ផ្ទុកផ្លាដានោះទេនៅឡើយ។
* `src/extension.ts` - នេះជាឯកសារចម្បងដែលអ្នកនឹងផ្ដល់ការអនុវត្តន៍នៃកាម៉ង់របស់អ្នក។
  * ឯកសារ​នេះនាំចេញមុខងារ​មួយ គឺ `activate` ដែលត្រូវបានហៅជាលើកដំបូង នៅពេលដែលការពង្រីករបស់អ្នកត្រូវបានបើក (ក្នុងករណីនេះដោយអនុវត្តកាម៉ង់)។នៅខាងក្នុងមុខងារ `activate` យើងហៅ `registerCommand`។
  * យើងផ្ញើមុខងារដែលមានការអនុវត្តន៍នៃកាម៉ង់ជាអាគុយម៉ង់ទីពីរទៅ `registerCommand`។

## ការតំឡើង

* ដំឡើងការពង្រីកដែលបានផ្ដល់អនុសាសន៍ (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, និង dbaeumer.vscode-eslint)


## ចាប់ផ្តើមរត់ភ្លាមៗ

* ចុច `F5` ដើម្បីបើកបង្អួចថ្មីជាមួយនឹងការពង្រីករបស់អ្នកបានផ្ទុក។
* រត់កាម៉ង់របស់អ្នកពីប៉ាឡែតកាម៉ង់ដោយចុច (`Ctrl+Shift+P` ឬ `Cmd+Shift+P` នៅលើ Mac) ហើយវាយពាក្យ `Hello World`។
* កំណត់ចំណុចព្រិចនៅក្នុងកូដរបស់អ្នកនៅក្នុង `src/extension.ts` ដើម្បីដោះស្រាយកំហុសនៃការពង្រីក។
* រកមើលលទ្ធផលពីការពង្រីករបស់អ្នកនៅក្នុងកូនសោតដំណោះស្រាយកំហុស។

## ប្រែក្លាយ

* អ្នកអាចចាប់ផ្តើមការពង្រីកម្តងទៀតពីរបារឧបករណ៍ដោះស្រាយកំហុសបន្ទាប់ពីផ្លាស់ប្តូរកូដនៅក្នុង `src/extension.ts`។
* អ្នកក៏អាចផ្ទុកឡើងវិញបង្អួច VS Code (Ctrl+R ឬ Cmd+R នៅលើ Mac) ជាមួយនឹងការពង្រីករបស់អ្នកដើម្បីផ្ទុកការផ្លាស់ប្តូររបស់អ្នក។

## រុករក API

* អ្នកអាចបើកសំណុំ API ពេញលេញរបស់យើងនៅពេលបើកឯកសារ `node_modules/@types/vscode/index.d.ts`។

## រ៉ន់តេស្ត

* ដំឡើង [Extension Test Runner](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner)
* រត់ភារកិច្ច "watch" តាមរយៈកាម៉ង់ **Tasks: Run Task**។ ត្រូវប្រាកដថានេះកំពុងដំណើរការ មិនដូច្នោះតេស្តអាចមិនត្រូវរកឃើញ។
* បើកទិដ្ឋភាព Testing ពីរបារហ្វិសកម្មហើយចុចប៊ូតុង Run Test ឬប្រើក្តារចុចក្តៅ `Ctrl/Cmd + ; A`
* មើលលទ្ធផលនៃតេស្តនៅក្នុងទិដ្ឋភាព Test Results។
* ប្រែក្លាយទៅ `src/test/extension.test.ts` ឬបង្កើតឯកសារតេស្តថ្មីនៅក្នុងថត `test`។
  * អ្នករត់តេស្តដែលបានផ្ដល់នឹងពិនិត្យឯកសារដែលមានឈ្មោះត្រូវនឹងលំនាំ `**.test.ts` ប៉ុណ្ណោះ។
  * អ្នកអាចបង្កើតថតនៅក្នុងថត `test` ដើម្បីរៀបចំតេស្តរបស់អ្នកក្នុងវិធីណាមួយដែលអ្នកចង់បាន។

* [ផ្សព្វផ្សាយការពង្រីករបស់អ្នក](https://code.visualstudio.com/api/working-with-extensions/publishing-extension?WT.mc_id=aiml-137032-kinfeylo) នៅលើទីផ្សារការពង្រីក VS Code។
