# សូមស្វាគមន៍​មកកាន់​ការ​ពង្រីក VS Code របស់អ្នក

## មានអ្វីនៅក្នុងថត

* ថតនេះ​មាន​ឯកសារ​ទាំងអស់​ដែលចាំបាច់សម្រាប់ការ​ពង្រីក​របស់​អ្នក។
* `package.json` - នេះ​គឺ​ជា​ឯកសារ​មេដែលអ្នកប្រកាស​អំពី​ការ​ពង្រីក និង​ការបញ្ជា​របស់​អ្នក។
  * ផ្លាក់អាំង​តំណាងនេះ​បានចុះបញ្ជី​ការ​បញ្ជា​ដើម្បីកំណត់ចំណងជើង និងឈ្មោះ​ការ​បញ្ជា។ ជាមួយព័ត៌មាននេះ VS Code អាច​បង្ហាញ​ការ​បញ្ជា​នៅក្នុង​បញ្ជីការ​បញ្ជា។ វាមិនត្រូវការ​ដំណើរការ​ផ្លាក់អាំងនៅឡើយទេ។
* `src/extension.ts` - នេះគឺ​ឯកសារ​ចម្បង​ដែលអ្នក​នឹងផ្ដល់​ការ​អនុវត្ត​ការបញ្ជា​របស់​អ្នក។
  * ឯកសារ​នេះនាំចេញមុខងារ​មួយ, `activate`, ដែល​នឹង​ត្រូវ​បានហៅជាលើកដំបូង​ពេល​ការ​ពង្រីក​របស់​អ្នក​ត្រូវ​បាន​សកម្ម (នៅក្នុងករណីនេះ​ដោយ​ការ​អនុវត្ត​ការ​បញ្ជា)។ នៅក្នុងមុខងារ `activate` យើងហៅ `registerCommand`។
  * យើង​ផ្ដល់​មុខងារ​ដែល​មាន​ការ​អនុវត្ត​នៃ​ការ​បញ្ជា​ជា​ប៉ារ៉ាម៉ែត្រ​ទីពីរ​ទៅ `registerCommand`។

## ការតំឡើង

* តំឡើងការ​ពង្រីក​ដែលបាន​ផ្ដល់​ណែនាំ (amodio.tsl-problem-matcher, ms-vscode.extension-test-runner, និង dbaeumer.vscode-eslint)


## ចាប់ផ្តើមបន្ទាន់​ខ្លាំង

* ចុច `F5` ដើម្បីបើកបង្អួច​ថ្មី​ជា​ភាពដែល​មាន​ការ​ពង្រីក​របស់​អ្នក។
* រត់​បញ្ជា​របស់​អ្នក​ពី​បញ្ជីការ​បញ្ជា​ដោយចុច (`Ctrl+Shift+P` ឬ `Cmd+Shift+P` នៅលើ Mac) ហើយវាយ `Hello World`។
* កំណត់​អត្តសញ្ញាណ​ចំនុចបំបែក​នៅ​ក្នុង​កូដ​របស់​អ្នក​នៅ​ក្នុង `src/extension.ts` ដើម្បី​ធ្វើ​ការ​ដំណើរការ​ដុបខុសត្រូវ​នៃ​ការ​ពង្រីក​របស់អ្នក។
* រកមើលលទ្ឋផល​ពី​ការ​ពង្រីក​របស់អ្នក​នៅ​ក្នុង​កុងសូល​ដាប់ឌី ដាច់។

## ធ្វើ​បម្រែ​បម្រួល

* អ្នក​អាច​បើកមុខងារ​ការ​ពង្រីក​ពី​បញ្ជីឧបករណ៍​ដាប់ឌី​បន្ទាប់ពីបម្លែងកូដ​នៅក្នុង `src/extension.ts`។
* អ្នក​ក៏អាច​ផ្ទុកឡើងវិញ (`Ctrl+R` ឬ `Cmd+R` នៅលើ Mac) បង្អួច VS Code ជាមួយ​ការ​ពង្រីក​របស់​អ្នក ដើម្បី​ផ្ទុកបម្រែបម្រួល​របស់អ្នកវិញ។

## ស្វែងយល់អំពី API

* អ្នកអាច​បើកសំណុំ API ពេញលេញរបស់យើង​ពេលអ្នក​បើកឯកសារ `node_modules/@types/vscode/index.d.ts`។

## បើក​ឆែក​តេស្ត

* តំឡើង [Extension Test Runner](https://marketplace.visualstudio.com/items?itemName=ms-vscode.extension-test-runner)
* រត់ភារកិច្ច "watch" តាមរយៈការបញ្ជា **Tasks: Run Task**។ ត្រូវពិនិត្យមើលថា គឺកំពុងរត់ ឬ​មិនបញ្ចេញ លទ្ធផលតេស្ត​នៅតាមកាលវិភាគ។
* បើក​មើល Testing view ពី activity bar ហើយចុចប៊ូតុង "Run Test" ឬ​ប្រើ​សញ្ញាចុចរហ្នឹង `Ctrl/Cmd + ; A`
* មើលលទ្ឋផលនៃ​លទ្ធផល​តេស្ត​នៅក្នុង Test Results view។
* ធ្វើ​បម្រែ​បម្រួល​ទៅ `src/test/extension.test.ts` ឬ​បង្កើត​ឯកសារ​តេស្ត​ថ្មី​នៅ​ក្នុងថត `test`។
  * អ្នក​រត់តេស្ត​ដែល​បានផ្ដល់​នឹងពិនិត្យតែ​ឯកសារ​ដែលមាន​ឈ្មោះ​តាម​គម្រប `**.test.ts` ប៉ុណ្ណោះ។ 
  * អ្នក​អាច​បង្កើតថត​នៅ​ក្នុងថត `test` ដើម្បីរៀបចំ​តេស្ត​ដោយរបៀបណាមួយដែលអ្នក​ចង់​បាន។

## ធ្វើ​ទៅមុខ​បន្ថែម

* បន្ថយទំហំ​ការ​ពង្រីក និងលឿន​ធ្វើការ​ចាប់ផ្តើម​កាន់តែប្រសើរដោយ [បញ្ចូល​ការ​ពង្រីក​របស់អ្នកជាកញ្ចប់](https://code.visualstudio.com/api/working-with-extensions/bundling-extension)។
* [បោះពុម្ព​ការ​ពង្រីក​របស់អ្នក](https://code.visualstudio.com/api/working-with-extensions/publishing-extension) នៅលើផ្សារទំនិញកម្មវិធីពង្រីក VS Code។
* ផ្សងព្រេងនឹង​ការ​បង្កើត​ដោយ​ការតំឡើង [Continuous Integration](https://code.visualstudio.com/api/working-with-extensions/continuous-integration)។
