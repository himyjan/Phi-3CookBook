# Phi perekond AITK-s

[AI tööriistakomplekt VS Code jaoks](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio) lihtsustab generatiivsete tehisintellekti rakenduste arendamist, tuues kokku tipptasemel AI arendusvahendid ja mudelid Microsoft Foundry kataloogist ning muudest kataloogidest nagu Hugging Face. Saate sirvida GitHubi mudelite ja Microsoft Foundry mudelikataloogide toega AI mudelite kataloogi, alla laadida mudeleid kohalikult või kaugelt, neid peenhäälestada, testida ja kasutada oma rakenduses.

AI tööriistakomplekti eelvaade töötab lokaalselt. Kohalik järeldamine või peenhäälestus sõltub valitud mudelist, teil võib olla vaja GPU-d nagu NVIDIA CUDA GPU. Saate samuti käivitada GitHubi mudeleid otse AITK-ga.

### AI tööriistakomplekti installimine

AI tööriistakomplekt on tarnitud kui [Visual Studio Code laiendus](https://code.visualstudio.com/docs/setup/additional-components#_vs-code-extensions), seega tuleb teil kõigepealt installida [VS Code](https://code.visualstudio.com/docs/setup/windows?WT.mc_id=aiml-137032-kinfeylo) ja alla laadida AI tööriistakomplekt [VS Marketplace'ist](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio).
[AI tööriistakomplekt on saadaval Visual Studio Marketplace'is](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio) ja seda saab installida nagu iga teist VS Code'i laiendust.

Kui teile ei ole tuttav VS Code laienduste installimine, järgige neid samme:

### Sisselogimine

1. Valige VS Code tegevusribal **Laiendused**
1. Tippige Laienduste otsinguribale "AI Toolkit"
1. Valige "AI Toolkit for Visual Studio code"
1. Valige **Installi**

Nüüd olete valmis laiendust kasutama!

Teil palutakse sisselogimiseks GitHubi, nii et palun vajutage "Allow" jätkamiseks. Teid suunatakse GitHubi sisselogimise lehele.

Palun logige sisse ja järgige protsessi samme. Pärast edukat lõpetamist suunatakse teid tagasi VS Code'i.

Kui laiend on installitud, näete AI tööriistakomplekti ikooni oma tegevusribal.

Vaatame saadaolevaid toiminguid!

> [!NOTE]
>
> **GPU vs CPU**
>
> Märkate, et mudelikeardid näitavad mudeli suurust, platvormi ja kiirendi tüüpi (CPU, GPU). Optimaalse jõudluse tagamiseks **Windowsi seadmetes, millel on vähemalt üks GPU**, valige ainult Windowsi sihtivad mudeliversioonid.
>
> See tagab mudeli optimeerimise DirectML kiirendile.
>
> Mudelinimed on kujul
>
> - `{mudeli_nimi}-{kiirendi}-{kvantiseerimine}-{vorming}`.
>
>GPU olemasolu Windowsi seadmes saate kontrollida, avades **Task Manageri** ja valida seejärel **Performance** vahekaardi. Kui teil on GPU(d), kuvatakse need nimetustega nagu "GPU 0" või "GPU 1".

### Kasutage oma rakenduses REST API-d

AI tööriistakomplekt sisaldab kohalikku REST API veebi serverit **pordil 5272**, mis kasutab [OpenAI vestluse täitmise vormingut](https://platform.openai.com/docs/api-reference/chat/create).

See võimaldab teil testida oma rakendust kohapeal ilma, et peaksite tuginema pilve AI mudeliteenusele. Näiteks järgmine JSON-fail näitab, kuidas konfigureerida päringu sisu:

```json
{
    "model": "Phi-4",
    "messages": [
        {
            "role": "user",
            "content": "what is the golden ratio?"
        }
    ],
    "temperature": 0.7,
    "top_p": 1,
    "top_k": 10,
    "max_tokens": 100,
    "stream": true
}
```

Võite REST API-t testida näiteks [Postmaniga](https://www.postman.com/) või CURL-utilityga:

```bash
curl -vX POST http://127.0.0.1:5272/v1/chat/completions -H 'Content-Type: application/json' -d @body.json
```

### Pythonis kasutades OpenAI klienditeeki

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:5272/v1/", 
    api_key="x" # nõutav API jaoks, kuid mitte kasutusel
)

chat_completion = client.chat.completions.create(
    messages=[
        {
            "role": "user",
            "content": "what is the golden ratio?",
        }
    ],
    model="Phi-4",
)

print(chat_completion.choices[0].message.content)
```

### .NET-is kasutades Azure OpenAI klienditeeki

Lisage [Azure OpenAI klienditeek .NET-ile](https://www.nuget.org/packages/Azure.AI.OpenAI/) oma projekti NuGeti kaudu:

```bash
dotnet add {project_name} package Azure.AI.OpenAI --version 1.0.0-beta.17
```

Lisage oma projekti C# fail nimega **OverridePolicy.cs** ja kleepige järgmine kood:

```csharp
// OverridePolicy.cs
using Azure.Core.Pipeline;
using Azure.Core;

internal partial class OverrideRequestUriPolicy(Uri overrideUri)
    : HttpPipelineSynchronousPolicy
{
    private readonly Uri _overrideUri = overrideUri;

    public override void OnSendingRequest(HttpMessage message)
    {
        message.Request.Uri.Reset(_overrideUri);
    }
}
```

Seejärel kleepige järgmine kood oma **Program.cs** faili:

```csharp
// Program.cs
using Azure.AI.OpenAI;

Uri localhostUri = new("http://localhost:5272/v1/chat/completions");

OpenAIClientOptions clientOptions = new();
clientOptions.AddPolicy(
    new OverrideRequestUriPolicy(localhostUri),
    Azure.Core.HttpPipelinePosition.BeforeTransport);
OpenAIClient client = new(openAIApiKey: "unused", clientOptions);

ChatCompletionsOptions options = new()
{
    DeploymentName = "Phi-4",
    Messages =
    {
        new ChatRequestSystemMessage("You are a helpful assistant. Be brief and succinct."),
        new ChatRequestUserMessage("What is the golden ratio?"),
    }
};

StreamingResponse<StreamingChatCompletionsUpdate> streamingChatResponse
    = await client.GetChatCompletionsStreamingAsync(options);

await foreach (StreamingChatCompletionsUpdate chatChunk in streamingChatResponse)
{
    Console.Write(chatChunk.ContentUpdate);
}
```
