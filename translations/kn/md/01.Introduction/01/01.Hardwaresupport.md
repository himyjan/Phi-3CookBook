# ಫೈ ಹಾರ್ಡ್‌‌ವೇರ್ ಬೆಂಬಲ

Microsoft Phi ಅನ್ನು ONNX Runtimeಗಾಗಿ ತಕ್ಕಮಟ್ಟಿಗೆ ಅನुकूलಗೊಳಿಸಲಾಗಿದೆ ಮತ್ತು Windows DirectML ಅನ್ನು ಬೆಂಬಲಿಸಿದೆ. ಇದು GPUಗಳು, CPUಗಳು ಮತ್ತು ಮಾನ್ಯ ಮೊಬೈಲ್ ಸಾಧನಗಳ ಸೇರಿದಂತೆ ವಿವಿಧ ಹಾರ್ಡ್‌‌ವೇರ್ ರೀತಿಗಳಲ್ಲಿ ಚೆನ್ನಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ.

## ಸಾಧನ ಹಾರ್ಡ್‌‌ವೇರ್  
ನಿರ್ದಿಷ್ಟವಾಗಿ, ಬೆಂಬಲಿತ ಹಾರ್ಡ್‌‌ವೇರ್ ಈ ಕೆಳಗಿನವುಗಳನ್ನು ಒಳಗೊಂಡಿದೆ:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPUs, 128 GiB ಮೆಮೊರಿ)

## мобೈಲ್ SKU

- ಆಂಡ್ರಾಯ್ಡ್ - Samsung Galaxy S21
- ಆಪಲ್ iPhone 14 ಅಥವಾ ಹೆಚ್ಚಿನ A16/A17 ಪ್ರೊಸೆಸರ್

## ಫೈ ಹಾರ್ಡ್‌‌ವೇರ್ ವೈಶಿಷ್ಟ್ಯಗಳು

- ಕನಿಷ್ಠ ಸಂರಚನೆಯ ಅಗತ್ಯವಿದೆ.
- ವಿಂಡೋಸ್: DirectX 12-ಸಾಧ್ಯ GPU ಮತ್ತು ಕನಿಷ್ಠ 4GB ಸಂಯುಕ್ತ RAM

CUDA: ಗಣನ ಸಾಮರ್ಥ್ಯ >= 7.02 ಇರುವ NVIDIA GPU

## ಹಲವಾರು GPUಗಳು ಮೇಲೆ onnxruntime ನಡೆಸುವುದು

ಪ್ರಸ್ತುತ ಲಭ್ಯವಿರುವ Phi ONNX ಮಾದರಿಗಳು ಕೇವಲ 1 GPUಗಾಗಿ ಮಾತ್ರ. Phi ಮಾದರಿಗಾಗಿ ಮಲ್ಟಿ-GPU ಬೆಂಬಲ ನಡಿಸಲು ಸಾಧ್ಯವೇ ಇರುತ್ತದೆ, ಆದರೆ 2 GPUಗಳೊಂದಿಗೆ ORT ಹೆಚ್ಚು ಅನುಪಾತ ನೀಡುತ್ತದೆ ಎಂದು ಖಚಿತಪಡಿಸಲಾಗುವುದಿಲ್ಲ. ದಯವಿಟ್ಟು [ONNX Runtime](https://onnxruntime.ai/) ನ ಇತ್ತೀಚಿನ ನವೀಕರಣಗಳನ್ನು ನೋಡಿ.

[Build 2024 the GenAI ONNX Team](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) ನಲ್ಲಿ Phi ಮಾದರಿಗಳಿಗೆ ಮಲ್ಟಿ-GPU ಬದಲಿಗೆ ಮಲ್ಟಿ-ಇನ್ಸ್ಟನ್ಸ್ ಸಕ್ರಿಯಗೊಳಿಸಿದ ಮಾಹಿತಿಯನ್ನು ಪ್ರಕಟಿಸಲಾಗಿದೆ.

ಈಗ ನೀವು CUDA_VISIBLE_DEVICES ಪರಿಸರ变量ದೊಂದಿಗೆ ಒಂದು onnxruntime ಅಥವಾ onnxruntime-genai ಇನ್ಸ್ಟನ್ಸ್ ಅನ್ನು ಈ ರೀತಿಯಾಗಿ ಚಲಾಯಿಸಬಹುದು.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

ಕೃಪಯಾ Phi ಅನ್ನು ಮತ್ತಷ್ಟು ಅನ್ವೇಷಿಸಲು [Microsoft Foundry](https://ai.azure.com) ನಲ್ಲಿರಿ.
