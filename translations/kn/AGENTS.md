# AGENTS.md

## Project Overview

PhiCookBook ಮೈಸೂರುಗಳ Microsoft Phi ಕುಟುಂಬದ ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳ (SLMs)ೊಂದಿಗೆ ಕೆಲಸ ಮಾಡಲು ಕೈಗೊಳ್ಳುವ ಉದಾಹರಣೆಗಳು, ಪಾಠಗಳು ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಅನ್ನು ಒಳಗೊಂಡ ಸಮಗ್ರ ಕುಕ್‌ಬುಕ್ ಸಂಗ್ರಹವಾಗಿದೆ. ಈ ಸಂಗ್ರಹವು ಇನ್ಫರೆನ್ಸ್, ಫೈನ್-ಟ್ಯೂನಿಂಗ್, ಕ್ವಾಂಟೈಜೇಶನ್, RAG ಅನುಷ್ಠಾನಗಳು ಮತ್ತು ಬಹುಮಾಧ್ಯಮ ಅಪ್ಲಿಕೇಶನ್ಗಳನ್ನು ವಿವಿಧ ವೇದಿಕೆಗಳು ಮತ್ತು ಫ್ರೇಂವರ್ಕ್‌ಗಳ ಮೇಲೆ ಪ್ರದರ್ಶಿಸುತ್ತದೆ.

**ಪ್ರಮುಖ ತಂತ್ರಜ್ಞಾನಗಳು:**
- **ಭಾಷೆಗಳು:** Python, C#/.NET, JavaScript/Node.js
- **ಫ್ರೇಂವರ್ಕ್‌ಗಳು:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **ವೇದಿಕೆಗಳು:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **ಮಾದರಿ ಪ್ರಕಾರಗಳು:** Phi-3, Phi-3.5, Phi-4 (ಪಠ್ಯ, ದೃಷ್ಟಿ, ಬಹೂಮಾಧ್ಯಮ, ನಿರ್ಣಯಾತ್ಮಕ ರೂಪಾಂತರಗಳು)

**ಸಂಗ್ರಹ ರಚನೆ:**
- `/code/` - ಕೆಲಸದ ಕೋಡ್ ಉದಾಹರಣೆಗಳು ಮತ್ತು ಮಾದರಿ ಅನುಷ್ಠಾನಗಳು
- `/md/` - ವಿವರವಾದ ಡಾಕ್ಯುಮೆಂಟೇಶನ್, ಪಾಠಗಳು ಮತ್ತು ಹೇಗೆ ಮಾಡುವುದು ಮಾರ್ಗದರ್ಶನಗಳು  
- `/translations/` - ಬಹುಭಾಷಾ ಅನುವಾದಗಳು (50+ ಭಾಷೆಗಳು ಸ್ವಯಂಚಾಲಿತ ಕಾರ್ಯಪ್ರವಾಹದ ಮೂಲಕ)
- `/.devcontainer/` - ಡೇವ್ ಕಂಟೈನರ್ ಸಂರಚನೆ (Python 3.12 Ollama ಜೊತೆಗೆ)

2. VS Code ಡೇವ್ ಕಂಟೈನರ್‌ಗಳಲ್ಲಿ ತೆರೆಯಿರಿ:

### ಸ್ಥಳೀಯ ಸೆಟ್‌ಅಪ್

**ಅಗತ್ಯತೆಗಳು:**
- Python 3.12 ಅಥವಾ ನಂತರದ ಆವೃತ್ತಿ
- .NET 8.0 SDK (C# ಉದಾಹರಣೆಗಳಿಗೆ)
- Node.js 18+ ಮತ್ತು npm (JavaScript ಉದಾಹರಣೆಗೆ)
- ಕನಿಷ್ಠ 16GB RAM ಶಿಫಾರಸು

**ಸ್ಥಾಪನೆ:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**Python ಉದಾಹರಣೆಗಳಿಗಾಗಿ:**
ನಿರ್ದಿಷ್ಟ ಉದಾಹರಣಾ ಡೈರೆಕ್ಟರಿಗಳಲ್ಲಿ ತೆರಳಿ ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # requirements.txt ಇದ್ದರೆ
```

**.NET ಉದಾಹರಣೆಗಳಿಗಾಗಿ:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**JavaScript/Web ಉದಾಹರಣೆಗಳಿಗಾಗಿ:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # ಅಭಿವೃದ್ಧಿ ಸರ್ವರ್ ಪ್ರಾರಂಭಿಸಿ
npm run build  # ಉತ್ಪಾದನೆಗಾಗಿ ನಿರ್ಮಿಸಿ
```

## Repository Organization

- **03.Inference/** - ವಿಭಿನ್ನ ಹಾರ್ಡ್‌ವೇರ್‌ಗಳ (AIPC, MLX) ಮೇಲೆ ಇನ್ಫರೆನ್ಸ್ ಉದಾಹರಣೆಗಳು
- **06.E2E/** - ಸಂಪೂರ್ಣ ಅಪ್ಲಿಕೇಶನ್ ಮಾದರಿಗಳು
- **07.Lab/** - ಪ್ರಯೋಗಶಾಲೆ/ಪ್ರಾಯೋಗಿಕ ಅನುಷ್ಠಾನಗಳು
- **08.RAG/** - ರಿಟ್ರೀವಲ್-ಆಗ್ಮೆಂಟೆಡ್ ಜನರೇಶನ್ ಮಾದರಿಗಳು
- **09.UpdateSamples/** - ಇತ್ತೀಚಿನ ನವೀಕರಿಸಿದ ಮಾದರಿಗಳು

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ (`/md/`)

- **01.Introduction/** - ಪರಿಚಯ ಮಾರ್ಗದರ್ಶಿಗಳು, ಪರಿಸರ ಸೆಟ್‌ಅಪ್, ವೇದಿಕೆ ಮಾರ್ಗದರ್ಶನಗಳು
- **02.Application/** - ವಿಧಾನದ ಪ್ರಕಾರ ಜೋಡಿಸಲಾದ ಅಪ್ಲಿಕೇಶನ್ ಮಾದರಿಗಳು (ಪಠ್ಯ, ಕೋಡ್, ದೃಷ್ಟಿ, ಧ್ವನಿ, ಇತ್ಯಾದಿ)
- **02.QuickStart/** - Microsoft Foundry ಮತ್ತು GitHub Models ನ ವೇಗದ ಪ್ರಾರಂಭ ಮಾರ್ಗದರ್ಶನಗಳು
- **03.FineTuning/** - ಫೈನ್-ಟ್ಯೂನಿಂಗ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಮತ್ತು ಪಾಠಗಳು
- **04.HOL/** - ಕೈಬಿಡುವ ಪ್ರಯೋಗಾಲಯಗಳು (.NET ಉದಾಹರಣೆಗಳೊಂದಿಗೆ)

### ಕಡತ ರೂಪಗಳು

## ಪರೀಕ್ಷೆ

ಈ ಸಂಗ್ರಹವು ಪರಂಪರাগত ಸಾಫ್ಟ್‌ವೇರ್ ಪ್ರಾಜೆಕ್ಟ್ ಜೊತೆಗೆ ಯೂನಿಟ್ ಟೆಸ್ಟ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿದ್ದಿಲ್ಲ, ಬದಲಿಗೆ ಉದಾಹರಣೆ ಕೋಡ್ ಮತ್ತು ಪಾಠಗಳನ್ನು ಹೊಂದಿದೆ. ಮಾನ್ಯತೆಯನ್ನು ಸಾಮಾನ್ಯವಾಗಿ ಕೆಳಗಿನಂತೆ ಮಾಡಲಾಗುತ್ತದೆ:

1. **ಉದಾಹರಣೆಗಳನ್ನು ಚಾಲನೆಯಲ್ಲಿಡು** - ಪ್ರತಿ ಉದಾಹರಣೆ ದೋಷರಹಿತವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸಬೇಕು
2. **ಫಲಿತಾಂಶಗಳನ್ನು ಪರಿಶೀಲಿಸು** - ಮಾದರಿ ಪ್ರತಿಕ್ರಿಯೆಗಳು ಸೂಕ್ತವಾಗಿರಬೇಕು
3. **ಪಾಠಗಳ ಅನುಸರಣೆ** - ಹಂತಗಳ ಮೂಲಕ ಮಾರ್ಗದರ್ಶಿಗಳು ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪ್ರಕಾರ ಕೆಲಸ ಮಾಡಬೇಕು

**ಸಾಮಾನ್ಯ ಮಾನ್ಯತೆ ವಿಧಾನ:**
- ಗುರಿ ಪರಿಸರದಲ್ಲಿ ಉದಾಹರಣೆಯ ಕಾರ್ಯಾಚರಣೆ ಪರೀಕ್ಷೆ
- ಅವಲಂಬನೆಗಳ ಸರಿಯಾದ ಸ್ಥಾಪನೆ ಪರಿಶೀಲನೆ
- ಮಾದರಿ ಡೌನ್‌ಲೋಡ್/ಲೋಡ್ ಯಶಸ್ವಿಯಾಗಿದೆಯೇ ಎಂದು ಪರಿಶೀಲನೆ
- ನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ಗೆ ಅನುಗುಣವಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

   - ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನವೀಕರಣಗಳು ಬೇರೆ PR ನಲ್ಲಿ
   - ಹೊಸ ಉದಾಹರಣೆಗಳು ಪ್ರತ್ಯೇಕ PR ಗಳಲ್ಲಿ
   - ಟೈಪೋ ದೋಷಗಳು ಸಂಯೋಜಿಸಬಹುದು

3. **ಮರ್ಡ್ ಘರ್ಷಣೆಗಳನ್ನು ನಿರ್ವಹಿಸಿ:**
   - ಬದಲಾವಣೆಯ ಮೊದಲು ನಿಮ್ಮ ಸ್ಥಳೀಯ `main` ಶಾಖೆಯನ್ನು ನವೀಕರಿಸಿ
   - ಅಪ್‌ಸ್ಟ್ರೀಮ್‌ನೊಂದಿಗೆ ನಿಯಮಿತವಾಗಿ ಸಮಿಕಾಲೀನ ಮಾಡಿ

4. **ಅನುವಾದ PR ಗಳು:**
   - ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳ ಅನುವಾದಗಳನ್ನು ಹೊಂದಿರಬೇಕು
   - ಮೂಲ ಭಾಷೆಯ ಸಂರಚನೆಯೊಂದಿಗೆ ಹೊಂದಾಣಿಕೆ ಹೊಂದಿರಬೇಕು

### ಅಗತ್ಯ چೆಕ್‌ಗಳು

PRಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ GitHub ಕಾರ್ಯಪ್ರವಾಹಗಳನ್ನು ಚಾಲನೆ ಮಾಡಿ ಮಾನ್ಯತೆಯನ್ನು ಮರುಪರಿಶೀಲನೆ ಮಾಡುತ್ತವೆ:

1. **ಸಾಪೇಕ್ಷ ಮಾರ್ಗ ಮಾನ್ಯತೆ** - ಎಲ್ಲಾ ಆಂತರಿಕ ಲಿಂಕ್ಗಳು ಕಾರ್ಯನಿರ್ವಹಿಸಬೇಕು
   - ಸ್ಥಳೀಯವಾಗಿลಿಂಕ್ಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ: VS Code ನಲ್ಲಿ Ctrl+Click ಬಳಸಿ
   - VS Code ನಿಂದ ಮಾರ್ಗ ಸಲಹೆಗಳು ಬಳಸಿ (`./` ಅಥವಾ `../`)

2. **URL ಸ್ಥಳೀಯತೆ ಪರಿಶೀಲನೆ** - ವೆಬ್ URL ಗಳು ದೇಶ-ನಿರ್ದಿಷ್ಟ ಸ್ಥಳೀಯತೆ ಹೊಂದಬಾರದು
   - `/en-us/`, `/en/` ಅಥವಾ ಇತರ ಭಾಷಾ ಕೋಡ್‌ಗಳನ್ನು ತೆಗೆದುಹಾಕಿ
   - ಸಾಮಾನ್ಯ ಅಂತರರಾಷ್ಟ್ರೀಯ URL ಗಳು ಬಳಸಿ

3. **ತಪ್ಪಾದ URL ಪರಿಶೀಲನೆ** - ಎಲ್ಲಾ URL ಗಳು 200 ಸ್ಥಿತಿ ಕಳಿಸಲು ಹೊಂದಿರಬೇಕು
   - ಸಲ್ಲಿಸುವ ಮೊದಲು ಲಿಂಕ್ಗಳು ಲಭ್ಯವಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಿ
   - ಕೆಲವೊಮ್ಮೆ ವೈಫೈ ಅಥವಾ ಅಂತರಜಾಲ ನಿರ್ಬಂಧಗಳ ಕಾರಣದಿಂದ ತಪ್ಪಬಹುದು
