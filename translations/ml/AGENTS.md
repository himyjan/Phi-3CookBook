# AGENTS.md

## Project Overview

PhiCookBook Microsoft-ന്റെ Phi കുടുംബത്തിലെ ചെറിയ ഭാഷാ മോഡലുകളുമായി (SLMs) പ്രവർത്തിക്കാനുള്ള പ്രായോഗിക ഉദാഹരണങ്ങൾ, പാഠങ്ങൾ, ഡോക്യുമെന്റേഷൻ എന്നിവ അടങ്ങിയ സമഗ്രമായ കുക്ക്ബുക്ക് സംഭരണിയാണ്. വിവിധ ഉപയോഗകേസുകൾ ഉൾപ്പെടെ ഇൻഫെറൻസ്, ഫൈൻ-ട്യൂണിങ്, ക്വാണ്ടൈസേഷൻ, RAG നടപ്പാക്കലുകൾ, മൾട്ടിമോഡൽ അപേക്ഷകൾ വ്യത്യസ്ത പ്ലാറ്റ്ഫോമുകളിൽ, ഫ്രെയിംവർക്കുകളിൽ എന്നിവ ഈ സംഭരണിയിൽ കാണിക്കുന്നു.

**പ്രധാന സാങ്കേതിക വിദ്യകൾ:**  
- **ഭാഷകൾ:** Python, C#/.NET, JavaScript/Node.js  
- **ഫ്രെയിംവർക്ക്:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel  
- **പ്ലാറ്റ്ഫോമുകൾ:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama  
- **മോഡൽ തരങ്ങൾ:** Phi-3, Phi-3.5, Phi-4 (ടെക്സ്റ്റ്, വിഷൻ, മൾട്ടിമോഡൽ, റീസണിംഗ് വകഭേദങ്ങൾ)  

**സംഭരണി ഘടന:**  
- `/code/` - പ്രവർത്തിക്കുന്ന കോഡ് ഉദാഹരണങ്ങളും സാമ്പിൾ നടപ്പാക്കലുകളും  
- `/md/` - വിശദമായ ഡോക്യുമെന്റേഷൻ, പാഠങ്ങൾ, ഹൗ-ടു മാർഗ്ഗനിർദ്ദേശങ്ങൾ  
- `/translations/` - സ്വയംപ്രവർത്തിക്കുന്ന പ്രവാഹത്തിലൂടെ 50-ലധികം ഭാഷകളിലേക്ക് വിവർത്തനങ്ങൾ  
- `/.devcontainer/` - ഡെവ് കണ്ടെയ്നർ കോൺഫിഗറേഷൻ (Python 3.12 ഒപ്പം Ollama)

### Local Setup

**അവശ്യങ്ങൾ:**  
- Python 3.12 അല്ലെങ്കിൽ അതിനുമുൻപ്  
- .NET 8.0 SDK (C# ഉദാഹരണങ്ങൾക്ക്)  
- Node.js 18+ കൂടാതെ npm (JavaScript ഉദാഹരണങ്ങൾക്ക്)  
- കുറഞ്ഞത് 16GB RAM ശിപാർശ ചെയ്യുന്നു  

**ഇൻസ്റ്റാളേഷൻ:**  
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```
  
**Python ഉദാഹരണങ്ങൾക്കായി:**  
പ്രത്യേക ഉദാഹരണ ഡയറക്ടറികളിലേക്ക് കടന്ന് ഡിപ്പെൻഡൻസികൾ ഇൻസ്റ്റാൾ ചെയ്യുക:  
```bash
cd code/<example-directory>
pip install -r requirements.txt  # requirements.txt ഉണ്ടെങ്കിൽ
```
  
**.NET ഉദാഹരണങ്ങൾക്കായി:**  
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```
  
**JavaScript/Web ഉദാഹരണങ്ങൾക്കായി:**  
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # വികസന സെർവർ ആരംഭിക്കുക
npm run build  # പ്രൊഡക്ഷൻksa വേണ്ടി നിർമ്മിക്കുക
```

## Working with Examples

### Running Jupyter Notebooks

അധികഭാഗം ഉദാഹരണങ്ങളും Jupyter നോ‍‍‍‍‍‍‍റ്റ്ബുക്കുകളുടെ രൂപത്തിൽ ലഭ്യമാണ്:  
```bash
pip install jupyter notebook
jupyter notebook  # ബ്രൗസർ ഇന്റർഫേസ് തുറക്കുന്നു
# ആഗ്രഹിക്കുന്ന .ipynb ഫയലിലേക്ക് നവിഗേറ്റ് ചെയ്യുക
```
  
### Running Python Scripts

```bash
cd code/<example-directory>
pip install -r requirements.txt
python <script-name>.py
```
  
### Running .NET Examples

```bash
cd md/04.HOL/dotnet/src/<project-name>
dotnet run
```
  
അല്ലെങ്കിൽ മുഴുവൻ സൊല്യൂഷൻ നിർമ്മിക്കുക:  
```bash
cd md/04.HOL/dotnet/src
dotnet run --project <project-name>
```
  
### Running JavaScript/Web Examples

```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # ഹോട്ട് റീലോഡോടുകൂടിയ വികസനം
```

## Testing

ഈ സംഭരണിയിൽ പൊതുവായ സോഫ്റ്റ്‌വെയർ യൂണിറ്റ് ടെസ്റ്റുകൾ ഉള്ള പ്രോജക്റ്റല്ല, ഉദാഹരണ കോഡ്, പാഠങ്ങൾ ആണ്. സാധാരണ വാലിഡേഷൻ മുറിപ്പിക്കപ്പെടുന്നത്:  

1. **ഉദാഹരണങ്ങൾ പ്രവർത്തിപ്പിക്കൽ** - ഓരോ ഉദാഹരണവും എറർ കൂടാതെ പ്രവർത്തിക്കണം  
2. **ഫലങ്ങൾ പരിശോധിക്കൽ** - മോഡൽ പ്രതികരണങ്ങൾ യോജിച്ചതാണെന്ന് ഉറപ്പാക്കുക  
3. **പാഠങ്ങൾ അനുസരിക്കൽ** - മാർഗ്ഗനിർദ്ദേശം അനുസരിച്ച് പ്രവർത്തിക്കണം  

**സാമാന്യ വാലിഡേഷൻ സമീപനം:**  
- ലക്‌ഷ്യ പരിസ്ഥിതി പരീക്ഷണത്തിൽ ഉദാഹരണങ്ങൾ  
- ഡിപ്പെൻഡൻസികൾ ശരിയായി ഇൻസ്റ്റാൾ ആയിട്ടുണ്ടെന്ന് പരിശോധന  
- മോഡൽ ഡൗൺലോഡ്/ലോഡ് വിജയകരമാണെന്നും ശ്രാധിക്കുക  
- പ്രതീക്ഷിച്ച പെരുമാറ്റം ഡോക്യുമെന്റേഷനുമായി പൊരുത്തപ്പെടുകയും ചെയ്യണം  

## Code Style and Conventions

### General Guidelines

- ഉദാഹരണങ്ങൾ വ്യക്തവും, നല്ല കമന്റുകളുള്ളതും വിദ്യാഭാസപരവുമായിരിക്കണം  
- ഭാഷാ പ്രത്യേക മാനദണ്ഡങ്ങൾ പാലിക്കുക (Python-ക്ക് PEP 8, .NET ക്ക് C# സ്റ്റാൻഡേർഡുകൾ)  
- Phi മോഡൽ സവിശേഷതകൾ നിർദ്ദേശിക്കാൻ കേന്ദ്രീകരിക്കുക  
- മുഖ്യ ആശയങ്ങളും മോഡൽ പാരാമീറ്ററുകളും കമന്റുപയോഗിച്ച് വിശദീകരിക്കുക  

### Documentation Standards

**URL Formatting:**  
- `[text](../../url)` ഫോർമാറ്റിൽ ഹ്രസ്വമാക്കുക, അധിക ഇടവേളകൾ ഒഴിവാക്കുക  
- സാന്ദ്ര ലിങ്കുകൾ: നിലവിലെ ഡയറക്ടറിയ്ക്ക് `./`, മാതൃ ഡയറക്ടറിയ്ക്ക് `../` ഉപയോഗിക്കുക  
- URL-കളിൽ രാജ്യനിരീക്ഷണ ലൊക്കലുകളുടെ ഉപയോഗം ഒഴിവാക്കുക (`/en-us/`, `/en/` ഒഴിവാക്കുക)  

**Images:**  
- എല്ലാ ചിത്രങ്ങളും `/imgs/` ഡയറക്ടറിയിൽ സൂക്ഷിക്കുക  
- ഇംഗ്ലീഷ് വരികൾ, അക്ഷരങ്ങൾ, അക്കങ്ങൾ, ഡാഷുകൾ ഉപയോഗിച്ച് വിവരണാത്മകമായ പേരുകൾ നൽകുക  
- ഉദാഹരണം: `phi-3-architecture.png`  

**Markdown Files:**  
- യഥാർത്ഥ പ്രവർത്തിക്കുന്ന ഉദാഹരണങ്ങൾ `/code/` ഡയറക്ടറിയിൽ സൂചിപ്പിക്കുക  
- ഡോക്യുമെന്റേഷൻ കോഡ് മാറ്റങ്ങൾക്കൊപ്പം സിങ്ക്രനൈസ് ചെയ്തിരിക്കുക  
- README-യിൽ Jupyter നോ‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍റ്റ്ബുക്ക് ലിങ്കുകൾ 📓 ഇമോജിയോടെ അടയാളപ്പെടുത്തുക  

### File Organization

- കോഡ് ഉദാഹരണങ്ങൾ `/code/` ൽ വിഷയം/സവിശേഷത അനുസരിച്ച് ക്രമീകരിക്കുക  
- `/md/` ൽ ഡോക്യുമെന്റേഷൻ ആവശ്യമുള്ളപ്പോൾ കോഡ് ഘടനയോട് പൊരുത്തപ്പെടും  
- ബന്ധപ്പെട്ട ഫയലുകൾ (നോ‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍റ്റ്ബുക്കുകൾ, സ്ക്രിപ്റ്റുകൾ, കോൺഫിഗുകൾ) ഉപഡയറക്ടറികളിൽ ഒരുമിച്ച് സൂക്ഷിക്കുക  

## Pull Request Guidelines

### Before Submitting

1. **റിപ്പോസിറ്ററി ഫോർക്ക് ചെയ്യുക** നിങ്ങളുടെ അക്കൗണ്ടിലേക്ക്  
2. **PR-കൾ തരംതിരിച്ച് വേർതിരിക്കുക:**  
   - ഒരു PR-ൽ ബഗ് പരിഹാരങ്ങൾ  
   - മറ്റൊരു PR-യിൽ ഡോക്യുമെന്റേഷൻ അപ്ഡേറ്റുകൾ  
   - പുതിയ ഉദാഹരണങ്ങൾക്കായി വേർതിരിച്ച PR-കൾ  
   - ടൈപ്പോ ഫിക്സുകൾ പ്രത്യേകം ചേർക്കാവുന്നതാണ്  

3. **മേഴ്ജ് കോൺഫ്ലിക്റ്റുകൾ കൈകാര്യം ചെയ്യുക:**  
   - മാറ്റങ്ങൾ വരുത്തുന്നതിനു മുമ്പ് ലൊക്കൽ `main` ബ്രാഞ്ച് അപ്ഡേറ്റ് ചെയ്യുക  
   - ആൻ അപ്‌സ്റ്റ്രീം നിരന്തരം സിങ്ക് ചെയ്യുക  

4. **വിവർത്തന PR-കൾ:**  
   - ഫോൾഡറിലുള്ള എല്ലാ ഫയലുകൾക്കും വിവർത്തനങ്ങൾ ഉൾപ്പെടണം  
   - യഥാർത്ഥ ഭാഷയുമായി ഘടനയിൽ സ്ഥിരത പാലിക്കുക  

### Required Checks

GitHub വർക്ക്‌ഫ്ലോകൾ PR-കളെ ചെക്ക് ചെയ്യുന്നു:  

1. **ആന്തരിക ലോക്കൽ സങ്കേത പരിശോധന** - എല്ലായിടത്തും ലിങ്കുകൾ ശരിയായി പ്രവർത്തിക്കണം  
   - ലോക്കലായി കുഞ്ഞു വീൽ പരിശോധന: VS Code-യിൽ Ctrl+Click ചെയ്യുക  
   - VS Code-യിലെ വഴി ശിഫാരസുകൾ ഉപയോഗിക്കുക (`./` അല്ലെങ്കിൽ `../`)  

2. **URL ലൊക്കെൽ പരിശോധന** - വെബ് URL-കളിൽ രാജ്യ ലൊക്കലുകൾ ഇല്ലാതിരിക്കണം  
   - `/en-us/`, `/en/`, മറ്റ് ഭാഷാ കോഡുകൾ ഒഴിവാക്കുക  
   - പൊതുജന അന്താരാഷ്ട്ര URL-കൾ ഉപയോഗിക്കുക  

3. **URL ബ്രേക്കേജ് പരിശോധന** - എല്ലാ URL-കളും 200 സ്റ്റാറ്റസുമായി തിരിച്ചുവരണം  
   - സമർപ്പിക്കുന്നതിനുമുമ്പ് ലിങ്കുകൾ പരിശോധനക്ക് വിധേയമാക്കുക  
   - ചില പരാജയങ്ങൾ നെറ്റ്‌വർക്കിംഗ് നിയന്ത്രണങ്ങളാൽ ഉണ്ടായിരിക്കാം

## Common Development Patterns

### Working with Phi Models

**Model Loading:**  
- Transformers, ONNX Runtime, MLX, OpenVINO പോലുള്ള വിവിധ ഫ്രെയിംവർക്കുകൾ ഉപയോഗിച്ച് ഉദാഹരണങ്ങൾ  
- മോഡലുകൾ സാധാരണയായി Hugging Face, Azure, GitHub Models-ൽ നിന്നാണ് ഡൗൺലോഡ് ചെയ്യുന്നത്  
- നിങ്ങളുടെ ഹാർഡ്‌വെയർ (CPU, GPU, NPU) അനുയോജ്യമായ മോഡലുകൾ ഉപയോഗിക്കുക  

**Inference Patterns:**  
- ടെക്സ്റ്റ് ജനറേഷൻ: മിക്ക ഉദാഹരണങ്ങളിലും ചാറ്റ്/ഇൻസ്ട്രക്ട് വകഭേദങ്ങൾ ഉപയോഗിക്കുന്നു  
- വിഷൻ: Phi-3-vision, Phi-4-multimodal ചിത്ര മനസ്സിലാക്കലിനായി  
- ഓഡിയോ: Phi-4-multimodal ഓഡിയോ ഇൻപുട്ടുകൾക്ക് പിന്തുണ  
- റീസണിംഗ്: Phi-4-reasoning പരിചയപ്പെടുന്ന ഉയർന്ന തലത്തിൽ റീസണിംഗ് ടാസ്കുകൾ

## Troubleshooting

### Common Issues

**Memory Issues:**  
- Phi മോഡലുകൾക്ക് വലിയ മെമ്മറി ആവശ്യമാണ് (വിസൻ/മൾട്ടിമോഡൽ വകഭേദങ്ങൾ പ്രത്യേകിച്ച്)  
- സ്രോതസ്സു-പരിമിത പരിസരങ്ങളിൽ ക്വാണ്ടൈസ്ഡ് മോഡലുകൾ ഉപയോഗിക്കുക  
- `/md/01.Introduction/04/QuantifyingPhi.md` കാണുക  

**Dependency Conflicts:**  
- Python ഉദാഹരണങ്ങൾക്ക് പ്രത്യേക വേർഷൻ ആവശ്യകതകളുണ്ടാകാം  
- ഓരോ ഉദാഹരണത്തിനും വേർതിരിച്ച virtual environment ഉപയോഗിക്കുക  
- വ്യക്തിഗത `requirements.txt` ഫയലുകൾ പരിശോധിക്കുക  

**Model Download Failures:**  
- വലിയ മോഡലുകൾ സ്ലോ കണക്ഷനിൽ സമയപരിധി ലംഘിക്കാം  
- ക്ലൗഡ് പരിസ്ഥിതികൾ (Codespaces, Azure) പരിഗണിക്കുക  
- Hugging Face കാഷെ പരിശോധിക്കുക: `~/.cache/huggingface/`  

**.NET Project Issues:**  
- .NET 8.0 SDK ഇൻസ്റ്റാൾ ചെയ്തിരിക്കണം  
- ബിൽഡ് ചെയ്യുന്നതിനു മുമ്പ് `dotnet restore` നടത്തുക  
- ചില പ്രോജക്റ്റുകൾക്ക് CUDA പ്രത്യേക കോൺഫിഗറേഷൻ (Debug_Cuda) ഉണ്ട്  

**JavaScript/Web Examples:**  
- Node.js 18+ ഉപയോഗിക്കുക  
- പ്രശ്‌നങ്ങൾ തുടർന്നാൽ `node_modules` ക്ലീൻ ചെയ്ത് വീണ്ടും ഇൻസ്റ്റാൾ ചെയ്യുക  
- വെബ്GPU സാങ്കേതികശേഷി പ്രശ്‌നങ്ങൾക്കായി ബ്രൗസർ കോൺസോൾ പരിശോധിക്കുക
