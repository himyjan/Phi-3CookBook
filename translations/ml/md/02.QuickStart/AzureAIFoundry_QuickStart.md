# **Microsoft Foundry-യിൽ Phi-3 ഉപയോഗിക്കുന്നു**

ജനറേറ്റീവ് എഐയുടെ വികസനത്തോടെ, വിവിധ LLM, SLM, എൻറൈസൈസ് ഡാറ്റ ഇന്റഗ്രേഷൻ, ഫൈൻ-ട്യൂണിംഗ്/റാഗ് ഓപ്പറേഷനുകൾ, LLM, SLM എന്നിവ സംയോജിപ്പിച്ചതിന് ശേഷം വിവിധ എൻറൈസൈസ് ബിസിനസ്സുകളുടെ മൂല്യനിർണ്ണയം എന്നിവയെ നിയന്ത്രിക്കാൻ ഒന്നിച്ചുള്ള പ്ലാറ്റ്ഫോം ഉപയോഗിക്കാൻ ഞങ്ങൾ പ്രതീക്ഷിക്കുന്നു, അതിലൂടെ ജനറേറ്റീവ് എഐ ബുദ്ധിമുട്ടുള്ള ആപ്ലിക്കേഷനുകൾ മെച്ചപ്പെട്ട രീതിയിൽ നടപ്പിലാക്കാൻ കഴിയും. [Microsoft Foundry](https://ai.azure.com) ഒരു എൻറൈസൈസ് തലത്തിലുള്ള ജനറേറ്റീവ് എഐ ആപ്ലിക്കേഷൻ പ്ലാറ്റ്ഫോമാണ്.

![aistudo](../../../../translated_images/ml/aifoundry_home.f28a8127c96c7d93.webp)

Microsoft Foundry ഉപയോഗിച്ച്, നിങ്ങൾക്ക് വലുത് ഭാഷാ മോഡൽ (LLM) പ്രതികരണങ്ങൾ മൂല്യനിർണ്ണയം ചെയ്യാനും പ്രാമ്പ്റ്റ് ഫ്ലോ ഉപയോഗിച്ച് പ്രാമ്പ്റ്റ് ആപ്ലിക്കേഷൻ ഘടകങ്ങളെ ഓർക്കസ്ട്രേറ്റ് ചെയ്യാനും കഴിയും, ഈ വഴി പ്രകടനം മെച്ചപ്പെടുന്നു. ഈ പ്ലാറ്റ്ഫോം പ്രൂഫ് ഓഫ് കോൺസെപ്റ്റുകൾ പൂർണ്ണമായ ഉത്പാദനത്തിലേക്ക് എളുപ്പത്തിൽ മാറ്റാനുള്ള സ്കെയിലബിലിറ്റി നൽകുന്നു. തുടർച്ചയായ മേൽനോട്ടവും പരിഷ്കാരവും ദീർഘകാല വിജയത്തിന് സഹായകമാണ്.

നാം ലളിതമായ ചുവടുകളിലൂടെ Phi-3 മോഡൽ Microsoft Foundry-യിൽ വേഗത്തിൽ നിയോഗിക്കാവുന്നതാണ്, പിന്നീട് Microsoft Foundry ഉപയോഗിച്ച് Phi-3 സംബന്ധിച്ച Playground/ചാറ്റ്, ഫൈൻ-ട്യൂണിംഗ്, മൂല്യനിർണ്ണയം മുതലായ മറ്റ് പ്രവർത്തനങ്ങളും സാധ്യമാക്കാം.

## **1. തയ്യാറെടുപ്പ്**

നിങ്ങൾക്ക് ഇതിനകം [Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/overview?WT.mc_id=aiml-138114-kinfeylo) നിങ്ങളുടെ മെഷീനിൽ ഇൻസ്റ്റാൾ ചെയ്താൽ, ഈ ടെംപ്ലേറ്റ് ഉപയോഗിക്കുന്നത് പുതിയ ഡയറക്ടറിയിൽ താഴെ കൊടുത്തിരിക്കുന്ന കമാൻഡ് ജോലിയുടെ പോലെ എളുപ്പമാണ്.

## മാനുവൽ സൃഷ്ടി

Microsoft Foundry പ്രോജക്ടും ഹബ് നിർമ്മാണം നിങ്ങളുടെ AI ജോലി ക്രമീകരിക്കുന്നതിനും നിയന്ത്രിക്കുന്നതിനും മികച്ച മാർഗമാണ്. നിങ്ങൾ തുടക്കം കുറിക്കാൻ ഇതാ ക്രമമായ മാർഗ്ഗനിർദ്ദേശം:

### Microsoft Foundry-യിൽ പ്രോജക്ട് സൃഷ്ടിക്കൽ

1. **Microsoft Foundry-യില് പോകുക**: Microsoft Foundry പോർട്ടലിൽ സൈൻ ഇൻ ചെയ്യുക.
2. **ഒരു പ്രോജക്ട് നിർമ്മിക്കുക**:
   - നിങ്ങൾ ഒരു പ്രോജക്ടിൽ ആണെങ്കിൽ, പേജിന്റെ ഇടതുവശത്ത് മുകൾഭാഗത്ത് "Microsoft Foundry" തിരഞ്ഞെടുക്കുക ചുറ്റിപ്പാട് പേജിലേക്ക് പോകാൻ.
   - "+ Create project" തിരഞ്ഞെടുക്കുക.
   - പ്രോജക്ടിന് ഒരു പേര് നൽകുക.
   - നിങ്ങൾക്കൊരു ഹബ് ഉണ്ടെങ്കിൽ, അത് ഡിഫോൾട്ടായി തിരഞ്ഞെടുക്കും. ഒരു ഹബ്ബിന് അധികാരമുള്ളെങ്കിൽ, ഡ്രോപ്പ്ഡൗണിൽ നിന്നു മറ്റൊരു ഹബ് തിരഞ്ഞെടുക്കാം. പുതിയ ഹബ് സൃഷ്ടിക്കാനായി "Create new hub" തിരഞ്ഞെടുക്കുകയും പേര് നൽകുകയും ചെയ്യുക.
   - "Create" തിരഞ്ഞെടുക്കുക.

### Microsoft Foundry-യിൽ ഹബ് സൃഷ്ടിക്കൽ

1. **Microsoft Foundry-ല്‍ പോകുക**: നിങ്ങളുടെ Azure അക്കൗണ്ടിൽ സൈൻ ഇൻ ചെയ്യുക.
2. **ഒരു ഹബ് സൃഷ്ടിക്കുക**:
   - ഇടതുവശത്തെ മെനുവിൽ നിന്നും മാനേജ്‌മെന്റ് സെന്റർ തിരഞ്ഞെടുക്കുക.
   - "All resources" തിരഞ്ഞെടുക്കുക, തുടർന്ന് "+ New project" പഴുതു കാണുന്ന താഴെയുള്ള അറൊവിൽ അമർത്ത് "+ New hub" തിരഞ്ഞെടുക്കുക.
   - "Create a new hub" ഡയലോഗിൽ, ഹബ്ബിന് ഒരു പേര് നൽകുക (ഉദാഹരണം: contoso-hub) പിന്നെ മറ്റു ഫീൽഡുകൾ ഇച്ഛാനുസരണം മാറ്റാം.
   - "Next" തിരഞ്ഞെടുക്കുക, വിവരങ്ങൾ പരിശോധിച്ച് ശേഷം "Create" തിരഞ്ഞെടുക്കുക.

കൂടുതൽ വിശദമായ നിർദ്ദേശങ്ങൾക്ക് ഔദ്യോഗിക [Microsoft ഡോക്യുമെന്റേഷൻ](https://learn.microsoft.com/azure/ai-studio/how-to/create-projects) കാണാം.

വിജയകരമായി നിർമ്മിച്ചതിനു ശേഷം, നിങ്ങൾ സൃഷ്ടിച്ച സ്റ്റുഡിയോ [ai.azure.com](https://ai.azure.com/) വഴി ആക്‌സസ് ചെയ്യാം.

ഒരേ AI Foundry-യിൽ ഒരിലധികം പ്രോജക്ടുകൾ ഉണ്ടാകാം. AI Foundry-യിൽ പ്രോജക്ട് സൃഷ്ടിച്ച് തയ്യാറെടുക്കുക.

Microsoft Foundry [QuickStarts](https://learn.microsoft.com/azure/ai-studio/quickstarts/get-started-code) സൃഷ്ടിക്കുക.

## **4. Microsoft Foundry-യിൽ നിന്ന് മോഡൽ നിയോഗിക്കൽ**

Azure മോഡൽ കാറ്റലോഗിൽ നിന്ന് മോഡൽ നിയോഗിക്കാൻ, താഴെ പറയുന്ന പടികൾ പാലിക്കുക:

- Microsoft Foundry-യിൽ സൈൻ ഇൻ ചെയ്യുക.
- Microsoft Foundry മോഡൽ കാറ്റലോഗിൽ നിന്നു നിയോഗിക്കാനുള്ള മോഡൽ തിരഞ്ഞെടുക്കുക.
- മോഡലിന്റെ വിവരങ്ങൾ ഉള്ള പേജിൽ, Deploy തിരഞ്ഞെടുക്കുക, തുടർന്ന് Serverless API with Azure AI Content Safety തിരഞ്ഞെടുക്കുക.
- നിങ്ങളുടെ മോഡലുകൾ നിയോഗിക്കേണ്ട പ്രോജക്ട് തിരഞ്ഞെടുക്കുക. Serverless API ഉപയോഗിക്കാൻ, നിങ്ങളുടെ വർക്ക്‌സ്പേസ് East US 2 അല്ലെങ്കിൽ Sweden Central പ്രദേശങ്ങളിൽ പെട്ടിരിക്കണം. Deployment പേര് ഇഷ്ടാനുസരണം മാറ്റാവുന്നതാണ്.
- Deployment വിസാർഡിൽ, വിലയും വ്യവഹാര നിബന്ധനകളും പഠിക്കാൻ Pricing and terms തിരഞ്ഞെടുക്കുക.
- Deploy തിരഞ്ഞെടുക്കുക. നിയോഗം സജ്ജമായ ശേഷം, നിങ്ങൾ Deployments പേജിലേക്ക് റീഡയറക്ട് ചെയ്യും.
- Open in playground തിരഞ്ഞെടുക്കുക, മോഡലുമായി സംവദനം തുടങ്ങാൻ.
- നിങ്ങളക്ക് Deployments പേജിലേക്ക് മടങ്ങി, നിയോഗം തിരഞ്ഞെടുക്കുകയും, Endpoint-ന്റെ Target URL, Secret Key എന്നിവ ശ്രദ്ധിക്കുക; ഇവ ഉപയോഗിച്ച് നിയോഗം വിളിച്ച് ഫലം ഉണ്ടാക്കാം.
- Build ടാബിൽ Components വിഭാഗത്തിൽ നിന്ന് Deployments തിരഞ്ഞെടുക്കുന്നത് വഴി എപ്പോഴും Endpoint വിശദാംശങ്ങൾ, URL, ആക്സസ് കീകൾ കണ്ടെത്താവുന്നതാണ്.

> [!NOTE]
> ഈ നടപടികൾ ചെയ്യാൻ നിങ്ങളുടെ അക്കൗണ്ടിന് റിസോഴ്‌സ് ഗ്രൂപ്പിലെ Azure AI Developer റോളിന്റെ അനുമതികൾ ഉണ്ടായിരിക്കണം.
