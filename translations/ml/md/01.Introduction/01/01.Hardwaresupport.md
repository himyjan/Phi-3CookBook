# Phi Hardware Support

Microsoft Phi ONNX Runtime-നായി ഓപ്‌ടിമൈസ് ചെയ്തതും Windows DirectML-നെ പിന്തുണയ്ക്കുന്നതുമായതാണ്. GPU-കൾ, CPU-കൾ, മൊബൈൽ ഉപകരണങ്ങൾ ഉൾപ്പെടെയുള്ള വിവിധ ഹാർഡ്വെയർ തരംകളിൽ ഇത് നല്ല പ്രവർത്തനം കാണിക്കുന്നു.

## Device Hardware 
നിർദ്ദിഷ്ടമായി പിന്തുണയ്ക്കുന്ന ഹാർഡ്വെയർ താഴെപ്പറയുന്നത് ആണ്:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPUs, 128 GiB memory)

## Mobile SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 അല്ലെങ്കിൽ അതിനും മുകളിൽ A16/A17 Processor

## Phi Hardware Specification

- കുറഞ്ഞ കോൺഫിഗറേഷൻ ആവശ്യമാണ്.
- Windows: DirectX 12-സാധ്യമായ GPU-യും കുറഞ്ഞത് 4GB സംയുക്ത RAM-ഉം ആവശ്യമുണ്ട്

CUDA: Compute Capability >= 7.02 ഉള്ള NVIDIA GPU

## Multiple GPUs-ൽ onnxruntime പ്രവർത്തിപ്പിക്കൽ

ഇപ്പോൾ പ്രസിദ്ധീകരിച്ച Phi ONNX മോഡലുകൾ 1 GPU-ക്കു മാത്രമേ ഉള്ളുളളവയാകൂ. Phi മോഡലിനായി മൾട്ടി-GPU സപ്പോർട്ട് നൽകുന്നത് സാധ്യതയുള്ള കാര്യമാണ്, എന്നാൽ 2 GPU-കളുള്ള ORT 2 instance-കളായ ort-നേക്കാൾ കൂടുതൽ throughput നൽകുമെന്ന് ഉറപ്പില്ല. ഏറ്റവും പുതിയ അപ്‌ഡേറ്റുകൾക്കായി [ONNX Runtime](https://onnxruntime.ai/) കാണുക.

[Build 2024 the GenAI ONNX Team](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) multi-gpuയ്ക്കുപകരം multi-instance പ്രവർത്തിപ്പിക്കുവാൻ Phi മോഡലുകൾക്ക് അനുമതി നൽകിയതായി പ്രഖ്യാപിച്ചു.

ഇപ്പോൾ ഇതിലൂടെ CUDA_VISIBLE_DEVICES എൻവയോൺമെന്റ് വേരിയബിൾ ഉപയോഗിച്ച് നിങ്ങൾക്ക് ഒരു onnxruntime അല്ലെങ്കിൽ onnxruntime-genai instance പ്രവർത്തിപ്പിക്കാൻ കഴിയും.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

Phi-നെ കുറിച്ച് കൂടുതൽ അന്വേഷിക്കാൻ [Microsoft Foundry](https://ai.azure.com) സന്ദർശിക്കാം.
