### Guidance-AI 與 Phi 模型作為服務 (MaaS)
我們將 [Guidance](https://github.com/guidance-ai/guidance) 引入 Microsoft Foundry 中的 Phi-3.5-mini 無伺服器端點，透過定義適用於應用程式的結構，使輸出更具可預測性。使用 Guidance，您可以消除昂貴的重試，例如，可限制模型從預定義清單（如醫療代碼）中選擇，限制輸出只能是所提供上下文中的直接引述，或遵循任意正則表達式。Guidance 在推論堆疊中逐字控制模型，降低 30-50% 的成本和延遲，使其成為 [Phi-3-mini 無伺服器端點](https://aka.ms/try-phi3.5mini) 的獨特且有價值的附加功能。

## [**Guidance-AI**](https://github.com/guidance-ai/guidance) 是一個專為幫助開發者高效建立及部署 AI 模型而設計的框架。它著重於提供構建穩健 AI 應用程式的工具和最佳實踐。

結合 **Phi 模型作為服務 (MaaS)**，提供一個既具成本效益又高性能的小型語言模型 (SLM) 部署強力方案。

**Guidance-AI** 是一個程式設計框架，旨在幫助開發者更有效地控制和引導大型語言模型 (LLM)。它允許精確結構化輸出，與傳統提示或微調方法相比，減少延遲和成本。

### Guidance-AI 的主要功能：
- <strong>高效控制</strong>：幫助開發者控制語言模型生成文本的方式，確保輸出高品質且相關。
- <strong>成本與延遲降低</strong>：優化生成過程，使其更具成本效益且速度更快。
- <strong>靈活整合</strong>：支援多種後端，包括 Transformers、llama.cpp、AzureAI、VertexAI 及 OpenAI。
- <strong>豐富的輸出結構</strong>：支援條件語句、迴圈和工具使用等複雜輸出結構，便於生成清晰且易解析的結果。
- <strong>相容性</strong>：允許單一 Guidance 程式在多個後端執行，提高靈活性與易用性。

### 範例使用情境：
- <strong>受限生成</strong>：使用正則表達式與無上下文文法引導模型輸出。
- <strong>工具整合</strong>：自動交錯控制與生成，例如在文本生成任務中使用計算機。

更多詳細資訊與範例，請參考 [Guidance-AI GitHub 倉庫](https://github.com/guidance-ai/guidance)。

### Phi 模型的主要功能：
1. <strong>成本效益</strong>：設計為高性能且負擔得起。
2. <strong>低延遲</strong>：適用於需要快速回應的即時應用。
3. <strong>彈性</strong>：可部署於雲端、邊緣及離線等多種環境。
4. <strong>客製化</strong>：可使用領域特定資料微調模型以提升表現。
5. <strong>安全與合規</strong>：依照微軟 AI 原則建構，確保問責制、透明、公平、可靠、安全、隱私與包容性。

### Phi 模型作為服務 (MaaS)：
Phi 模型可透過推論 API 採用按使用量付費制輕鬆整合至應用程式，無需大量前期投入。
