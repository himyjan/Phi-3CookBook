# phi হার্ডওয়্যার সমর্থন

Microsoft Phi ONNX Runtime এর জন্য অপ্টিমাইজ করা হয়েছে এবং Windows DirectML সমর্থন করে। এটি বিভিন্ন হার্ডওয়্যার প্রকার, যেমন GPU, CPU, এবং এমনকি মোবাইল ডিভাইসগুলিতেও ভাল কাজ করে।

## ডিভাইস হার্ডওয়্যার
নির্দিষ্টভাবে, সমর্থিত হার্ডওয়্যারে অন্তর্ভুক্ত:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPU, 128 GiB মেমোরি)

## মোবাইল SKU

- অ্যান্ড্রয়েড - Samsung Galaxy S21
- Apple iPhone 14 বা তার উপরে A16/A17 প্রসেসর

## Phi হার্ডওয়্যার স্পেসিফিকেশন

- ন্যূনতম কনফিগারেশন প্রয়োজন।
- উইন্ডোজ: DirectX 12 সক্ষম GPU এবং কমপক্ষে 4GB যৌথ RAM

CUDA: Compute Capability >= 7.02 সহ NVIDIA GPU

![HardwareSupport](../../../../../translated_images/bn/01.phihardware.5d51b2377cba18af.webp)

## একাধিক GPU তে onnxruntime চালানো

বর্তমানে উপলব্ধ Phi ONNX মডেলগুলি শুধুমাত্র ১টি GPU এর জন্য। Phi মডেলের জন্য মাল্টি-GPU সমর্থন সম্ভব, তবে ২টি GPU সহ ORT নিশ্চয়তা দেয় না যে এটি ২টি ort ইনস্ট্যান্সের তুলনায় বেশি থ্রুপুট দেবে। সর্বশেষ আপডেটের জন্য [ONNX Runtime](https://onnxruntime.ai/) দেখুন।

[Build 2024 এর GenAI ONNX টিম](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) ঘোষণা করেছে যে তারা Phi মডেলগুলির জন্য মাল্টি-GPU এর পরিবর্তে মাল্টি-ইনস্ট্যান্স সক্রিয় করেছে।

বর্তমানে এটি আপনাকে CUDA_VISIBLE_DEVICES পরিবেশ ভেরিয়েবল ব্যবহার করে একটি onnxruntime বা onnxruntime-genai ইনস্ট্যান্স চালাতে দেয়।

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

[Microsoft Foundry](https://ai.azure.com) এ Phi আরও অন্বেষণ করার জন্য স্বাচ্ছন্দ্য বোধ করুন।
