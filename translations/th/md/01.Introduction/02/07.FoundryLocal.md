## การเริ่มต้นใช้งานโมเดล Phi-Family ใน Foundry Local

### บทนำสู่ Foundry Local

Foundry Local คือโซลูชัน AI inference บนเครื่องที่ทรงพลังซึ่งนำความสามารถ AI ระดับองค์กรมาสู่ฮาร์ดแวร์ท้องถิ่นของคุณโดยตรง บทแนะนำนี้จะแนะนำวิธีการตั้งค่าและใช้งานโมเดล Phi-Family กับ Foundry Local เพื่อให้คุณควบคุมงาน AI ได้อย่างเต็มที่พร้อมรักษาความเป็นส่วนตัวและลดค่าใช้จ่าย

Foundry Local มอบประสิทธิภาพ ความเป็นส่วนตัว การปรับแต่ง และความคุ้มค่าโดยการรันโมเดล AI บนอุปกรณ์ของคุณโดยตรง มันผสานการทำงานได้อย่างราบรื่นกับเวิร์กโฟลว์และแอปพลิเคชันที่คุณมีอยู่ผ่าน CLI, SDK และ REST API ที่ใช้งานง่าย


![arch](../../../../../translated_images/th/foundry-local-arch.8823e321dd8258d7.webp)

### ทำไมต้องเลือก Foundry Local?

การเข้าใจประโยชน์ของ Foundry Local จะช่วยให้คุณตัดสินใจวางกลยุทธ์การใช้งาน AI ได้อย่างเหมาะสม:

- **Inference บนอุปกรณ์:** รันโมเดลบนฮาร์ดแวร์ของคุณเอง ลดค่าใช้จ่ายพร้อมเก็บข้อมูลทั้งหมดไว้ที่อุปกรณ์ของคุณ

- **การปรับแต่งโมเดล:** เลือกจากโมเดลที่ตั้งไว้ล่วงหน้าหรือใช้โมเดลของคุณเองเพื่อตอบโจทย์และกรณีการใช้งานเฉพาะ

- **ประหยัดค่าใช้จ่าย:** ขจัดค่าใช้บริการคลาวด์ที่เกิดขึ้นซ้ำๆ ด้วยการใช้ฮาร์ดแวร์ที่คุณมีอยู่ ทำให้ AI เข้าถึงได้ง่ายขึ้น

- **การผนวกรวมอย่างไร้รอยต่อ:** เชื่อมต่อกับแอปพลิเคชันผ่าน SDK, API endpoints หรือ CLI พร้อมปรับขนาดขึ้น Microsoft Foundry ตามที่คุณต้องการ

> **หมายเหตุการเริ่มต้น:** บทแนะนำนี้เน้นการใช้ Foundry Local ผ่านทางอินเทอร์เฟซ CLI และ SDK คุณจะได้เรียนรู้ทั้งสองวิธีเพื่อช่วยเลือกวิธีที่เหมาะสมที่สุดสำหรับการใช้งานของคุณ

## ส่วนที่ 1: การตั้งค่า Foundry Local CLI

### ขั้นตอนที่ 1: การติดตั้ง

Foundry Local CLI คือประตูสู่การจัดการและรันโมเดล AI บนอุปกรณ์ของคุณ เริ่มต้นด้วยการติดตั้งบนระบบของคุณ

**แพลตฟอร์มที่รองรับ:** Windows และ macOS

สำหรับคำแนะนำการติดตั้งอย่างละเอียด กรุณาดูที่ [เอกสาร Foundry Local อย่างเป็นทางการ](https://github.com/microsoft/Foundry-Local/blob/main/README.md)

### ขั้นตอนที่ 2: สำรวจโมเดลที่มีให้ใช้งาน

เมื่อคุณติดตั้ง Foundry Local CLI แล้ว คุณสามารถค้นหาว่าโมเดลใดที่รองรับสำหรับการใช้งานของคุณ คำสั่งนี้จะแสดงโมเดลทั้งหมดที่รองรับ:


```bash
foundry model list
```

### ขั้นตอนที่ 3: ทำความเข้าใจโมเดล Phi Family

Phi Family มีโมเดลหลากหลายที่ปรับแต่งมาเพื่อการใช้งานและฮาร์ดแวร์ที่แตกต่างกัน นี่คือโมเดล Phi ที่มีใน Foundry Local:

**โมเดล Phi ที่มีให้ใช้งาน:** 

- **phi-3.5-mini** - โมเดลขนาดกะทัดรัดสำหรับงานพื้นฐาน
- **phi-3-mini-128k** - รุ่นบริบทขยายสำหรับบทสนทนายาวขึ้น
- **phi-3-mini-4k** - โมเดลบริบทมาตรฐานสำหรับใช้งานทั่วไป
- **phi-4** - โมเดลขั้นสูงที่มีความสามารถดีขึ้น
- **phi-4-mini** - เวอร์ชันน้ำหนักเบาของ Phi-4
- **phi-4-mini-reasoning** - เชี่ยวชาญสำหรับงานเหตุผลซับซ้อน

> **ความเข้ากันได้ของฮาร์ดแวร์:** แต่ละโมเดลสามารถกำหนดค่าเพื่อฮาร์ดแวร์แตกต่างกัน (CPU, GPU) ตามความสามารถของระบบคุณ

### ขั้นตอนที่ 4: รันโมเดล Phi ตัวแรกของคุณ

เริ่มต้นด้วยตัวอย่างจริง เราจะรันโมเดล `phi-4-mini-reasoning` ซึ่งเก่งในการแก้ปัญหาซับซ้อนอย่างเป็นขั้นตอน


**คำสั่งรันโมเดล:**

```bash
foundry model run Phi-4-mini-reasoning-generic-cpu
```

> **การตั้งค่าแรกใช้งาน:** เมื่อรันโมเดลครั้งแรก Foundry Local จะดาวน์โหลดโมเดลมายังอุปกรณ์ของคุณโดยอัตโนมัติ ระยะเวลาการดาวน์โหลดขึ้นอยู่กับความเร็วเน็ตเวิร์กของคุณ กรุณารอสักครู่ในระหว่างการตั้งค่าเริ่มต้น

### ขั้นตอนที่ 5: ทดสอบโมเดลด้วยปัญหาจริง

ตอนนี้มาทดสอบโมเดลกับปัญหาตรรกะคลาสสิกเพื่อดูว่ามันทำเหตุผลเป็นขั้นตอนได้อย่างไร:

**ตัวอย่างปัญหา:**

```txt
Please calculate the following step by step: Now there are pheasants and rabbits in the same cage, there are thirty-five heads on top and ninety-four legs on the bottom, how many pheasants and rabbits are there?
```

**พฤติกรรมที่คาดหวัง:** โมเดลควรแบ่งปัญหานี้เป็นขั้นตอนทางตรรกะ โดยใช้ข้อเท็จจริงว่าห่านมีขา 2 ขาและกระต่ายมีขา 4 ขาเพื่อแก้ระบบสมการ

## ส่วนที่ 2: พัฒนาแอปพลิเคชันด้วย Foundry Local SDK

### ทำไมต้องใช้ SDK?

ในขณะที่ CLI เหมาะกับการทดสอบและการโต้ตอบอย่างรวดเร็ว SDK ช่วยให้คุณผนวกรวม Foundry Local เข้ากับแอปพลิเคชันของคุณในรูปแบบโปรแกรมได้ ทำให้เปิดโอกาสสำหรับ:

- สร้างแอปพลิเคชัน AI แบบกำหนดเอง
- สร้างเวิร์กโฟลว์อัตโนมัติ
- ผนวกรวมความสามารถ AI เข้ากับระบบที่มีอยู่
- พัฒนาช่องแชทบอทและเครื่องมือโต้ตอบ

### ภาษาการเขียนโปรแกรมที่รองรับ

Foundry Local มี SDK รองรับหลายภาษาเพื่อให้เหมาะกับความชอบในการพัฒนาของคุณ:

**📦 SDK ที่มีให้:**

- **C# (.NET):** [เอกสารและตัวอย่าง SDK](https://github.com/microsoft/Foundry-Local/tree/main/sdk/cs)
- **Python:** [เอกสารและตัวอย่าง SDK](https://github.com/microsoft/Foundry-Local/tree/main/sdk/python)
- **JavaScript:** [เอกสารและตัวอย่าง SDK](https://github.com/microsoft/Foundry-Local/tree/main/sdk/js)
- **Rust:** [เอกสารและตัวอย่าง SDK](https://github.com/microsoft/Foundry-Local/tree/main/sdk/rust)

### ขั้นตอนต่อไป

1. **เลือก SDK ที่คุณชอบ** ตามสภาพแวดล้อมการพัฒนาของคุณ
2. **อ่านเอกสารเฉพาะ SDK** สำหรับคำแนะนำการใช้งานอย่างละเอียด
3. **เริ่มด้วยตัวอย่างง่ายๆ** ก่อนสร้างแอปพลิเคชันที่ซับซ้อน
4. **สำรวจโค้ดตัวอย่าง** ที่มีให้ในแต่ละ repository ของ SDK
