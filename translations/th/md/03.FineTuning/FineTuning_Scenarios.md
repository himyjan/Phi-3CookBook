## กรณีการปรับแต่งแบบละเอียด

![FineTuning with MS Services](../../../../translated_images/th/FinetuningwithMS.3d0cec8ae693e094.webp)

ส่วนนี้ให้ภาพรวมของกรณีการปรับแต่งแบบละเอียดในสภาพแวดล้อม Microsoft Foundry และ Azure รวมถึงโมเดลการปรับใช้ ชั้นโครงสร้างพื้นฐาน และเทคนิคการปรับแต่งที่ใช้กันทั่วไป

**แพลตฟอร์ม**  
รวมถึงบริการที่มีการจัดการ เช่น Microsoft Foundry (เดิมชื่อ Microsoft Foundry) และ Azure Machine Learning ซึ่งให้บริการการจัดการโมเดล การประสานงาน การติดตามการทดลอง และกระบวนการปรับใช้

**โครงสร้างพื้นฐาน**  
การปรับแต่งแบบละเอียดต้องการทรัพยากรคอมพิวเตอร์ที่ปรับขนาดได้ ในสภาพแวดล้อม Azure โดยปกติจะรวมถึงเครื่องเสมือนที่ใช้ GPU และทรัพยากร CPU สำหรับงานที่มีน้ำหนักเบา รวมถึงการจัดเก็บที่ปรับขนาดได้สำหรับชุดข้อมูลและจุดตรวจสอบ

**เครื่องมือ & เฟรมเวิร์ก**  
เวิร์กโฟลว์การปรับแต่งแบบละเอียดโดยทั่วไปพึ่งพาเฟรมเวิร์กและไลบรารีการปรับแต่ง เช่น Hugging Face Transformers, DeepSpeed และ PEFT (Parameter-Efficient Fine-Tuning)

กระบวนการปรับแต่งแบบละเอียดด้วยเทคโนโลยีของ Microsoft ครอบคลุมบริการแพลตฟอร์ม โครงสร้างพื้นฐานคอมพิวเตอร์ และเฟรมเวิร์กการฝึก ด้วยความเข้าใจว่าชิ้นส่วนเหล่านี้ทำงานร่วมกันอย่างไร นักพัฒนาสามารถปรับแบบจำลองพื้นฐานให้เข้ากับงานเฉพาะและกรณีผลิตได้อย่างมีประสิทธิภาพ

## โมเดลในรูปแบบบริการ

ปรับแต่งโมเดลโดยใช้การปรับแต่งแบบโฮสต์ โดยไม่ต้องสร้างและจัดการคอมพิวต์

![MaaS Fine Tuning](../../../../translated_images/th/MaaSfinetune.3eee4630607aff0d.webp)

การปรับแต่งแบบ serverless มีให้ใช้งานสำหรับตระกูลโมเดล Phi-3, Phi-3.5 และ Phi-4 ทำให้นักพัฒนาสามารถปรับแต่งโมเดลสำหรับกรณีใช้งานบนคลาวด์และ edge ได้อย่างรวดเร็วและง่ายดายโดยไม่ต้องจัดเตรียมคอมพิวต์
