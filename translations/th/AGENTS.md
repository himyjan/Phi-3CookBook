# AGENTS.md

## ภาพรวมโครงการ

PhiCookBook เป็นที่เก็บสูตรอาหารครอบคลุมที่รวมตัวอย่างใช้งานจริง, บทเรียน, และเอกสารสำหรับการทำงานกับตระกูล Phi ของโมเดลภาษาขนาดเล็ก (SLMs) ของ Microsoft ที่เก็บนี้แสดงตัวอย่างการใช้งานหลากหลาย รวมถึงการสรุปผล, การปรับแต่ง (fine-tuning), การลดความละเอียด (quantization), การใช้งาน RAG, และแอปพลิเคชันแบบมัลติ-โหมดผ่านแพลตฟอร์มและเฟรมเวิร์กต่าง ๆ

**เทคโนโลยีสำคัญ:**
- **ภาษา:** Python, C#/.NET, JavaScript/Node.js
- **เฟรมเวิร์ก:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **แพลตฟอร์ม:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **ประเภทโมเดล:** Phi-3, Phi-3.5, Phi-4 (รูปแบบข้อความ, วิสัยทัศน์, มัลติ-โหมด, การใช้เหตุผล)

**โครงสร้างที่เก็บ:**
- `/code/` - ตัวอย่างโค้ดทำงานและการใช้งานตัวอย่าง
- `/md/` - เอกสารรายละเอียด, บทเรียน, และคู่มือวิธีใช้งาน  
- `/translations/` - การแปลหลายภาษา (50+ ภาษา ผ่านระบบอัตโนมัติ)
- `/.devcontainer/` - การตั้งค่านำเข้า dev container (Python 3.12 กับ Ollama)

## การตั้งค่าสภาพแวดล้อมการพัฒนา

### การใช้งาน GitHub Codespaces หรือ Dev Containers (แนะนำ)

1. เปิดใช้ใน GitHub Codespaces (เร็วที่สุด):
   - คลิกที่ป้าย "Open in GitHub Codespaces" ใน README
   - คอนเทนเนอร์ตั้งค่าอัตโนมัติด้วย Python 3.12 และ Ollama ที่มี Phi-3

2. เปิดใน VS Code Dev Containers:
   - ใช้ป้าย "Open in Dev Containers" จาก README
   - คอนเทนเนอร์ต้องการหน่วยความจำโฮสต์ขั้นต่ำ 16GB

### การตั้งค่าในเครื่อง

**ข้อกำหนดเบื้องต้น:**
- Python 3.12 หรือสูงกว่า
- .NET 8.0 SDK (สำหรับตัวอย่าง C#)
- Node.js 18+ และ npm (สำหรับตัวอย่าง JavaScript)
- แนะนำหน่วยความจำอย่างน้อย 16GB

**การติดตั้ง:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**สำหรับตัวอย่าง Python:**
ไปที่ไดเรกทอรีของตัวอย่างที่เฉพาะเจาะจงและติดตั้ง dependencies:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # ถ้าไฟล์ requirements.txt มีอยู่
```

**สำหรับตัวอย่าง .NET:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**สำหรับตัวอย่าง JavaScript/Web:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # เริ่มเซิร์ฟเวอร์สำหรับการพัฒนา
npm run build  # สร้างสำหรับการผลิต
```

## แนวทางการส่ง Pull Request

### ก่อนส่ง

1. **Fork ที่เก็บนี้** ไปยังบัญชีของคุณ
2. **แยก PR ตามประเภท:**
   - แก้บั๊กใน PR เดียว
   - อัปเดตเอกสารในอีก PR
   - ตัวอย่างใหม่แยก PR
   - แก้ไขพิมพ์ผิดรวมกันได้

3. **จัดการความขัดแย้งของ merge:**
   - อัพเดตสาขา `main` ภายในเครื่องก่อนเริ่มเปลี่ยนแปลง
   - ซิงค์กับ upstream บ่อย ๆ

4. **สำหรับ PR การแปล:**
   - ต้องรวมการแปลของไฟล์ทั้งหมดในโฟลเดอร์
   - รักษาโครงสร้างสอดคล้องกับภาษาต้นฉบับ

### การตรวจสอบที่จำเป็น

PR จะรัน workflow ของ GitHub เพื่อยืนยัน:

1. **ตรวจสอบเส้นทางสัมพัทธ์** - ลิงก์ภายในต้องทำงานได้ทั้งหมด
   - ทดสอบลิงก์ภายในเครื่องโดย Ctrl+คลิกใน VS Code
   - ใช้คำแนะนำเส้นทางจาก VS Code (`./` หรือ `../`)

2. **ตรวจสอบโลเคิลใน URL** - URL เว็บไซต์ต้องไม่มีโลเคิลประเทศ
   - ลบ `/en-us/`, `/en/` หรือรหัสภาษาอื่น
   - ใช้ URL สากลทั่วไป

3. **ตรวจสอบ URL ไม่เสียหาย** - URL ทุกอันต้องคืนสถานะ 200
   - ตรวจสอบการเข้าถึงก่อนส่ง PR
   - หมายเหตุ: บางลิงก์อาจล้มเหลวเพราะข้อจำกัดเครือข่าย


- `[translation] Add Japanese translation for intro guides`

## รูปแบบการพัฒนาทั่วไป

### การใช้งานโมเดล Phi

**การโหลดโมเดล:**
- ตัวอย่างใช้เฟรมเวิร์กต่าง ๆ: Transformers, ONNX Runtime, MLX, OpenVINO
- โมเดลมักดาวน์โหลดจาก Hugging Face, Azure, หรือ GitHub Models
- ตรวจสอบความเข้ากันได้ของโมเดลกับฮาร์ดแวร์ของคุณ (CPU, GPU, NPU)

**รูปแบบการสรุปผล:**
- การสร้างข้อความ: ตัวอย่างส่วนใหญ่ใช้รูปแบบ chat/instruct

- เสียง: Phi-4-multimodal รองรับอินพุตเสียง
- การใช้เหตุผล: รุ่น Phi-4-reasoning สำหรับงานใช้เหตุผลขั้นสูง

- ต้องมีการสมัครใช้ Azure และ API keys
- ดู `/md/02.QuickStart/AzureAIFoundry_QuickStart.md`

**GitHub Models:**
- มีระดับฟรีสำหรับทดสอบ
- ดู `/md/02.QuickStart/GitHubModel_QuickStart.md`

**การสรุปผลในเครื่อง:**
- ONNX Runtime: รองรับหลายแพลตฟอร์ม, สรุปผลแบบเพิ่มประสิทธิภาพ

## การแก้ปัญหา

### ปัญหาทั่วไป

**ปัญหาหน่วยความจำ:**
- โมเดล Phi ต้อง RAM มาก (โดยเฉพาะรุ่นวิสัยทัศน์/มัลติ-โหมด)
- ใช้โมเดลที่ผ่านการลดความละเอียดกับสภาพแวดล้อมที่ทรัพยากรจำกัด

**ความขัดแย้งของไลบรารี:**
- ตัวอย่าง Python อาจมีข้อกำหนดเวอร์ชันเฉพาะ
- ใช้ virtual environment แยกสำหรับแต่ละตัวอย่าง
- ตรวจสอบไฟล์ `requirements.txt` แต่ละตัวอย่าง

**ปัญหาดาวน์โหลดโมเดล:**
- โมเดลขนาดใหญ่โหลดช้าอาจหมดเวลา
- ใช้สภาพแวดล้อมคลาวด์ เช่น Codespaces, Azure
- ตรวจสอบแคช Hugging Face: `~/.cache/huggingface/`

**ปัญหาโปรเจค .NET:**
- ติดตั้ง .NET 8.0 SDK ให้เรียบร้อย

- ลบ `node_modules` และติดตั้งใหม่ถ้ามีปัญหา
