# **Microsoft Foundry में Phi-3 का उपयोग**

Generative AI के विकास के साथ, हम एक एकीकृत प्लेटफॉर्म का उपयोग करने की उम्मीद करते हैं ताकि विभिन्न LLM और SLM, एंटरप्राइज़ डेटा एकीकरण, फाइन-ट्यूनिंग/RAG ऑपरेशन, और LLM और SLM को एकीकृत करने के बाद विभिन्न एंटरप्राइज़ व्यवसायों के मूल्यांकन का प्रबंधन किया जा सके, जिससे कि जनरेटिव AI स्मार्ट एप्लिकेशन बेहतर तरीके से लागू किया जा सके। [Microsoft Foundry](https://ai.azure.com) एक एंटरप्राइज़-स्तरीय जनरेटिव AI एप्लिकेशन प्लेटफॉर्म है।

![aistudo](../../../../translated_images/hi/aifoundry_home.f28a8127c96c7d93.webp)

Microsoft Foundry के साथ, आप बड़े भाषा मॉडल (LLM) प्रतिक्रियाओं का मूल्यांकन कर सकते हैं और बेहतर प्रदर्शन के लिए प्रॉम्प्ट फ्लो के साथ प्रॉम्प्ट एप्लिकेशन घटकों का आयोजन कर सकते हैं। प्लेटफॉर्म प्रमाण-पत्र अवधारणाओं को पूर्ण-स्तर के उत्पादन में परिवर्तित करने के लिए विस्तार क्षमता को सक्षम करता है। निरंतर निगरानी और सुधार दीर्घकालिक सफलता का समर्थन करते हैं।

हम सरल चरणों के माध्यम से Microsoft Foundry पर Phi-3 मॉडल को तेजी से परिनियोजित कर सकते हैं, और फिर Microsoft Foundry का उपयोग करके Phi-3 से संबंधित Playground/Chat, फाइन-ट्यूनिंग, मूल्यांकन और अन्य संबंधित कार्य पूरे कर सकते हैं।

## **1. तैयारी**

यदि आपके पास पहले से [Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/overview?WT.mc_id=aiml-138114-kinfeylo) आपके मशीन पर इंस्टॉल है, तो इस टेम्पलेट का उपयोग करना एक नए डिरेक्टरी में इस कमांड को चलाने जितना सरल है।

## मैनुअल निर्माण

Microsoft Foundry प्रोजेक्ट और हब बनाना आपके AI कार्य को संगठित करने और प्रबंधित करने का एक बेहतरीन तरीका है। यहां एक चरण-दर-चरण मार्गदर्शिका दी गई है जो आपको शुरुआत करने में मदद करेगी:

### Microsoft Foundry में प्रोजेक्ट बनाना

1. **Microsoft Foundry पर जाएं**: Microsoft Foundry पोर्टल में साइन इन करें।
2. **प्रोजेक्ट बनाएं**:
   - यदि आप किसी प्रोजेक्ट में हैं, तो होम पेज पर जाने के लिए पृष्ठ के ऊपर बाएं "Microsoft Foundry" चुनें।
   - "+ Create project" चुनें।
   - प्रोजेक्ट के लिए एक नाम दर्ज करें।
   - यदि आपके पास कोई हब है, तो वह डिफ़ॉल्ट रूप से चुना जाएगा। यदि आपके पास एक से अधिक हब तक पहुँच है, तो आप ड्रॉपडाउन से कोई अन्य चुन सकते हैं। यदि आप नया हब बनाना चाहते हैं, तो "Create new hub" चुनें और नाम प्रदान करें।
   - "Create" चुनें।

### Microsoft Foundry में हब बनाना

1. **Microsoft Foundry पर जाएं**: अपने Azure खाते से साइन इन करें।
2. **हब बनाएं**:
   - बाएँ मेनू से Management center चुनें।
   - "All resources" चुनें, फिर "+ New project" के नीचे डाउन एरो पर क्लिक करें और "+ New hub" चुनें।
   - "Create a new hub" संवाद में, अपने हब के लिए एक नाम दर्ज करें (उदा., contoso-hub) और अन्य फ़ील्ड आवश्यकतानुसार संशोधित करें।
   - "Next" चुनें, जानकारी की समीक्षा करें, और फिर "Create" चुनें।

अधिक विस्तृत निर्देशों के लिए, आप आधिकारिक [Microsoft दस्तावेज़](https://learn.microsoft.com/azure/ai-studio/how-to/create-projects) देख सकते हैं।

सफल निर्माण के बाद, आप अपने बनाए स्टूडियो को [ai.azure.com](https://ai.azure.com/) के माध्यम से एक्सेस कर सकते हैं।

एक AI Foundry पर एक से अधिक प्रोजेक्ट हो सकते हैं। तैयार करने के लिए AI Foundry में प्रोजेक्ट बनाएं।

Microsoft Foundry [QuickStarts](https://learn.microsoft.com/azure/ai-studio/quickstarts/get-started-code) बनाएं।

## **4. Microsoft Foundry से मॉडल परिनियोजित करना**

Azure Model Catalog से मॉडल को परिनियोजित करने के लिए, आप निम्न चरणों का पालन कर सकते हैं:

- Microsoft Foundry में साइन इन करें।
- Microsoft Foundry मॉडल कैटलॉग से उस मॉडल को चुनें जिसे आप परिनियोजित करना चाहते हैं।
- मॉडल के विवरण पृष्ठ पर, Deploy चुनें और फिर Azure AI Content Safety के साथ Serverless API चुनें।
- उस प्रोजेक्ट का चयन करें जिसमें आप अपने मॉडल परिनियोजित करना चाहते हैं। Serverless API का उपयोग करने के लिए, आपका कार्यक्षेत्र East US 2 या Sweden Central क्षेत्र में होना चाहिए। आप Deployment नाम को अनुकूलित कर सकते हैं।
- परिनियोजन विज़ार्ड पर, मूल्य निर्धारण और उपयोग की शर्तों को जानने के लिए Pricing and terms चुनें।
- Deploy चुनें। परिनियोजन तैयार होने तक प्रतीक्षा करें और जब आप Deployments पृष्ठ पर पुनः निर्देशित हों।
- मॉडल के साथ इंटरैक्ट करने के लिए Open in playground चुनें।
- आप वापस Deployments पृष्ठ पर जा सकते हैं, परिनियोजन का चयन कर सकते हैं, और endpoint के Target URL और Secret Key नोट कर सकते हैं, जिनका उपयोग आप परिनियोजन को कॉल करने और पूर्णताएँ उत्पन्न करने के लिए कर सकते हैं।
- आप हमेशा Build टैब में जाकर और Components सेक्शन से Deployments चुनकर endpoint के विवरण, URL और एक्सेस कुंजी पा सकते हैं।

> [!NOTE]
> कृपया ध्यान दें कि इन चरणों को करने के लिए आपके खाते के पास Resource Group पर Azure AI Developer रोल अनुमतियाँ होनी चाहिए।

## **5. Microsoft Foundry में Phi API का उपयोग**

आप https://{Your project name}.region.inference.ml.azure.com/swagger.json को Postman GET के माध्यम से एक्सेस कर सकते हैं और Key के साथ इसे संयोजित कर प्रदान किए गए इंटरफेस के बारे में जान सकते हैं

आप अनुरोध पैरामीटर बहुत आसानी से प्राप्त कर सकते हैं, साथ ही प्रतिक्रिया पैरामीटर भी।
