# Phi Hardware Support

Microsoft Phi ay na-optimize para sa ONNX Runtime at sumusuporta sa Windows DirectML. Ito ay mahusay na gumagana sa iba't ibang uri ng hardware, kabilang ang GPUs, CPUs, at pati na rin ang mga mobile na device. 

## Device Hardware 
Partikular, ang sinusuportahang hardware ay kinabibilangan ng:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPUs, 128 GiB memory)

## Mobile SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 o mas mataas pa na may A16/A17 Processor

## Phi Hardware Specification

- Minimum na Kinakailangang Konpigirasyon.
- Windows: DirectX 12-capable na GPU at minimum na 4GB na pinagsamang RAM

CUDA: NVIDIA GPU na may Compute Capability >= 7.02

## Pagpapatakbo ng onnxruntime sa maramihang GPUs

Sa kasalukuyan, ang mga magagamit na Phi ONNX models ay para lamang sa 1 GPU. Posibleng suportahan ang multi-gpu para sa Phi model, ngunit ang ORT na may 2 gpu ay hindi ginagarantiyahan na magbibigay ito ng mas mataas na throughput kumpara sa 2 instance ng ort. Mangyaring tingnan ang [ONNX Runtime](https://onnxruntime.ai/) para sa pinakabagong mga update.

Sa [Build 2024 ang GenAI ONNX Team](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) ay nag-anunsyo na pinagana na nila ang multi-instance imbes na multi-gpu para sa mga Phi models.

Sa kasalukuyan, ito ay nagbibigay-daan upang patakbuhin ang isang onnxruntime o onnxruntime-genai instance gamit ang CUDA_VISIBLE_DEVICES environment variable nang ganito.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

Malaya kang tuklasin pa ang Phi sa [Microsoft Foundry](https://ai.azure.com)
