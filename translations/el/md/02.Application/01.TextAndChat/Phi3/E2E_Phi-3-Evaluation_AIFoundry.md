#### Αρχές Υπεύθυνης Τεχνητής Νοημοσύνης της Microsoft

Πριν ξεκινήσετε τα τεχνικά βήματα, είναι απαραίτητο να κατανοήσετε τις Αρχές Υπεύθυνης Τεχνητής Νοημοσύνης της Microsoft, ένα ηθικό πλαίσιο σχεδιασμένο να καθοδηγεί την υπεύθυνη ανάπτυξη, διάθεση και λειτουργία συστημάτων AI. Αυτές οι αρχές κατευθύνουν τον υπεύθυνο σχεδιασμό, ανάπτυξη και διάθεση των συστημάτων AI, εξασφαλίζοντας ότι οι τεχνολογίες AI κατασκευάζονται με δίκαιο, διαφανή και περιεκτικό τρόπο. Αυτές οι αρχές αποτελούν τη βάση για την αξιολόγηση της ασφάλειας των μοντέλων AI.

Οι Αρχές Υπεύθυνης Τεχνητής Νοημοσύνης της Microsoft περιλαμβάνουν:

- **Δικαιοσύνη και Περιεκτικότητα**: Τα συστήματα AI θα πρέπει να αντιμετωπίζουν όλους δίκαια και να αποφεύγουν να επηρεάζουν ομάδες ανθρώπων με παρόμοια χαρακτηριστικά με διαφορετικό τρόπο. Για παράδειγμα, όταν τα συστήματα AI παρέχουν καθοδήγηση για ιατρικές θεραπείες, αιτήσεις δανείων ή απασχόληση, θα πρέπει να κάνουν τις ίδιες συστάσεις σε όλους όσους έχουν παρόμοια συμπτώματα, οικονομικές συνθήκες ή επαγγελματικά προσόντα.

- **Αξιοπιστία και Ασφάλεια**: Για να οικοδομηθεί εμπιστοσύνη, είναι κρίσιμο να λειτουργούν τα συστήματα AI αξιόπιστα, με ασφάλεια και συνεπώς. Αυτά τα συστήματα πρέπει να λειτουργούν όπως αρχικά σχεδιάστηκαν, να ανταποκρίνονται με ασφάλεια σε απροσδόκητες καταστάσεις και να αντιστέκονται σε επιβλαβείς χειρισμούς. Η συμπεριφορά τους και η ποικιλία των συνθηκών που μπορούν να χειριστούν αντανακλούν το φάσμα καταστάσεων και συνθηκών που οι προγραμματιστές ανέμεναν κατά το σχεδιασμό και τις δοκιμές.

- **Διαφάνεια**: Όταν τα συστήματα AI βοηθούν στη λήψη αποφάσεων με σημαντικές επιπτώσεις στη ζωή των ανθρώπων, είναι κρίσιμο οι άνθρωποι να κατανοούν πώς ελήφθησαν αυτές οι αποφάσεις. Για παράδειγμα, μια τράπεζα μπορεί να χρησιμοποιεί ένα σύστημα AI για να αποφασίσει αν ένα άτομο είναι πιστωτικά αξιόπιστο. Μια εταιρεία μπορεί να χρησιμοποιεί ένα σύστημα AI για να καθορίσει τους πιο κατάλληλους υποψηφίους για πρόσληψη.

- **Απόρρητο και Ασφάλεια**: Καθώς η AI γίνεται πιο διαδεδομένη, η προστασία του απορρήτου και η ασφάλεια προσωπικών και επιχειρηματικών πληροφοριών γίνονται ολοένα πιο σημαντικές και σύνθετες. Με την AI, το απόρρητο και η ασφάλεια δεδομένων απαιτούν στενή προσοχή επειδή η πρόσβαση στα δεδομένα είναι απαραίτητη για να κάνουν τα συστήματα AI ακριβείς και ενημερωμένες προβλέψεις και αποφάσεις για τους ανθρώπους.

- **Λογοδοσία**: Οι άνθρωποι που σχεδιάζουν και αναπτύσσουν συστήματα AI πρέπει να είναι υπόλογοι για τον τρόπο λειτουργίας των συστημάτων τους. Οι οργανισμοί θα πρέπει να χρησιμοποιούν πρότυπα του κλάδου για να αναπτύξουν κανόνες λογοδοσίας. Αυτοί οι κανόνες μπορούν να εξασφαλίσουν ότι τα συστήματα AI δεν είναι η τελική αρχή σε καμία απόφαση που επηρεάζει τη ζωή των ανθρώπων. Μπορούν επίσης να εξασφαλίσουν ότι οι άνθρωποι διατηρούν ουσιαστικό έλεγχο επί των διαφορετικά πολύ αυτόνομων συστημάτων AI.

#### Μετρικές ασφάλειας

Σε αυτό το μάθημα, θα αξιολογήσετε την ασφάλεια του εμπλουτισμένου μοντέλου Phi-3 χρησιμοποιώντας τις μετρικές ασφάλειας του Microsoft Foundry. Αυτές οι μετρικές σας βοηθούν να αξιολογήσετε την πιθανότητα του μοντέλου να παράγει επιβλαβές περιεχόμενο και την ευπάθειά του σε επιθέσεις jailbreak. Οι μετρικές ασφάλειας περιλαμβάνουν:

- **Περιεχόμενο σχετικό με αυτοκαταστροφή**: Αξιολογεί αν το μοντέλο έχει τάση να παράγει περιεχόμενο σχετικό με αυτοκαταστροφή.
- **Μίσος και άδικο περιεχόμενο**: Αξιολογεί αν το μοντέλο έχει τάση να παράγει περιεχόμενο μίσους ή άδικο.
- **Βίαιο περιεχόμενο**: Αξιολογεί αν το μοντέλο έχει τάση να παράγει βίαιο περιεχόμενο.
- **Σεξουαλικό περιεχόμενο**: Αξιολογεί αν το μοντέλο έχει τάση να παράγει ακατάλληλο σεξουαλικό περιεχόμενο.

Η αξιολόγηση αυτών των πτυχών διασφαλίζει ότι το μοντέλο AI δεν παράγει επιβλαβές ή προσβλητικό περιεχόμενο, ευθυγραμμίζοντάς το με τις κοινωνικές αξίες και τα ρυθμιστικά πρότυπα.

#### Μετρικές απόδοσης

Σε αυτό το μάθημα, θα αξιολογήσετε την απόδοση του εμπλουτισμένου μοντέλου Phi-3 / Phi-3.5 χρησιμοποιώντας τις μετρικές απόδοσης του Microsoft Foundry. Αυτές οι μετρικές σας βοηθούν να αξιολογήσετε την αποτελεσματικότητα του μοντέλου στην παραγωγή ακριβών, σχετικών και συνεκτικών απαντήσεων. Οι μετρικές απόδοσης περιλαμβάνουν:

- **Επικύρωση βάσει στοιχείων (Groundedness)**: Αξιολογεί πόσο καλά οι παραγόμενες απαντήσεις ευθυγραμμίζονται με τις πληροφορίες από την πηγή εισόδου.
- **Συνάφεια (Relevance)**: Αξιολογεί τη συνάφεια των παραγόμενων απαντήσεων με τις δοθείσες ερωτήσεις.
- **Συνοχή (Coherence)**: Αξιολογεί πόσο ομαλή είναι η ροή του παραγόμενου κειμένου, πόσο φυσικά διαβάζεται και πόσο μοιάζει με ανθρώπινη γλώσσα.
- **Ρευστότητα (Fluency)**: Αξιολογεί την γλωσσική δεξιότητα του παραγόμενου κειμένου.
- **Ομοιότητα με GPT**: Συγκρίνει την παραγόμενη απάντηση με την πραγματική ως προς την ομοιότητα.
- **F1 Score**: Υπολογίζει τον λόγο των κοινών λέξεων μεταξύ της παραγόμενης απάντησης και της αρχικής πηγής.

Αυτές οι μετρικές σας βοηθούν να αξιολογήσετε την αποτελεσματικότητα του μοντέλου στην παραγωγή ακριβών, σχετικών και συνεκτικών απαντήσεων.

## **Σενάριο 2: Αξιολόγηση του μοντέλου Phi-3 / Phi-3.5 στο Microsoft Foundry**

### Προτού ξεκινήσετε

Αυτό το μάθημα ακολουθεί τις προηγούμενες δημοσιεύσεις στο blog, "[Fine-Tune and Integrate Custom Phi-3 Models with Prompt Flow: Step-by-Step Guide](https://techcommunity.microsoft.com/t5/educator-developer-blog/fine-tune-and-integrate-custom-phi-3-models-with-prompt-flow/ba-p/4178612?wt.mc_id=studentamb_279723)" και "[Fine-Tune and Integrate Custom Phi-3 Models with Prompt Flow in Microsoft Foundry](https://techcommunity.microsoft.com/t5/educator-developer-blog/fine-tune-and-integrate-custom-phi-3-models-with-prompt-flow-in/ba-p/4191726?wt.mc_id=studentamb_279723)." Σε αυτές τις δημοσιεύσεις, καθοδηγηθήκαμε στη διαδικασία εμπλουτισμένης εκπαίδευσης ενός μοντέλου Phi-3 / Phi-3.5 στο Microsoft Foundry και στην ενσωμάτωσή του με το Prompt flow.

Σε αυτό το μάθημα, θα αναπτύξετε ένα μοντέλο Azure OpenAI ως αξιολογητή στο Microsoft Foundry και θα το χρησιμοποιήσετε για να αξιολογήσετε το εμπλουτισμένο μοντέλο Phi-3 / Phi-3.5 σας.

Πριν ξεκινήσετε αυτό το μάθημα, βεβαιωθείτε ότι έχετε τα εξής προαπαιτούμενα, όπως περιγράφονται στα προηγούμενα μαθήματα:

1. Ένα προετοιμασμένο σύνολο δεδομένων για την αξιολόγηση του εμπλουτισμένου μοντέλου Phi-3 / Phi-3.5.

1. Ένα Prompt flow ενσωματωμένο με το εμπλουτισμένο μοντέλο Phi-3 / Phi-3.5 στο Microsoft Foundry.

> [!NOTE]
> Θα χρησιμοποιήσετε το αρχείο *test_data.jsonl*, που βρίσκεται στον φάκελο δεδομένων από το σύνολο δεδομένων **ULTRACHAT_200k** που κατέβηκε στις προηγούμενες δημοσιεύσεις, ως το σύνολο δεδομένων για να αξιολογήσετε το εμπλουτισμένο μοντέλο Phi-3 / Phi-3.5.

#### Ενσωμάτωση του προσαρμοσμένου μοντέλου Phi-3 / Phi-3.5 με το Prompt flow στο Microsoft Foundry (πρώτη προσέγγιση κώδικα)

> [!NOTE]
> Αν ακολουθήσατε την προσέγγιση χαμηλού κώδικα όπως περιγράφεται στο "[Fine-Tune and Integrate Custom Phi-3 Models with Prompt Flow in Microsoft Foundry](https://techcommunity.microsoft.com/t5/educator-developer-blog/fine-tune-and-integrate-custom-phi-3-models-with-prompt-flow-in/ba-p/4191726?wt.mc_id=studentamb_279723)", μπορείτε να παραλείψετε αυτή την άσκηση και να προχωρήσετε στην επόμενη.
> Ωστόσο, αν ακολουθήσατε την προσέγγιση πρώτα κώδικα που περιγράφεται στο "[Fine-Tune and Integrate Custom Phi-3 Models with Prompt Flow: Step-by-Step Guide](https://techcommunity.microsoft.com/t5/educator-developer-blog/fine-tune-and-integrate-custom-phi-3-models-with-prompt-flow/ba-p/4178612?wt.mc_id=studentamb_279723)" για να εμπλουτίσετε και να αναπτύξετε το μοντέλο Phi-3 / Phi-3.5 σας, η διαδικασία σύνδεσης του μοντέλου σας με το Prompt flow είναι ελαφρώς διαφορετική. Θα μάθετε αυτή τη διαδικασία σε αυτή την άσκηση.

Για να προχωρήσετε, πρέπει να ενσωματώσετε το εμπλουτισμένο μοντέλο Phi-3 / Phi-3.5 σας στο Prompt flow στο Microsoft Foundry.

1. Επιλέξτε **Όλοι οι κόμβοι** από την καρτέλα στα αριστερά.

1. Επιλέξτε **+ Νέος κόμβος** από το μενού πλοήγησης.

    ![Δημιουργία κόμβου.](../../../../../../translated_images/el/create-hub.5be78fb1e21ffbf1.webp)

    - Επιλέξτε την **Συνδρομή** Azure σας.
    - Επιλέξτε την **Ομάδα πόρων** που θα χρησιμοποιήσετε (δημιουργήστε μία νέα, αν χρειάζεται).
    - Επιλέξτε την **Τοποθεσία** που θέλετε να χρησιμοποιήσετε.
    - Επιλέξτε το **Σύνδεση με Azure AI Services** που θα χρησιμοποιήσετε (δημιουργήστε μία νέα, αν χρειάζεται).
    - Επιλέξτε **Σύνδεση με Azure AI Search** για **Παράλειψη σύνδεσης**.

#### Προσθήκη προσαρμοσμένης σύνδεσης για το προσαρμοσμένο μοντέλο Phi-3 / Phi-3.5

Για να ενσωματώσετε το προσαρμοσμένο μοντέλο Phi-3 / Phi-3.5 με το Prompt flow, πρέπει να αποθηκεύσετε το endpoint και το κλειδί του μοντέλου σε μια προσαρμοσμένη σύνδεση. Αυτή η ρύθμιση εξασφαλίζει πρόσβαση στο προσαρμοσμένο μοντέλο Phi-3 / Phi-3.5 μέσα στο Prompt flow.

#### Ρύθμιση κλειδιού api και uri endpoint του προσαρμοσμένου μοντέλου Phi-3 / Phi-3.5

1. Επισκεφθείτε το [Azure ML Studio](https://ml.azure.com/home?wt.mc_id=studentamb_279723).

1. Μεταβείτε στο χώρο εργασίας Azure Machine learning που δημιουργήσατε.

1. Επιλέξτε **Endpoints** από την καρτέλα στα αριστερά.

    ![Επιλογή endpoints.](../../../../../../translated_images/el/select-endpoints.ee7387ecd68bd18d.webp)

1. Επιλέξτε το endpoint που δημιουργήσατε.

    ![Επιλογή endpoints.](../../../../../../translated_images/el/select-endpoint-created.9f63af5e4cf98b2e.webp)

1. Επιλέξτε **Χρήση** από το μενού πλοήγησης.

#### Ρύθμιση Prompt flow για συνομιλία με το προσαρμοσμένο μοντέλο Phi-3 / Phi-3.5

Πρέπει να ενσωματώσετε το προσαρμοσμένο μοντέλο Phi-3 / Phi-3.5 σε ένα Prompt flow. Ωστόσο, το υπάρχον Prompt flow που παρέχεται δεν έχει σχεδιαστεί γι’ αυτόν τον σκοπό. Επομένως, πρέπει να σχεδιάσετε ξανά το Prompt flow για να επιτρέψετε την ενσωμάτωση του προσαρμοσμένου μοντέλου.

1. Στο Prompt flow, εκτελέστε τις ακόλουθες εργασίες για να αναδομήσετε το υπάρχον flow:

    - Επιλέξτε **Λειτουργία ακατέργαστου αρχείου**.
    - Διαγράψτε όλο τον υπάρχοντα κώδικα στο αρχείο *flow.dag.yml*.
    - Προσθέστε τον παρακάτω κώδικα στο *flow.dag.yml*.

        ```yml
        inputs:
          input_data:
            type: string
            default: "Who founded Microsoft?"

        outputs:
          answer:
            type: string
            reference: ${integrate_with_promptflow.output}

        nodes:
        - name: integrate_with_promptflow
          type: python
          source:
            type: code
            path: integrate_with_promptflow.py
          inputs:
            input_data: ${inputs.input_data}
        ```

    - Επιλέξτε **Αποθήκευση**.

    ![Επιλογή λειτουργίας ακατέργαστου αρχείου.](../../../../../../translated_images/el/select-raw-file-mode.06c1eca581ce4f53.webp)

1. Προσθέστε τον παρακάτω κώδικα στο *integrate_with_promptflow.py* για να χρησιμοποιήσετε το προσαρμοσμένο μοντέλο Phi-3 / Phi-3.5 στο Prompt flow.

    ```python
    import logging
    import requests
    from promptflow import tool
    from promptflow.connections import CustomConnection

    # Ρύθμιση καταγραφής
    logging.basicConfig(
        format="%(asctime)s - %(levelname)s - %(name)s - %(message)s",
        datefmt="%Y-%m-%d %H:%M:%S",
        level=logging.DEBUG
    )
    logger = logging.getLogger(__name__)

    def query_phi3_model(input_data: str, connection: CustomConnection) -> str:
        """
        Send a request to the Phi-3 / Phi-3.5 model endpoint with the given input data using Custom Connection.
        """

        # "connection" είναι το όνομα της Προσαρμοσμένης Σύνδεσης, "endpoint", "key" είναι τα κλειδιά στην Προσαρμοσμένη Σύνδεση
        endpoint_url = connection.endpoint
        api_key = connection.key

        headers = {
            "Content-Type": "application/json",
            "Authorization": f"Bearer {api_key}"
        }
    data = {
        "input_data": [input_data],
        "params": {
            "temperature": 0.7,
            "max_new_tokens": 128,
            "do_sample": True,
            "return_full_text": True
            }
        }
        try:
            response = requests.post(endpoint_url, json=data, headers=headers)
            response.raise_for_status()
            
            # Καταγράψτε την πλήρη απόκριση JSON
            logger.debug(f"Full JSON response: {response.json()}")

            result = response.json()["output"]
            logger.info("Successfully received response from Azure ML Endpoint.")
            return result
        except requests.exceptions.RequestException as e:
            logger.error(f"Error querying Azure ML Endpoint: {e}")
            raise

    @tool
    def my_python_tool(input_data: str, connection: CustomConnection) -> str:
        """
        Tool function to process input data and query the Phi-3 / Phi-3.5 model.
        """
        return query_phi3_model(input_data, connection)

    ```

### Ανασκόπηση των Αποτελεσμάτων Αξιολόγησης

> [!NOTE]
> Τα αποτελέσματα που παρουσιάζονται παρακάτω προορίζονται να απεικονίσουν τη διαδικασία αξιολόγησης. Σε αυτό το σεμινάριο, χρησιμοποιήσαμε ένα μοντέλο προσαρμοσμένο σε σχετικά μικρό σύνολο δεδομένων, το οποίο μπορεί να οδηγήσει σε υποβέλτιστα αποτελέσματα. Τα πραγματικά αποτελέσματα ενδέχεται να διαφέρουν σημαντικά ανάλογα με το μέγεθος, την ποιότητα και τη διαφορετικότητα του χρησιμοποιούμενου συνόλου δεδομένων, καθώς και την ειδική διαμόρφωση του μοντέλου.

Μόλις ολοκληρωθεί η αξιολόγηση, μπορείτε να ανασκοπήσετε τα αποτελέσματα τόσο για τις μετρήσεις απόδοσης όσο και για τις μετρήσεις ασφαλείας.
