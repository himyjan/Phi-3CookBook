# Wsparcie sprzętowe Phi

Microsoft Phi został zoptymalizowany pod kątem ONNX Runtime i obsługuje Windows DirectML. Działa dobrze na różnych typach sprzętu, w tym GPU, CPU, a nawet urządzeniach mobilnych.

## Sprzęt urządzenia
Konkretnie, obsługiwany sprzęt obejmuje:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPU, 128 GiB pamięci)

## Mobilny SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 lub nowszy Procesor A16/A17

## Specyfikacja sprzętowa Phi

- Wymagana minimalna konfiguracja.
- Windows: GPU kompatybilne z DirectX 12 i minimum 4 GB łącznej pamięci RAM

CUDA: GPU NVIDIA z Compute Capability >= 7.02

## Uruchamianie onnxruntime na wielu GPU

Aktualnie dostępne modele Phi ONNX są przeznaczone tylko dla 1 GPU. Możliwe jest wsparcie multi-GPU dla modeli Phi, jednak ORT z 2 GPU nie gwarantuje większej przepustowości w porównaniu do 2 instancji ort. Proszę zapoznać się z [ONNX Runtime](https://onnxruntime.ai/) dla najnowszych aktualizacji.

Na konferencji [Build 2024 zespół GenAI ONNX](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) ogłosił, że włączył multi-instancję zamiast multi-GPU dla modeli Phi.

Obecnie pozwala to na uruchomienie jednej instancji onnxruntime lub onnxruntime-genai z użyciem zmiennej środowiskowej CUDA_VISIBLE_DEVICES w ten sposób.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

Zapraszamy do dalszego eksplorowania Phi w [Microsoft Foundry](https://ai.azure.com)
