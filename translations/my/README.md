Phi သည် Microsoft မှတီထွင်ထားသော အခမဲ့မူရင်း AI မော်ဒယ်စီးရီးတစ်ခုဖြစ်သည်။

Phi သည် လတ်တလောအတွက် အကျွမ်းတဝင်ဆုံးနှင့် စျေးကွက်ထဲတွင် အကျိုးရှိဆုံးသေးငယ်သော ဘာသာစကားမော်ဒယ် (SLM) ဖြစ်ပြီး၊ ဘာသာစကားသိပ္ပံ၊ သွေးထွက်စဉ်၊ စာသား/စကားပြော ထုတ်လုပ်ခြင်း၊ ကုဒ်ရေးခြင်း၊ ဓာတ်ပုံ၊ အသံနှင့် အခြားအခြေအနေများတွင် အရမ်းကောင်းမွန်သော စမ်းသပ်မှုများရှိသည်။

သင်သည် Phi ကို မိုးကောင်းကင်သို့ သို့မဟုတ် အရှေ့နှင့်နေရာဆိုင်ရာ စက်များသို့ တပ်ဆင်နိုင်ပြီး၊ ကွန်ပျူတာစွမ်းအားအနည်းငယ်ဖြင့် သွင်ပြင်ရေး AI အပလီကေးရှင်းများကို အလွယ်တကူ တည်ဆောက်နိုင်ပါသည်။

> **ဒေသဆိုင်ရာတွင် Clone လုပ်လိုပါသလား?**
>
> ဤ repository တွင် ဘာသာစကား ၅၀ ကျော် ပါဝင်ပြီး၊ ဒါက ဒေါင်းလုပ်အရွယ်အစားကို အလွန် တိုးစေပါသည်။ ဘာသာပြန်ချက်များမပါဘဲ clone လုပ်ချင်ပါက sparse checkout ကို အသုံးပြုပါ။
>
> **Bash / macOS / Linux:**
> ```bash
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone '/*' '!translations' '!translated_images'
> ```
>
> **CMD (Windows):**
> ```cmd
> git clone --filter=blob:none --sparse https://github.com/microsoft/PhiCookBook.git
> cd PhiCookBook
> git sparse-checkout set --no-cone "/*" "!translations" "!translated_images"
> ```
>

## တာဝန်ရှိသော AI

Microsoft သည် ကျွန်ုပ်တို့၏ AI ထုတ်ကုန်များကို တာဝန်ရှိစွာ အသုံးပြုနိုင်ရန် ရည်ရွယ်ကာ ကျွန်ုပ်တို့၏ သင်ယူမှုများကိုမျှဝေခြင်းနှင့် Transparency Notes နှင့် Impact Assessments ကဲ့သို့သော ကိရိယာများမှတဆင့် ယုံကြည်မှုအခြေပြု မဟာဘောင်များ တည်ဆောက်ခြင်းကို ကတိကဝတ်ပြုပါသည်။ ဤစနစ်များအများအား [https://aka.ms/RAI](https://aka.ms/RAI) တွင် ရှာဖွေတွေ့ရှိနိုင်ပါသည်။
Microsoft ၏ တာဝန်ရှိသော AI များဆောင်ရွက်ပုံသည် တရားမျှတမှု၊ ယုံကြည်စိတ်ချရမှုနှင့် လုံခြုံမှု၊ ကိုယ်ရေးရာဇဝင်နှင့် လုံခြုံရေး၊ ပါဝင်မှု၊ ထွက်ကြောင်းပြတ်သားမှုနှင့် တာဝန်ခံမှုတို့ ပေါ်တွင် အခြေခံထားပါသည်။

ဤနမူနာတွင် အသုံးပြုထားသည့် အကြီးစား သဘာဝဘာသာစကား၊ ပုံနှင့် အသံ မော်ဒယ်များသည် တရားမျှတမှုမရှိခြင်း၊ ယုံကြည်နိုင်စိတ်ချမရခြင်း သို့မဟုတ် ရိုက်ခတ်မှုဖြစ်နိုင်သည့် လုပ်ရပ်များ ပြုလုပ်နိုင်သဖြင့် ထိခိုက်မှုများ ဖြစ်ပေါ်စေနိုင်သည်။ အန္တရာယ်များနှင့် ကန့်သတ်ချက်များအား သိရှိရန် [Azure OpenAI service Transparency note](https://learn.microsoft.com/legal/cognitive-services/openai/transparency-note?tabs=text) ကို ဆက်လက် ကြည့်ရှုရန် တောင်းဆိုပါသည်။

ဤအန္တရာယ်များကို လျှော့ချရန် အကြံပြုထားသော နည်းလမ်းမှာ သင်၏ စနစ်ဒီဇိုင်းအတွင်း တွက်ချက်စောင့်ကြည့်ခြင်းနှင့် ထိခိုက်မှုဖြစ်စေသော လုပ်ရပ်များကို ရှောင်ကြဉ်နိုင်သော လုံခြုံရေး စနစ်တစ်ခု ထည့်သွင်းခြင်းဖြစ်သည်။ [Azure AI Content Safety](https://learn.microsoft.com/azure/ai-services/content-safety/overview) သည် လွတ်လပ်စွာ ကာကွယ်ပေးသော အလွှာတစ်ခုဖြစ်ပြီး အသုံးပြုသူမှ ဖန်တီးထားသည့် content နှင့် AI ဖန်တီးထားသည့် content များကို ကာကွယ်စောင့်ကြည့်နိုင်ပါသည်။ Azure AI Content Safety သည် မတရားသော အကြောင်းအရာများကို ကာကွယ်ပေးနိုင်သည့် စာသားနှင့် ပုံ APIs များ ပါဝင်သည်။ Microsoft Foundry တွင် Content Safety တစ်ခုရှိပြီး မတရားသော အကြောင်းအရာများကို မတူညီသော မိုးဒယ်များဖြင့် ရှာဖွေအကြောင်းအရာ အသုံးပြု နမူနာကုဒ် များကြည့်ရှု၊ စမ်းသပ်နိုင်ပါသည်။ အောက်ပါ [quickstart documentation](https://learn.microsoft.com/azure/ai-services/content-safety/quickstart-text?tabs=visual-studio%2Clinux&pivots=programming-language-rest) မှ စတင်ကာ ဝန်ဆောင်မှုအတွက် တောင်းဆိုမှုများ ပြုလုပ်နည်းကို လမ်းညွှန်ပေးပါသည်။

နောက်ထပ် ၁ခု သတိပြုရန် အချက်မှာ အပလီကေးရှင်း၏ စုပေါင်းဆောင်ရွက်မှု ဖြစ်သည်။ multi-modal နှင့် multi-models အပလီကေးရှင်းများတွင် စနစ်သည် သင်နှင့် သင့်အသုံးပြုသူများ မျှော်မှန်းသလို ဆောင်ရွက်ရမည်ဟု ဆိုလိုသည်၊ ထိခိုက်မှု ဖြစ်စေသော ထွက်ရှိမှု မဖြစ်စေသည့် ဆောင်ရွက်မှု ပါဝင်သည်။ သင့် စုပေါင်းအပလီကေးရှင်း၏ ဆောင်ရွက်မှုကို [Performance and Quality and Risk and Safety evaluators](https://learn.microsoft.com/azure/ai-studio/concepts/evaluation-metrics-built-in) အသုံးပြုကာ အကဲဖြတ်ရန် အရေးကြီးပါသည်။ သင်၏ လိုအပ်ချက်အရ [custom evaluators](https://learn.microsoft.com/azure/ai-studio/how-to/develop/evaluate-sdk#custom-evaluators) များ ဖန်တီး၍ အကဲဖြတ်နိုင်ပါသည်။

သင့် AI အပလီကေးရှင်းကို သင့်ဖွံ့ဖြိုးမှုပတ်ဝန်းကျင်တွင် [Azure AI Evaluation SDK](https://microsoft.github.io/promptflow/index.html) အသုံးပြုကာ အကဲဖြတ်နိုင်ပါသည်။ စမ်းသပ်ဒေတာသိုသို့မဟုတ် ပစ်မှတ်အားယူ၍ သင်၏ generative AI အပလီကေးရှင်း ဖန်တီးမှုများကို မူရင်းအကဲဖြတ်သူများ သို့မဟုတ် သင်ရွေးချယ်ထားသော custom evaluators များဖြင့် အရေအတွက်ဖြင့် တိုင်းတာနိုင်ပါသည်။ azure ai evaluation sdk အသုံးပြုပြီး စနစ်အတွက် အကဲဖြတ်မှုစတင်ရန်အတွက် [quickstart guide](https://learn.microsoft.com/azure/ai-studio/how-to/develop/flow-evaluate-sdk) ကိုလိုက်နာနိုင်ပါသည်။ အကဲဖြတ်မှု အသုံးပြုခွင့် ရယူပြီးနောက် [Microsoft Foundry တွင် ရလဒ်များကို မြင်တွေ့နိုင်](https://learn.microsoft.com/azure/ai-studio/how-to/evaluate-flow-results) ပါသည်။
