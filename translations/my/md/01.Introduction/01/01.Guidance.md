### Guidance-AI နှင့် Phi မော်ဒယ်များကို ဝန်ဆောင်မှု (MaaS) အနေနှင့်  
ကျွန်ုပ်တို့သည် [Guidance](https://github.com/guidance-ai/guidance) ကို Microsoft Foundry တွင် Phi-3.5-mini serverless endpoint ထိပ်တန်းထံ ဆက်သွယ်ထားခြင်းဖြင့် အထွက်များကို အလွယ်တကူ ခန့်မှန်းနိုင်စေရန် အက်ပလီကေးရှင်းအတွက် သင့်တော်သည့် ဖွဲ့စည်းပုံ သတ်မှတ်ခြင်းဖြင့် ပိုမိုသေချာမြောက်စေရန် တင်ဆက်လိုက်ပါသည်။ Guidance ဖြင့် သင့်တွင် စျေးကြီးသော ထပ်မံကြိုးစားမှုများကို ဖယ်ရှားနိုင်ပြီး ဥပမာအားဖြင့် မော်ဒယ်ကို ကြိုတင် သတ်မှတ်ထားသော စာရင်းများ (ဥပမာ- ဆေးဘက်ကုဒ်များ) မှ ရွေးချယ်စေရန် ကန့်သတ်နိုင်သည်၊ ပေးထားသော အကြောင်းအရာမှ တိုက်ရိုက် ရှင်းလင်းစွာ ကိုးကားချက်များသာ ထုတ်လွှင့်စေရန် သို့မဟုတ် မည်သည့် regex ကိုမဆို အတိအကျလိုက်နာစေသည်။ Guidance သည် inference stack မှာ token တစ်ခုချင်းစီအလိုက် မော်ဒယ်ကို ဦးတည်ညွှန်ပေးကာ ကုန်ကျစရိတ်နှင့် အချိန်အလိုချိန်ကို ၃၀-၅၀% လျော့ချပွားစေပြီး ၎င်းသည် [Phi-3-mini serverless endpoint](https://aka.ms/try-phi3.5mini) များအတွက် ထူးခြား၍ တန်ဖိုးရှိသော ပြုပြင်ဖြည့်စွက်ချက်တစ်ခုဖြစ်သည်။

## [**Guidance-AI**](https://github.com/guidance-ai/guidance) သည် developer များကို AI မော်ဒယ်များကို ထိထိရောက်ရောက် ဖန်တီးတပ်ဆင်ရန်အတွက် ဒီဇိုင်းဆွဲထားသော ဖရိမ်းဝပ်တစ်ခုဖြစ်သည်။ ၎င်းသည် ခိုင်မာသော AI အက်ပလီကေးရှင်းများ တည်ဆောက်ရန် ကိရိယာများနှင့် အကောင်းဆုံး လေ့လာမှုများကို ပံ့ပိုးပေးရန် ဦးတည်သည်။

**Phi Models as a Service (MaaS)** နှင့် ပေါင်းစည်းစွာ အသုံးပြုလျှင် သက်သာစွာ၊ စွမ်းဆောင်ရည်မြင့် မျိုးဆက်အသစ်的小型စာသား မော်ဒယ်များ(SLMs) ကို စွမ်းဆောင်နိုင်သော အလွန်အစွမ်းထက်သော ဖြေရှင်းချက်တစ်ခု ဖြစ်ပါသည်။

**Guidance-AI** သည် လှုပ်ရှားမှုကြီးမားသော စာသားမော်ဒယ်များ (LLMs) ကို ထိန်းချုပ် ဦးတည်ရန် developer များအတွက် ဒီဇိုင်းဆွဲထားသော programming ဖရိမ်းဝပ်ဖြစ်သည်။ ၎င်းသည် ရလဒ်များကို အတိအကျစီစဉ်ပေးနိုင်ပြီး စံပြ prompting သို့မဟုတ် fine-tuning နည်းလမ်းများနှင့်ယှဉ်လျှင် latency နှင့်ကုန်ကျစရိတ် လျော့ပါးစေသည်။

### Guidance-AI ၏ အဓိက အင်္ဂါရပ်များ  
- **ထိရောက်သော ထိန်းချုပ်မှု**: စာသား မော်ဒယ် ထွက်ရှိမှုကို ထိန်းချုပ်ရန် developer များ အတွက် အကောင်အထည်ဖော်နိုင်ရန်၊ အရည်အသွေးမြင့်၊ သင့်တော်သော ရလဒ်များထုတ်နိုင်ခြင်းကို အာမခံသည်။  
- **ကုန်ကျစရိတ်နှင့် Latency လျော့ချမှု**: ထုတ်လုပ်ခြင်း လည်ပတ်မှုကို သက်သာစေပြီး မြန်ဆန်စေသည်။  
- **အလွယ်တကူ ပေါင်းစည်းနိုင်မှု**: Transformers, llama.cpp, AzureAI, VertexAI, နှင့် OpenAI အပါအဝင် backend မျိုးစုံနှင့် ပေါင်းစည်း အသုံးပြုနိုင်သည်။  
- **စုံလင်သော အထွက်ဖွဲ့စည်းပုံများ**: ဇယားအခြေအနေများ၊ လုပ်ဆောင်ချက်များနှင့် ကိရိယာအသုံးပြုခြင်းကဲ့သို့သော ထုတ်လွှင့်ချက်များကို ထောက်ပံ့ပေး၊ ရွှေ့ပြောင်းဖတ်ရှုရတရှင်းလင်းသော ရလဒ်များ ထုတ်ပေးနိုင်စေသည်။  
- **ကွဲပြားသော backend များပေါ်တွင် Guidance အစီအစဉ် တစ်ခုတည်း ပြေးဆွဲနိုင်ခြင်း**: သာမန် ပြောင်းလဲထားသော backend များတွင်အလွယ်တကူ အသုံးပြုနိုင်၍ လွယ်ကူမှုနှင့် ပိုမိုအသုံးဝင်မှု ရရှိစေသည်။

### နမူနာအသုံးပြုမှုများ  
- **ကန့်သတ်ထားသော ထုတ်လုပ်မှု**: မော်ဒယ်၏ ထွက်ရှိမှုကို regular expressions နှင့် context-free grammars ဖြင့် ဦးတည်ညွှန်ကြားခြင်း။  
- **ကိရိယာ ပေါင်းစည်းခြင်း**: ဘာသာရေးတွင် ကိန်းဂဏန်း calculator ကို စာသားထုတ်လုပ်မှုအလုပ်အတွက် အလိုအလျောက် ထပ်လွှင့်ပေးခြင်း။

အသေးစိတ် နှင့် နမူနာများအတွက် [Guidance-AI GitHub repository](https://github.com/guidance-ai/guidance) ကို လေ့လာကြည့်နိုင်သည်။

### Phi Models ၏ အဓိက အင်္ဂါရပ်များ  
1. **သက်သာသောကုန်ကျစရိတ်**: မျက်နှာသာထားသော စွမ်းဆောင်ရည်မြင့်ပြီး သက်သာစွာ ရရှိနိုင်သည်။  
2. **အချိန်လိုတယ်လီစွာအနိမ့်**: တုံ့ပြန်ချက် မြန်နှုန်းမြင့်လိုအပ်သည့် အချိန်နဲ့အီအလျစ် လိုအပ်ချက်များအတွက် သင့်တော်သည်။  
3. **ပြောင်းလဲမှု အလွယ်တကူသိမ်းဆည်းနိုင်မှု**: Cloud, edge နေရာများနှင့် offline ဖြစ်ရပ်များတွင် တပ်ဆင်နိုင်သည်။  
4. **စိတ်ကြိုက် ပြုပြင်နိုင်မှု**: domain-specific ဒေတာများဖြင့် fine-tune ပြုလုပ်၍ စွမ်းဆောင်ရည် မြှင့်တင်နိုင်သည်။  
5. **လုံခြုံရေးနှင့် ကန့်သတ်ချက်များ**: Microsoft ၏ AI မူဝါဒအတိုင်း တာဝန်ရှိခြင်း၊ ဖော်ပြချက်ရှင်းလင်းမှု၊ တရားမျှတမှု၊ ကိုင်တွယ်နိုင်မှု၊ လုံခြုံမှု၊ ကိုယ်ရေးဖွင့်ခြင်းနှင့် ပါဝင်ဆောင်ရွက်မှုတို့ကို ကြီးကြပ်ထားသည်။

### Phi Models as a Service (MaaS)  
Phi မော်ဒယ်များကို inference APIs များမှတဆင့် pay-as-you-go billing စနစ်ဖြင့် အသုံးပြုနိုင်သောကြောင့် သင့် application များတွင် ကြီးမားသော အစည်းအဝေးမလိုဘဲ လွယ်ကူစွာ ပေါင်းစည်းနိုင်သည်။

### Phi-3 ဖြင့် စတင်အသုံးပြုခြင်း  
Phi မော်ဒယ်များ အသုံးပြုလိုသူသည် [Azure AI model catalog](https://ai.azure.com/explore/models) သို့မဟုတ် GitHub Marketplace Models](https://github.com/marketplace/models) တွင် ကြိုတင်တည်ဆောက်ထားပြီး စိတ်ကြိုက် ပြင်ဆင်နိုင်သော မော်ဒယ်များကို ရှာဖွေ လေ့လာနိုင်သည်။ ထို့ပြင် [Microsoft Foundry](https://ai.azure.com) ကဲ့သို့ ကိရိယာများဖြင့် သင်၏ AI application များကို တည်ဆောက် တပ်ဆင်နိုင်သည်။
