# Suporte de Hardware Phi

O Microsoft Phi foi otimizado para ONNX Runtime e suporta Windows DirectML. Ele funciona bem em vários tipos de hardware, incluindo GPUs, CPUs e até dispositivos móveis.

## Hardware do Dispositivo  
Especificamente, o hardware suportado inclui:

- SKU de GPU: RTX 4090 (DirectML)
- SKU de GPU: 1 A100 80GB (CUDA)
- SKU de CPU: Standard F64s v2 (64 vCPUs, 128 GiB de memória)

## SKU Móvel

- Android - Samsung Galaxy S21
- Apple iPhone 14 ou superior com Processador A16/A17

## Especificação de Hardware Phi

- Configuração mínima necessária.
- Windows: GPU compatível com DirectX 12 e um mínimo de 4GB de RAM combinada

CUDA: GPU NVIDIA com Capacidade de Computação >= 7.02

## Executando onnxruntime em múltiplas GPUs

Atualmente, os modelos Phi ONNX disponíveis são apenas para 1 GPU. É possível suportar múltiplas GPUs para o modelo Phi, mas o ORT com 2 GPUs não garante que oferecerá maior rendimento em comparação com 2 instâncias do ORT. Por favor, consulte [ONNX Runtime](https://onnxruntime.ai/) para as últimas atualizações.

No [Build 2024, a Equipe GenAI ONNX](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) anunciou que eles habilitaram multi-instância em vez de multi-GPU para modelos Phi.

Atualmente, isso permite que você execute uma instância onnnxruntime ou onnxruntime-genai com a variável de ambiente CUDA_VISIBLE_DEVICES assim.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```
  
Fique à vontade para explorar mais sobre o Phi em [Microsoft Foundry](https://ai.azure.com)
