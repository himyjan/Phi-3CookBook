# Phi Family in AITK

[AI Toolkit for VS Code](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio) 通過整合來自 Microsoft Foundry Catalog 及其他目錄如 Hugging Face 的尖端 AI 開發工具及模型，簡化生成式 AI 應用程序的開發。你可以瀏覽由 GitHub Models 和 Microsoft Foundry Model Catalogs 提供的 AI 模型目錄，下載本地或遠程模型，進行微調、測試並在應用程序中使用它們。

AI Toolkit 預覽版將在本地運行。依據你選擇的模型，當地推理或微調可能需要 GPU，如 NVIDIA CUDA GPU。你也可以使用 AITK 直接運行 GitHub Models。

### 安裝 AI Toolkit

AI Toolkit 以 [Visual Studio Code 擴充套件](https://code.visualstudio.com/docs/setup/additional-components#_vs-code-extensions) 的形式提供，因此您需要先安裝 [VS Code](https://code.visualstudio.com/docs/setup/windows?WT.mc_id=aiml-137032-kinfeylo)，然後從 [VS Marketplace](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio) 下載 AI Toolkit。  
[AI Toolkit 可從 Visual Studio Marketplace 取得](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio)，可以像安裝其他 VS Code 擴充套件一樣安裝。

若您不熟悉安裝 VS Code 擴充套件，請依循以下步驟：

### 登入

1. 在 VS Code 活動欄選擇 **Extensions**
1. 在擴充套件搜尋欄輸入「AI Toolkit」
1. 選取「AI Toolkit for Visual Studio code」
1. 選擇 <strong>安裝</strong>

現在，您已準備好使用此擴充套件！

您將會被提示登入 GitHub，請點選「允許」繼續。您將被導向至 GitHub 登入頁面。

請登入並按照流程操作。成功後您將被導回 VS Code。

安裝完成後，您會在活動欄看到 AI Toolkit 的圖示。

讓我們開始探索可用操作！

### 可用操作

AI Toolkit 的主要側邊欄組織為  

- **Models**
- **Resources**
- **Playground**  
- **Fine-tuning**
- **Evaluation**

這些在 Resources 區段可用。初次使用請選擇 **Model Catalog**。

### 從型錄下載模型

從 VS Code 側邊欄啟動 AI Toolkit 後，您可以選擇以下選項：

- 從 **Model Catalog** 找到支援的模型並下載到本地

- 透過 AI Toolkit 命令面板將微調後模型部署至雲端

> [!NOTE]
>
> **GPU 對 CPU**
>
> 您會注意到模型卡上會顯示模型大小、平台及加速裝置類型（CPU、GPU）。在<strong>具備至少一個 GPU 的 Windows 設備</strong>上，選擇僅針對 Windows 的模型版本，將獲得最佳化效能。
>
> 這樣確保您的模型適配 DirectML 加速器。
>
> 模型名稱格式為：
>
> - `{model_name}-{accelerator}-{quantization}-{format}`。
>
>要檢查您的 Windows 裝置是否有 GPU，請開啟 <strong>工作管理員</strong>，並切換至 <strong>效能</strong> 頁籤。若有 GPU，將會以「GPU 0」或「GPU 1」等名稱顯示。

### 在您的應用程序中使用 REST API

AI Toolkit 內建本地 REST API 網頁伺服器，**埠號為 5272**，使用[OpenAI chat completions 格式](https://platform.openai.com/docs/api-reference/chat/create)。

這讓您可以本地測試應用程序，無需依賴雲端 AI 模型服務。例如，下列 JSON 檔展示如何配置請求主體：

```json
{
    "model": "Phi-4",
    "messages": [
        {
            "role": "user",
            "content": "what is the golden ratio?"
        }
    ],
    "temperature": 0.7,
    "top_p": 1,
    "top_k": 10,
    "max_tokens": 100,
    "stream": true
}
```

您可以使用（例如）[Postman](https://www.postman.com/) 或 CURL（Client URL）工具測試此 REST API：

```bash
curl -vX POST http://127.0.0.1:5272/v1/chat/completions -H 'Content-Type: application/json' -d @body.json
```

### 使用 Python 的 OpenAI 用戶端庫

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:5272/v1/", 
    api_key="x" # API 必需但未使用
)

chat_completion = client.chat.completions.create(
    messages=[
        {
            "role": "user",
            "content": "what is the golden ratio?",
        }
    ],
    model="Phi-4",
)

print(chat_completion.choices[0].message.content)
```

### 使用 .NET 的 Azure OpenAI 用戶端庫

使用 NuGet 將 [.NET 用 Azure OpenAI 用戶端庫](https://www.nuget.org/packages/Azure.AI.OpenAI/) 新增到您的專案：

```bash
dotnet add {project_name} package Azure.AI.OpenAI --version 1.0.0-beta.17
```

向專案新增名為 **OverridePolicy.cs** 的 C# 檔案，貼上以下程式碼：

```csharp
// OverridePolicy.cs
using Azure.Core.Pipeline;
using Azure.Core;

internal partial class OverrideRequestUriPolicy(Uri overrideUri)
    : HttpPipelineSynchronousPolicy
{
    private readonly Uri _overrideUri = overrideUri;

    public override void OnSendingRequest(HttpMessage message)
    {
        message.Request.Uri.Reset(_overrideUri);
    }
}
```

接著，將以下程式碼貼到您的 **Program.cs** 檔案中：

```csharp
// Program.cs
using Azure.AI.OpenAI;

Uri localhostUri = new("http://localhost:5272/v1/chat/completions");

OpenAIClientOptions clientOptions = new();
clientOptions.AddPolicy(
    new OverrideRequestUriPolicy(localhostUri),
    Azure.Core.HttpPipelinePosition.BeforeTransport);
OpenAIClient client = new(openAIApiKey: "unused", clientOptions);

ChatCompletionsOptions options = new()
{
    DeploymentName = "Phi-4",
    Messages =
    {
        new ChatRequestSystemMessage("You are a helpful assistant. Be brief and succinct."),
        new ChatRequestUserMessage("What is the golden ratio?"),
    }
};

StreamingResponse<StreamingChatCompletionsUpdate> streamingChatResponse
    = await client.GetChatCompletionsStreamingAsync(options);

await foreach (StreamingChatCompletionsUpdate chatChunk in streamingChatResponse)
{
    Console.Write(chatChunk.ContentUpdate);
}
```
