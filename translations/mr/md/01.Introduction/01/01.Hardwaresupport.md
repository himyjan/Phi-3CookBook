# Phi हार्डवेअर समर्थन

Microsoft Phi ला ONNX Runtime साठी ऑप्टिमाइझ केले गेले आहे आणि ते Windows DirectML ला समर्थन करते. हे GPU, CPU आणि अगदी मोबाइल डिव्हायसेस यांसह विविध हार्डवेअर प्रकारांवर चांगले काम करते.

## डिव्हाइस हार्डवेअर  
विशिष्टपणे, समर्थित हार्डवेअरमध्ये खालील गोष्टी समाविष्ट आहेत:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPUs, 128 GiB स्मृती)

## मोबाइल SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 किंवा त्यापुढील A16/A17 प्रोसेसर

## Phi हार्डवेअर विनिर्देश

- किमान कॉन्फिगरेशन आवश्यक.
- Windows: DirectX 12-समर्थित GPU आणि एकूण 4GB RAM किमान आवश्यक

CUDA: Compute Capability >= 7.02 असलेला NVIDIA GPU

![HardwareSupport](../../../../../translated_images/mr/01.phihardware.5d51b2377cba18af.webp)

## एकापेक्षा जास्त GPU वर onnxruntime चालवणे

सध्या उपलब्ध Phi ONNX मॉडेल्स फक्त 1 GPU साठी आहेत. Phi मॉडेलसाठी मल्टी-GPU समर्थन शक्य आहे, पण 2 GPU सह ORT हे 2 ORT इंस्टन्सच्या तुलनेत अधिक थ्रूपुट देईल याची हमी देत नाही. कृपया ताज्या अद्यतनोंसाठी [ONNX Runtime](https://onnxruntime.ai/) येथे भेट द्या.

[Build 2024 the GenAI ONNX Team](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) यांनी जाहीर केले की त्यांनी Phi मॉडेलसाठी मल्टी-GPU ऐवजी मल्टी-इंस्टन्स सक्षम केले आहे.

आत्तापर्यंत यामुळे आपण CUDA_VISIBLE_DEVICES पर्यावरणातील व्हेरिएबलसह एक onnnxruntime किंवा onnxruntime-genai इंस्टन्स चालवू शकता.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```
  
[Microsoft Foundry](https://ai.azure.com) मध्ये अजून Phi बद्दल मोकळेपणाने अन्वेषण करा.
