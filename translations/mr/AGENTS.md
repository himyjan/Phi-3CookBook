# AGENTS.md

## प्रोजेक्ट आढावा

PhiCookBook ही Microsoft च्या Phi परिवारातील लहान भाषा मॉडेल्स (SLMs) सह काम करण्यासाठी प्रत्यक्ष उदाहरणे, ट्यूटोरियल्स आणि दस्तऐवजीकरणांनचा समावेश असलेली सर्वसमावेशक कुकबुक रेपॉझिटरी आहे. रेपॉझिटरीमध्ये विविध वापर प्रकरणे दाखविली आहेत जसे की इनफेरन्स, फाइन-ट्युनिंग, क्वांटायजेशन, RAG अंमलबजावणी, आणि वेगवेगळ्या प्लॅटफॉर्म्स आणि फ्रेमवर्कवर मल्टीमोडल अनुप्रयोग.

**महत्त्वाच्या तंत्रज्ञान:**
- **भाषा:** Python, C#/.NET, JavaScript/Node.js
- **फ्रेमवर्क:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **प्लॅटफॉर्म:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **मॉडेल प्रकार:** Phi-3, Phi-3.5, Phi-4 (टेक्स्ट, व्हिजन, मल्टीमोडल, रिझनिंग प्रकार)

**रेपॉझिटरी संरचना:**
- `/code/` - कामातील कोड उदाहरणे आणि नमुना अंमलबजावणी
- `/md/` - सविस्तर दस्तऐवजीकरण, ट्यूटोरियल्स आणि वापर कसे करावे मार्गदर्शक  
- `/translations/` - बहुभाषिक अनुवाद (50+ भाषा स्वयंचलित वर्कफ्लोद्वारे)
- `/.devcontainer/` - डेव्ह कंटेनर कॉन्फिगरेशन (Python 3.12 सह Ollama)

   - कंटेनर आपोआप Python 3.12 आणि Ollama सह Phi-3 सह सेटअप होतो

### स्थानिक सेटअप

**पूर्वअट:**
- Python 3.12 किंवा नंतरचे
- .NET 8.0 SDK (C# उदाहरणांसाठी)
- Node.js 18+ आणि npm (JavaScript उदाहरणांसाठी)
- 16GB RAM किमान गरज

**इंस्टॉलेशन:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**Python उदाहरणांसाठी:**
विशिष्ट उदाहरण निर्देशिकांमध्ये जा आणि डिपेंडन्सी इंस्टॉल करा:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # जर requirements.txt अस्तित्वात असेल तर
```

**.NET उदाहरणांसाठी:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**JavaScript/Web उदाहरणांसाठी:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # विकास सर्व्हर सुरू करा
npm run build  # उत्पादनासाठी तयार करा
```

## चाचणी

ही रेपॉझिटरी पारंपरिक युनिट टेस्ट्ससह सॉफ्टवेअर प्रोजेक्ट ऐवजी उदाहरण कोड आणि ट्यूटोरियल्सचा समावेश करते. प्रमाणीकरण सामान्यतः खालीलप्रमाणे केले जाते:

1. **उदाहरणे चालवा** - प्रत्येक उदाहरण त्रुटीशिवाय चालणे आवश्यक आहे
2. **आउटपुट्स तपासा** - मॉडेल प्रतिसाद योग्य आहेत की नाही तपासा
3. **ट्यूटोरियल्सचे अनुसरण करा** - स्टेप-द्वारे-स्टेप मार्गदर्शक दस्तऐवजीकरणानुसार काम करावे

**सामान्य प्रमाणीकरण पद्धत:**
- लक्ष्य पर्यावरणात उदाहरणांची अंमलबजावणी तपासा
- आवश्यक असलेले अवलंबन योग्यरित्या इंस्टॉल झाले आहे का ते तपासा
- मॉडेल डाउनलोड/लोड यशस्वी आहे का ते तपासा
- अपेक्षित वर्तन दस्तऐवजीकरणास अनुरूप आहे का ते खात्री करा

## पुल रिक्वेस्ट मार्गदर्शक

### पाठवण्यापूर्वी

1. कृपया रेपॉझिटरी स्वतःच्या खात्यात Fork करा
2. प्रकारानुसार वेगवेगळे PR तयार करा:
   - एका PR मध्ये बग फिक्सेस
   - दुसऱ्या PR मध्ये दस्तऐवजीकरण अद्यतने
   - नवीन उदाहरणांसाठी स्वतंत्र PRs
   - टायपो फिक्सेस एकत्र करता येतात

3. मर्ज कॉन्फ्लिक्ट्स हाताळा:
   - बदल करण्याच्या आधी आपल्या स्थानीय `main` शाखा अपडेट करा
   - फॉर्मनली अपस्ट्रीमशी सिंक करा

4. अनुवाद PRs:
   - फोल्डरमधील सर्व फाइल्सचा अनुवाद असणे आवश्यक
   - मूळ भाषेशी सुसंगत संरचना राखा

### आवश्यक तपासण्या

PRs आपोआप GitHub वर्कफ्लोद्वारे प्रमाणीकरण करतात:

1. **सापेक्ष मार्ग तपासणी** - सर्व आंतरिक दुवे कार्यरत असावेत
   - VS Code मध्ये Ctrl+Click करून स्थानिकदृष्ट्या तपासा
   - मार्ग सुचनेसाठी VS Code वापरा (`./` किंवा `../`)

2. **URL स्थानिक तपासणी** - वेब URL मध्ये देश-स्थानिक नसावेत
   - `/en-us/`, `/en/` किंवा इतर भाषिक कोड काढा
   - सामान्य आंतरराष्ट्रीय URL वापरा

3. **ब्रोकन URL तपासणी** - सर्व URL 200 स्थिती परत करावेत
   - सबमिट करण्यापूर्वी दुवे प्रवेशयोग्य आहेत का तपासा
   - लक्षात ठेवा: काही अयशस्वीपणं नेटवर्क प्रतिबंधांमुळे असू शकते


उदाहरणे:
- `[docs] Add Phi-4 inference tutorial`

### बहुभाषिक समर्थन

हा एक बहुभाषिक रेपॉझिटरी आहे ज्यात उदाहरणे आहेत:
- **Python** - ML/AI वर्कफ्लोज, Jupyter नोटबुक्स, फाइन-ट्यूनिंग
- **C#/.NET** - एंटरप्राइझ अनुप्रयोग, ONNX Runtime एकत्रीकरण
- **JavaScript** - वेब-आधारित AI, WebGPU सह ब्राउझर इनफेरन्स

तुमच्या वापर केस आणि वितरण लक्ष्याला अनुरूप भाषा निवडा.
