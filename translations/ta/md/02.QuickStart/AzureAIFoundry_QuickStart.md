# **Microsoft Foundry-ல் Phi-3 பயன்படுத்துவது**

சுயமரியாதை ஆயிரமான Generative AI -யின் வளர்ச்சியுடன், பல்வேறு LLM மற்றும் SLM, நிறுவன தரவுகளின் ஒருங்கிணைப்பு, நுட்பப்படுத்தல்/RAG செயல்பாடுகள் மற்றும் LLM மற்றும் SLM ஒருங்கிணைக்கும் பிறகு பல்வேறு நிறுவன வணிகங்களின் மதிப்பீடு ஆகியவற்றை ஒருங்கிணைக்க ஒன்றிணைந்த பிளாட்ஃபாரத்தை பயன்படுத்துவோம் என்று நம்புகிறோம், இதனால் சுயமரியாதை ஆயிரமான செயற்கை நுண்ணறிவு சிறந்த செயலிகளைச் சுலபமாக அமல்படுத்த முடியும். [Microsoft Foundry](https://ai.azure.com) என்பது நிறுவன மட்டத்திலான சுயமரியாதை ஆயிரமான செயற்கை நுண்ணறிவு பயன்பாட்டு பிளாட்ஃபாரம் ஆகும்.

![aistudo](../../../../translated_images/ta/aifoundry_home.f28a8127c96c7d93.webp)

Microsoft Foundry-உடன், நீங்கள் பெரிய மொழி மாதிரி (LLM) பதில்களை மதிப்பாய்வு செய்து, சிறந்த செயல்பாட்டிற்காக prompt flow மூலம் prompt பயன்பாட்டு கூறுகளை ஒருங்கிணைக்கலாம். இந்த பிளாட்ஃபாரம் சான்று கொள்கை கோடுகளை முழு தயாரிப்பாக மாற்ற விரிவாக்கத்தை எளிதாக்குகின்றது. தொடர்ச்சியான கண்காணிப்பு மற்றும் மேம்படுத்தல் நீண்ட கால வெற்றிக்கு ஆதரவு வழங்குகின்றன.

நாங்கள் எளிய படிகளைப் பின்பற்றி Microsoft Foundry-ல் Phi-3 மாதிரியை விரைவில் பிணைத்தமைக்க முடியும், பிறகு Microsoft Foundry-யைப் பயன்படுத்தி Phi-3 தொடர்புடைய Playground/Chat, நுட்பப்படுத்தல், மதிப்பீடு மற்றும் பிற தொடர்புடைய பணிகளை முடிக்கலாம்.

## **1. தயார் செயல்**

உங்கள் இயந்திரத்தில் ஏற்கனவே [Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/overview?WT.mc_id=aiml-138114-kinfeylo) நிறுவப்பட்டிருந்தால், இந்த வார்ப்புருவை புதிய கோப்பகத்தில் இந்த கட்டளை இயக்குவதையேயே பயன்படுத்தலாம்.

## கைமுறை உருவாக்குதல்

Microsoft Foundry திட்டம் மற்றும் ஹப்பை உருவாக்குவதன் மூலம் உங்கள் AI பணிகளை நன்கு ஒழுங்குபடுத்தி நிர்வகிக்க முடியும். தொடங்குவதற்கான கட்டணம் படி வழிகாட்டி இதோ:

### Microsoft Foundry-ல் ஒரு திட்டம் உருவாக்குதல்

1. **Microsoft Foundry-க்கு செல்லவும்**: Microsoft Foundry போர்ட்டலில் உள்நுழைக.
2. **ஒரு திட்டம் உருவாக்கவும்**:
   - நீங்கள் ஏற்கனவே திட்டத்தில் இருந்தால், பக்கத்தின் இடது மேல் பகுதியில் உள்ள "Microsoft Foundry" என்பதைத் தேர்ந்தெடுத்து ஹோம் பக்கத்திற்கு செல்லுங்கள்.
   - "+ Create project" என்பதைக் கிளிக் செய்யுங்கள்.
   - திட்டத்திற்கான பெயரை உள்ளிடுங்கள்.
   - உங்கள் ஹப்பில் ஏதேனும் இருந்தால் அது இயலுமையாக தேர்ந்தெடுக்கப்படும். நீங்கள் பல ஹப்புகளை அணுகினால், டிராப்டவுனில் வேறொரு ஹப்பைத் தேர்ந்தெடுக்கலாம். புதிய ஹப்பை உருவாக்க விரும்பினால், "Create new hub" என்பதைக் கிளிக் செய்து ஒரு பெயரை வழங்குங்கள்.
   - "Create" என்பதைக் தேர்ந்தெடுங்க.

### Microsoft Foundry-ல் ஹப் உருவாக்குதல்

1. **Microsoft Foundry-க்கு செல்லவும்**: உங்கள் Azure கணக்குடன் உள்நுழைக.
2. **ஒரு ஹப் உருவாக்கவும்**:
   - இடது மெனுனிலிருந்து Management center-ஐத் தேர்ந்தெடுக்கவும்.
   - "All resources" என்பதைத் தேர்ந்தெடுத்து, "+ New project" அருகேயுள்ள கீழே உள்ள அம்புத்தானை கிளிக் செய்து "+ New hub" என்பதைக் தேர்ந்தெடுக்கவும்.
   - "Create a new hub" உரையாடலில், உங்கள் ஹப்புக்கான பெயரை (எ.கா., contoso-hub) உள்ளிடவும் மற்றும் பிற புலங்களை விருப்பப்படி மாற்றவும்.
   - "Next" என்பதைக் கிளிக் செய்து விவரங்களை பரிசீலனை செய்து பிறகு "Create" என்பதைக் கிளிக் செய்யவும்.

மேலும் விரிவான விளக்கத்திற்கான அதிகாரப்பூர்வ [Microsoft வழிகாட்டியை](https://learn.microsoft.com/azure/ai-studio/how-to/create-projects) பார்க்கலாம்.

வெற்றிகரமாக உருவாக்கிய பிறகு, நீங்கள் [ai.azure.com](https://ai.azure.com/) மூலமாக நீங்கள் உருவாக்கிய ஸ்டுடியோவை அணுகலாம்.

ஒரே AI Foundry-ல் பல திட்டங்கள் இருக்க முடியும். AI Foundry-ல் ஒரு திட்டத்தை உருவாக்கி தயார் செய்யவும்.

Microsoft Foundry [QuickStarts](https://learn.microsoft.com/azure/ai-studio/quickstarts/get-started-code) உருவாக்குங்கள்

## **4. Microsoft Foundry-ல் இருந்து மாதிரியை நிறுவுதல்**

Azure Model Catalog-இல் இருந்து மாதிரியை நிறுவ, இந்த படிகளைக் காணலாம்:

- Microsoft Foundry-ல் உள்நுழைக.
- Microsoft Foundry மாதிரி காட்சியிலிருந்து நீங்கள் நிறுவ விரும்பும் மாதிரியைத் தேர்ந்தெடுக்கவும்.
- மாதிரியின் Details பக்கத்தில் Deploy-ஐ தேர்வு செய்த பின் Serverless API with Azure AI Content Safety-ஐ தேர்ந்தெடுக்கவும்.
- உங்கள் மாதிரிகளை நிறுவ விரும்பும் திட்டத்தைத் தேர்ந்தெடுக்கவும். Serverless API வாய்ப்பை பயன்படுத்த உங்கள் வேலைப்பகுதி East US 2 அல்லது Sweden Central பிராந்தியத்தில் இருக்க வேண்டும். Deployment பெயரை மாற்றலாம்.
- நிறுவல் வழிகாட்டியில் வெளியீட்டுக் கட்டணம் மற்றும் நிபந்தனைகளைப் பற்றி அறிந்து கொள்ள Select Pricing and terms ஐ கிளிக் செய்யவும்.
- Deploy-ஐத் தேர்ந்தெடுக்கவும். நிறுவல் தயாராகும் வரை காத்திருக்கவும், பின்னர் Deployments பக்கத்திற்கு தானாக மாற்றப்படும்.
- வகையில் Open in playground-ஐ தேர்ந்தெடுத்து மாதிரி உடன் தொடர்பு கொள்ளத் தொடங்கவும்.
- Deployments பக்கத்திற்கு திரும்பி நிறுவலைத் தேர்ந்தெடுத்து, முகவரியின் Target URL மற்றும் ரகசிய விசையை நோட்டுப்படுத்தலாம், இவற்றைப் பயன்படுத்தி நிறுவலை அழைத்து முடிவுகளை உருவாக்கலாம்.
- Build தாவலைச் சேர்ந்து Components பகுதியிலிருந்து Deployments ஐத் தேர்ந்தெடுத்து எப்போதும் முகவரி தொடர்புடைய விவரங்கள், URL மற்றும் அணுகல் விசைகளைப் பெறலாம்.

> [!NOTE]
> இந்த நடவடிக்கைகள் மேற்கொள்ள உங்கள் கணக்கில் Azure AI Developer இருக்கை Resource Group-இல் இருக்க வேண்டும் என்பதை கவனிக்கவும்.
