# AGENTS.md

## Project Overview

PhiCookBook என்பது Microsoft இன் Phi குடும்பத்தைச் சேர்ந்த சிறிய மொழி மாதிரிகள் (SLMs) உடன் பணியாற்றுவதற்கான கைப்பயிற்சி உதாரணங்கள், பயிற்சிகள் மற்றும் ஆவணங்களை உள்ளடக்கிய விரிவான சமையல் புத்தக களஞ்சியமாகும். இந்த களஞ்சியம் பல்்வேதகங்கள் மற்றும் கட்டமைப்புகள் வழியாக பிற்பாடு, நுட்பமயமாக்கல், அளவீடு, RAG செயலாக்கங்கள் மற்றும் பன்முக செயலிகளின் பல்வேறு பயன்பாடுகளை காட்டுகிறது.

**முக்கிய தொழில்நுட்பங்கள்:**
- **மொழிகள்:** Python, C#/.NET, JavaScript/Node.js
- **கட்டமைப்புகள்:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **வளம்:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **மாதிரி வகைகள்:** Phi-3, Phi-3.5, Phi-4 (உரை, காட்சியியல், பன்முக, காரணம் வகைகள்)

**களஞ்சிய அமைப்பு:**
- `/code/` - செயல்பாட்டுக் கோடு உதாரணங்கள் மற்றும் மாதிரிகள்
- `/md/` - விரிவான ஆவணங்கள், பயிற்சிகள் மற்றும் வழிகாட்டிகள்  
- `/translations/` - பன்மொழி மொழிபெயர்ப்புகள் (50+ மொழிகள் தானாக இயங்கும் வேலைப்பாடுகளின் மூலம்)
- `/.devcontainer/` - Dev container கட்டமைப்பு (Python 3.12 உடன் Ollama)

### Local Setup

**முன்னிலை தேவைகள்:**
- Python 3.12 அல்லது மேலதிக பதிப்பு
- .NET 8.0 SDK (C# உதாரணங்களுக்கு)
- Node.js 18+ மற்றும் npm (JavaScript உதாரணங்களுக்கு)
- குறைந்தது 16GB RAM பரிந்துரைக்கப்படுகிறது

**உட்பொதிக்கல்:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**Python உதாரணங்களுக்கு:**
குறிப்பிட்ட உதாரண அடைவேலையில் சென்று சார்ந்த பொருட்களை நிறுவுக:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # requirements.txt இருந்தால்
```

**.NET உதாரணங்களுக்கு:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**JavaScript/Web உதாரணங்களுக்கு:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # மேம்பாட்டு சர்வரை துவக்கவும்
npm run build  # உற்பத்திக்கான கட்டமைப்பு
```

## Testing

இந்தக் களஞ்சியம் பாரம்பரிய மென்பொருள் திட்டமாக அல்லாமல் உதாரணக் கோடுகள் மற்றும் பயிற்சிகளை கொண்டுள்ளது. சரிபார்த்தல் சாதாரணமாக பின்வருமாறு செய்யப்படுகிறது:

1. **உதாரணங்களை இயக்குதல்** - ஒவ்வொரு உதாரணமும் பிழைகள் இல்லாமல் இயங்க வேண்டும்
2. **வெளியீடுகளை சரிபார்த்தல்** - மாதிரி பதில்கள் முறைக்கு உட்பட்டதா என பரிசோதிக்கவும்
3. **பயிற்சிகளை பின்பற்றுதல்** - படிநிலை வழிகாட்டிகள் ஆவணப்படுத்தப்பட்டபடி இயங்க வேண்டும்

**பொதுவான சரிபார்த்தல் முறைகள்:**
- இலக்குச் சூழலில் உதாரணக் கோடுகளை சரிபார்த்தல்
- சார்ந்த பொருட்கள் சரியாக நிறுவப்பட்டுள்ளதா என்று உறுதிப்படுத்தல்
- மாதிரிகள் ஏற்றப்பட்டுள்ளதா என்று பார்
- எதிர்பார்த்த நடத்தைகள் ஆவணங்களுடன் பொருந்துமா என்பதைக் கண்டு பிடித்தல்

## Code Style and Conventions

### General Guidelines

- உதாரணங்கள் தெளிவாகவும், நன்கு கருத்திடப்பட்டதும், கல்வியப் பயனுள்ளதாக வேண்டும்
- மொழியின்படி நடைமுறைகளை பின்பற்றவும் (Python க்கு PEP 8, C# க்கு .NET தரநிலைகள்)
- Phi மாதிரியின் குறிப்பிட்ட திறன்களை விளக்க உதாரணங்கள் மட்டுமே
- முக்கியக் கருத்துக்களையும் மாதிரி-சேர்க்கை நிரலாளரையும் விளக்கும் குறிப்புகள் சேர்க்கவும்

### Documentation Standards

**URL வடிவமைப்பு:**
- `[text](../../url)` வடிவத்தில், கூடுதல் இடைவெளி இல்லாமல் பயன்படுத்தவும்
- சார்ந்த இணைப்புகள்: தற்போதைய அடைவேலை `./` மற்றும் மேல் அடைவேலை `../` என பயன்படுத்தவும்
- URLs இல் நாட்டுக்கோட் அடிப்படைக் கணினிகள் சேராதிருக்க வேண்டும் (எ.கா., `/en-us/`, `/en/`)

**படங்கள்:**
- அனைத்து படங்களும் `/imgs/` அடைவேலையில் வைக்கப்பட வேண்டும்
- பெயர்கள் ஆங்கில எழுத்துக்கள், எண்கள் மற்றும் கோடுகள் கொண்டிருக்கும்
- உதாரணம்: `phi-3-architecture.png`

**Markdown கோப்புகள்:**
- நடைமுறையில் உள்ள உதாரணங்கள் `/code/` இலிருந்து மேற்கோளிட வேண்டும்
- ஆவணங்கள் மற்றும் கோடு தொடர்புடைய மாற்றங்களுடன் ஒருங்கிணைக்கப்பட வேண்டும்
- README இல் Jupyter notebook க்கான 📓 எமோஜி குறியீட்டை பயன்படுத்தவும்

### File Organization

- `/code/` இலுள்ள கோடு உதாரணங்கள் தலைமை/வசதி அடிப்படையில் ஒழுங்குபடுத்தப்படும்
- `/md/` இலுள்ள ஆவணங்கள் கோடு அமைப்புடன் ஒத்திசைக்கப்படும்
- தொடர்புடைய கோப்புகள் (notebooks, scriptகள், கட்டமைப்புகள்) அடைவேலைகளில் ஒன்றாக வைக்கப்படும்

## Pull Request Guidelines

### Before Submitting

1. **களஞ்சியத்தை Fork செய்யவும்** உங்கள் கணக்கில்
2. **PR களை வகைப்படி பிரிக்கவும்:**
   - பிழை திருத்தங்கள் ஒன்றிலிருந்து
   - ஆவண மேம்பாடுகள் மற்றொன்றில்
   - புதிய உதாரணங்கள் தனித்தனியான PR களில்
   - தட்டுப்பாடு திருத்தங்கள் இணைக்கலாம்

3. **மெர்ஜ் முரண்பாடுகள் கையாள்க:**
   - மாற்றங்களை செய்வதற்கு முன் உங்கள் உள்ளூர் `main` கிளையை புதுப்பிக்கவும்
   - மேல்உள்ள கிளையில் அடிக்கடி ஒத்திசைக்கவும்

4. **மொழிபெயர்ப்பு PR கள்:**
   - அடைவேலைக்குள்ள அனைத்து கோப்புகளுக்கும் மொழிபெயர்ப்புகள் பூர்த்தி செய்யப்பட வேண்டும்
   - முதன்மை மொழியுடன் ஒரே அமைப்பை பராமரிக்கவும்

### Required Checks

PR கள் தானாக GitHub வேலைப்பூக்களை இயக்கி சரிபார்க்கும்:

1. **சார்ந்த பாதை சரிபார்த்தல்** - அனைத்து உள்ளக இணைப்புகளும் வேலை செய்ய வேண்டும்
   - VS Code இல் Ctrl+Click-ஆல் இணைப்புகளை பரிசோதிக்கவும்
   - பாதை பரிந்துரைகளை பயன்படுத்தவும் (`./` அல்லது `../`)

2. **URL நாட்டுக் கோடு சரிபார்த்தல்** - இணைய URLs இல் நாட்டுக்கோட் சேர்க்க வேண்டாம்
   - `/en-us/`, `/en/` போன்றவை நீக்கப்பட வேண்டும்
   - பொதுவான சர்வதேச URLs பயன்படுத்தவும்

3. **உருவாக்கப்பட்ட URL பரிசோதனை** - அனைத்து URLs 200 நிலையை தர வேண்டும்
   - இணைப்புகள் இடமாற்றத்துக்கு முன் பரிசோதிக்கவும்
   - சில விக்கிளைகள் நெட்வொர்க் தடைகளால் ஏற்படலாம்

## Common Development Patterns

### Working with Phi Models

**மாதிரி ஏற்றல்:**
- உதாரணங்கள் பல்வேறு கட்டமைப்புகளைப் பயன்படுத்துகின்றன: Transformers, ONNX Runtime, MLX, OpenVINO
- மாதிரிகள் பொதுவாக Hugging Face, Azure அல்லது GitHub Models இருந்து பதிவிறக்கம் செய்யப்படுகின்றன
- உங்கள் ஹார்டுவேருடன் (CPU, GPU, NPU) மாதிரி பொருந்துவதைச் சரிபார்க்கவும்

**பிற்பாட்டு முறைகள்:**

- காட்சி: Phi-3-vision மற்றும் Phi-4-multimodal படத்தை புரிந்துகொள்ள
- ஒலி: Phi-4-multimodal ஒலி உள்ளீடுகளை ஆதரிக்கிறது
- காரணம்: Phi-4-reasoning மேம்பட்ட காரண விளக்க நடவடிக்கைகள்

## Troubleshooting

### Common Issues

**நினைவகப் பிரச்சினைகள்:**
- Phi மாதிரிகள் பெரும்பாலும் அதிக நினைவகம் தேவை (முகாமகம் மற்றும் பன்முக வகைகள்)
- வள குறைந்த சூழலுக்கு அளவீடு செய்யப்பட்ட மாதிரிகளை பயன்படுத்தவும்
- பார்வைக்கு `/md/01.Introduction/04/QuantifyingPhi.md`

**சார்பு முரண்பாடுகள்:**
- Python உதாரணங்களுக்கு தனித்துவமான பதிப்பு தேவைகள் இருக்கலாம்
- ஒவ்வொரு உதாரணத்திற்கும் தனி virtual environment பயன்படுத்தவும்
- தனிப்பட்ட `requirements.txt` கோப்புகளை பார்

**மாதிரி பதிவிறக்கம் தோல்வி:**
- பெரிய மாதிரிகள் மெதுவான இணைப்புகளில் காலாவதியாகலாம்
- மேக சூழல்கள் (Codespaces, Azure) யை பயன்படுத்த பரிந்துரைக்கப்படுகிறது
- Hugging Face சேமிப்பு இடம்: `~/.cache/huggingface/`

**.NET திட்டப் பிரச்சினைகள்:**
- .NET 8.0 SDK நிறுவப்பட்டுள்ளது என உறுதி செய்யவும்
- கட்டுமானத்திற்கு முன் `dotnet restore` பயன்படுத்தவும்
- சில திட்டங்களில் CUDA-சார்ந்த கட்டமைப்புகள் (Debug_Cuda) உள்ளன

**JavaScript/Web உதாரணங்கள்:**
- நொடேஜ் 18+ தேவையானது
- `node_modules` அழித்து மறுபடியும் நிறுவவும், பிரச்சினைகள் இருந்தால்
- WebGPU பொருந்துவதை உலாவி கன்சோலில் சரிபார்க்கவும்
