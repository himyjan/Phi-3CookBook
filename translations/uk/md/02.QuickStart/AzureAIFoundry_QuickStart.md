# **Використання Phi-3 у Microsoft Foundry**

З розвитком генеративного ШІ ми сподіваємося використовувати єдину платформу для керування різними LLM та SLM, інтеграції корпоративних даних, операцій тонкого налаштування/RAG та оцінки різних корпоративних бізнесів після інтеграції LLM та SLM тощо, щоб генерований ШІ міг краще реалізовуватися у Smart додатках. [Microsoft Foundry](https://ai.azure.com) — це корпоративна платформа для застосувань генеративного ШІ.

![aistudo](../../../../translated_images/uk/aifoundry_home.f28a8127c96c7d93.webp)

За допомогою Microsoft Foundry ви можете оцінювати відповіді великих мовних моделей (LLM) та оркеструвати компоненти застосування підказок за допомогою prompt flow для кращої продуктивності. Платформа полегшує масштабованість для перетворення експериментальних концепцій у повноцінне виробництво. Безперервний моніторинг і вдосконалення підтримують довгостроковий успіх.

Ми можемо швидко розгорнути модель Phi-3 у Microsoft Foundry через прості кроки, а потім використовувати Microsoft Foundry для виконання пов’язаних із Phi-3 Playground/Chat, тонкого налаштування, оцінки та інших завдань.

## **1. Підготовка**

Якщо у вас уже встановлено [Azure Developer CLI](https://learn.microsoft.com/azure/developer/azure-developer-cli/overview?WT.mc_id=aiml-138114-kinfeylo) на вашому комп’ютері, використання цього шаблону настільки ж просте, як запуск цієї команди в новій директорії.

## Ручне створення

Створення проекту та хаба в Microsoft Foundry — чудовий спосіб організувати та керувати вашою роботою зі штучним інтелектом. Ось покрокова інструкція для початку роботи:

### Створення проекту в Microsoft Foundry

1. **Перейдіть до Microsoft Foundry**: Увійдіть у портал Microsoft Foundry.
2. **Створіть проект**:
   - Якщо ви вже перебуваєте у проекті, оберіть "Microsoft Foundry" у верхньому лівому кутку сторінки, щоб перейти на головну сторінку.
   - Виберіть "+ Create project".
   - Введіть назву проекту.
   - Якщо у вас є хаб, він буде вибраний за замовчуванням. Якщо у вас є доступ до кількох хабів, ви можете обрати інший із випадаючого списку. Якщо хочете створити новий хаб, виберіть "Create new hub" і вкажіть назву.
   - Виберіть "Create".

### Створення хаба в Microsoft Foundry

1. **Перейдіть до Microsoft Foundry**: Увійдіть за допомогою свого облікового запису Azure.
2. **Створіть хаб**:
   - Виберіть Центр керування (Management center) із лівого меню.
   - Виберіть "All resources", потім стрілку вниз біля "+ New project" і виберіть "+ New hub".
   - У діалоговому вікні "Create a new hub" введіть назву вашого хаба (наприклад, contoso-hub) та за бажанням змініть інші поля.
   - Виберіть "Next", перегляньте інформацію і натисніть "Create".

Для більш детальних інструкцій звертайтесь до офіційної [документації Microsoft](https://learn.microsoft.com/azure/ai-studio/how-to/create-projects).

Після успішного створення ви зможете отримати доступ до створеної студії через [ai.azure.com](https://ai.azure.com/)

В одному AI Foundry може бути кілька проектів. Створіть проект у AI Foundry для підготовки.

Створіть Microsoft Foundry [QuickStarts](https://learn.microsoft.com/azure/ai-studio/quickstarts/get-started-code)


## **2. Розгортання моделі Phi в Microsoft Foundry**

Натисніть опцію Explore у проекті, щоб перейти у Модельний каталог і вибрати Phi-3.

Виберіть Phi-3-mini-4k-instruct.

Натисніть 'Deploy', щоб розгорнути модель Phi-3-mini-4k-instruct.

> [!NOTE]
>
> Під час розгортання можна вибрати обчислювальну потужність.

## **3. Playground Chat Phi в Microsoft Foundry**

Перейдіть на сторінку розгортання, оберіть Playground та чатіться з Phi-3 у Microsoft Foundry.

## **4. Розгортання моделі з Microsoft Foundry**

Щоб розгорнути модель з каталогу моделей Azure, виконайте наступні кроки:

- Увійдіть у Microsoft Foundry.
- Оберіть модель, яку хочете розгорнути, з каталогу моделей Microsoft Foundry.
- На сторінці деталей моделі виберіть Deploy, а потім Serverless API з Azure AI Content Safety.
- Оберіть проект, у якому хочете розгорнути модель. Щоб використовувати Serverless API, ваш робочий простір повинен належати до регіону East US 2 або Sweden Central. Ви можете налаштувати ім’я розгортання.
- У майстрі розгортання виберіть Pricing and terms, щоб дізнатись про ціни та умови використання.
- Виберіть Deploy. Очікуйте, поки розгортання буде готове, і вас перенаправлять на сторінку розгортань.
- Виберіть Open in playground, щоб почати взаємодію з моделлю.
- Ви можете повернутися на сторінку розгортань, вибрати розгортання та звернути увагу на Target URL кінцевої точки та Secret Key, які можна використовувати для звернення до розгортання і генерації відповідей.
- Завжди можна знайти деталі кінцевої точки, URL і ключі доступу, перейшовши на вкладку Build та вибравши Deployments у розділі Components.

> [!NOTE]
> Зверніть увагу, що ваш обліковий запис повинен мати роль Azure AI Developer на рівні групи ресурсів для виконання цих кроків.

## **5. Використання Phi API у Microsoft Foundry**

Ви можете отримати доступ до https://{Your project name}.region.inference.ml.azure.com/swagger.json через Postman за допомогою GET і поєднати це з ключем для ознайомлення з наданими інтерфейсами.

Ви зможете дуже зручно отримувати параметри запиту, а також параметри відповіді.
