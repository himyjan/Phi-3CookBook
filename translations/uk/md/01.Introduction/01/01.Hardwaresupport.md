# Підтримка апаратного забезпечення Phi

Microsoft Phi оптимізовано для ONNX Runtime і підтримує Windows DirectML. Воно добре працює на різних типах апаратного забезпечення, включаючи GPU, CPU та навіть мобільні пристрої.

## Апаратне забезпечення пристрою
Зокрема, підтримується наступне обладнання:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPU, 128 ГіБ ОЗУ)

## Мобільний SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 або новіший з процесором A16/A17

## Специфікація апаратного забезпечення Phi

- Мінімальна конфігурація, що вимагається.
- Windows: GPU з підтримкою DirectX 12 і мінімум 4 ГБ загальної оперативної пам’яті

CUDA: NVIDIA GPU з обчислювальною здатністю >= 7.02

## Запуск onnxruntime на кількох GPU

Наразі доступні моделі Phi ONNX лише для 1 GPU. Можлива підтримка multi-gpu для моделі Phi, але ORT з 2 GPU не гарантує більшої пропускної здатності порівняно з двома екземплярами ort. Будь ласка, дивіться [ONNX Runtime](https://onnxruntime.ai/) для останніх оновлень.

На [Build 2024 команда GenAI ONNX](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) оголосила, що вони ввели мульти-екземпляр замість multi-gpu для моделей Phi.

Наразі це дозволяє запускати один onnnxruntime або onnxruntime-genai екземпляр з використанням змінної середовища CUDA_VISIBLE_DEVICES, наприклад.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

Не соромтеся досліджувати Phi далі в [Microsoft Foundry](https://ai.azure.com)
