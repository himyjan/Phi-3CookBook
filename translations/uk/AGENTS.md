# AGENTS.md

## Огляд проекту

PhiCookBook — це всеохопний репозиторій кулінарних рецептів, що містить практичні приклади, навчальні посібники та документацію для роботи з сімейством малих мовних моделей (SLMs) Phi від Microsoft. Репозиторій демонструє різноманітні варіанти використання, включно з інференсом, донавчанням, квантизацією, реалізаціями RAG та мультимодальними застосунками на різних платформах і фреймворках.

**Ключові технології:**
- **Мови:** Python, C#/.NET, JavaScript/Node.js
- **Фреймворки:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **Платформи:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **Типи моделей:** Phi-3, Phi-3.5, Phi-4 (текстові, зору, мультимодальні, з варіантами для логіки)

**Структура репозиторію:**
- `/code/` - Робочі приклади коду та зразкові реалізації
- `/md/` - Детальна документація, навчальні посібники та керівництва  
- `/translations/` - Переклади на більш ніж 50 мов за допомогою автоматизованого процесу
- `/.devcontainer/` - Конфігурація контейнера розробки (Python 3.12 з Ollama)

## Налаштування середовища розробки

### Використання GitHub Codespaces або Dev Containers (рекомендовано)

1. Відкрийте в GitHub Codespaces (найшвидший спосіб):
   - Натисніть на бейдж "Open in GitHub Codespaces" в README
   - Контейнер автоматично налаштовується з Python 3.12 та Ollama з Phi-3

2. Відкрийте у VS Code Dev Containers:
   - Використайте бейдж "Open in Dev Containers" з README
   - Для контейнера потрібно мінімум 16 ГБ оперативної пам’яті

### Локальне налаштування

**Вимоги:**
- Python 3.12 або новіша версія
- .NET 8.0 SDK (для прикладів на C#)
- Node.js 18+ та npm (для прикладів на JavaScript)
- Рекомендовано мінімум 16 ГБ оперативної пам’яті

**Інсталяція:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**Для прикладів на Python:**
Перейдіть у відповідні каталоги з прикладами та встановіть залежності:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # якщо існує файл requirements.txt
```

**Для прикладів на .NET:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**Для прикладів на JavaScript/Web:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # Запустити сервер розробки
npm run build  # Зібрати для виробництва
```

PR автоматично запускають GitHub workflows для перевірки:

1. **Перевірка відносних шляхів** — всі внутрішні посилання мають працювати

## Усунення неполадок

### Типові проблеми

**Проблеми з пам’яттю:**
- Моделі Phi потребують значної оперативної пам’яті (особливо варіанти зору/мультимодальні)
- Використовуйте квантизовані моделі для обмежених ресурсів
- Див. `/md/01.Introduction/04/QuantifyingPhi.md`

**Конфлікти залежностей:**
- Приклади на Python можуть потребувати конкретних версій
- Використовуйте віртуальні середовища для кожного прикладу
- Перевіряйте файли `requirements.txt` кожного прикладу

**Помилки завантаження моделей:**
- Великі моделі можуть таймаутити при повільному підключенні
- Розгляньте використання хмарних середовищ (Codespaces, Azure)
- Перевірте кеш Hugging Face: `~/.cache/huggingface/`

**Проблеми з .NET проектами:**
- Переконайтеся, що встановлено .NET 8.0 SDK
- Запускайте `dotnet restore` перед збіркою
- Деякі проекти мають специфічні налаштування CUDA (Debug_Cuda)

**Приклади JavaScript/Web:**
- Використовуйте Node.js 18+ для сумісності
- Очищуйте `node_modules` та повторно інсталюйте, якщо є проблеми
- Перевіряйте консоль браузера на сумісність з WebGPU
