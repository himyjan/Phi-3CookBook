# Phi Family у AITK

[AI Toolkit за VS Code](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio) поједностављује развој генеративних AI апликација окупљајући врхунске алате за развој AI и моделе из Microsoft Foundry каталога и других каталога као што је Hugging Face. Бићете у могућности да прегледате каталог AI модела који покрећу GitHub Models и Microsoft Foundry Model каталози, преузмете их локално или удаљено, фино подесите, тестирате и користите у вашој апликацији.

AI Toolkit Preview радиће локално. Локална инференца или фино подешавање, у зависности од изабраног модела, можда ће вам требати GPU као што је NVIDIA CUDA GPU. Такође можете директно покретати GitHub Models преко AITK.

### Инсталирање AI Toolkit-а

AI Toolkit се испоручује као [Visual Studio Code екстензија](https://code.visualstudio.com/docs/setup/additional-components#_vs-code-extensions), па је потребно најпре инсталирати [VS Code](https://code.visualstudio.com/docs/setup/windows?WT.mc_id=aiml-137032-kinfeylo), а затим преузети AI Toolkit са [VS Marketplace](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio).
[AI Toolkit је доступан у Visual Studio Marketplace-у](https://marketplace.visualstudio.com/items?itemName=ms-windows-ai-studio.windows-ai-studio) и може се инсталирати као и било која друга VS Code екстензија.

Ако нисте упознати са инсталацијом VS Code екстензија, пратите ове кораке:

### Пријава

1. У Activity Bar у VS Code-у изаберите **Extensions**
1. У претрагу екстензија укуцајте "AI Toolkit"
1. Изаберите "AI Toolkit for Visual Studio code"
1. Изаберите **Install**

Сада сте спремни да користите екстензију!

Бићете упитани да се пријавите на GitHub, па кликните "Allow" да наставите. Бићете преусмерени на страницу за пријаву GitHub-а.

Молимо пријавите се и пратите кораке. Након успешног завршетка, бићете преусмерени назад у VS Code.

Када се екстензија инсталира, видећете иконицу AI Toolkit-а у вашем Activity Bar-у.

Хајде да истражимо доступне акције!

### Доступне акције

Примарна бочна трака AI Toolkit-а организована је у

- **Models**
- **Resources**
- **Playground**  
- **Fine-tuning**
- **Evaluation**

који су доступни у одељку Resources. За почетак изаберите **Model Catalog**.

### Преузмите модел из каталога

При покретању AI Toolkit-а из VS Code бочне траке, можете изабрати следеће опције:

![AI toolkit model catalog](../../../../../translated_images/sr/AItoolkitmodel_catalog.7a7be6a7d8468d31.webp)

- Пронађите подржани модел у **Model Catalog** и преузмите локално
- Тестирајте инференцу модела у **Model Playground**
- Фино подесите модел локално или удаљено у **Model Fine-tuning**
- Деплоy формиране моделе у облак преко командне палете за AI Toolkit
- Евалуација модела

> [!NOTE]
>
> **GPU против CPU**
>
> Приметићете да картице модела приказују величину модела, платформу и тип акцелератора (CPU, GPU). За оптимизовану перформансу на **Windows уређајима који имају најмање један GPU**, изаберите верзије модела које циљају само Windows.
>
> Ово обезбеђује да добијете модел оптимизован за DirectML акцелератор.
>
> Имена модела су у формату
>
> - `{model_name}-{accelerator}-{quantization}-{format}`.
>
>Да проверите да ли имате GPU на вашем Windows уређају, отворите **Task Manager** и изаберите таб **Performance**. Ако имате GPU, они ће бити наведени под именима као "GPU 0" или "GPU 1".

### Користите REST API у вашој апликацији

AI Toolkit долази са локалним REST API веб сервером **на порту 5272** који користи [OpenAI chat completions формaт](https://platform.openai.com/docs/api-reference/chat/create).

Ово вам омогућава да тестирате вашу апликацију локално без потребе да се ослањате на cloud AI сервисе модела. На пример, следећи JSON фајл показује како конфигурисати тело захтева:

```json
{
    "model": "Phi-4",
    "messages": [
        {
            "role": "user",
            "content": "what is the golden ratio?"
        }
    ],
    "temperature": 0.7,
    "top_p": 1,
    "top_k": 10,
    "max_tokens": 100,
    "stream": true
}
```

REST API можете тестирати користећи (рецимо) [Postman](https://www.postman.com/) или CURL (Client URL) алат:

```bash
curl -vX POST http://127.0.0.1:5272/v1/chat/completions -H 'Content-Type: application/json' -d @body.json
```

### Коришћење OpenAI клиент библиотеке за Python

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:5272/v1/", 
    api_key="x" # потребно за API али се не користи
)

chat_completion = client.chat.completions.create(
    messages=[
        {
            "role": "user",
            "content": "what is the golden ratio?",
        }
    ],
    model="Phi-4",
)

print(chat_completion.choices[0].message.content)
```

### Коришћење Azure OpenAI клиент библиотеке за .NET

Додајте [Azure OpenAI клиент библиотеку за .NET](https://www.nuget.org/packages/Azure.AI.OpenAI/) у ваш пројекат користећи NuGet:

```bash
dotnet add {project_name} package Azure.AI.OpenAI --version 1.0.0-beta.17
```

Додајте C# фајл под називом **OverridePolicy.cs** у ваш пројекат и налепите следећи код:

```csharp
// OverridePolicy.cs
using Azure.Core.Pipeline;
using Azure.Core;

internal partial class OverrideRequestUriPolicy(Uri overrideUri)
    : HttpPipelineSynchronousPolicy
{
    private readonly Uri _overrideUri = overrideUri;

    public override void OnSendingRequest(HttpMessage message)
    {
        message.Request.Uri.Reset(_overrideUri);
    }
}
```

Затим, налепите следећи код у ваш **Program.cs** фајл:

```csharp
// Program.cs
using Azure.AI.OpenAI;

Uri localhostUri = new("http://localhost:5272/v1/chat/completions");

OpenAIClientOptions clientOptions = new();
clientOptions.AddPolicy(
    new OverrideRequestUriPolicy(localhostUri),
    Azure.Core.HttpPipelinePosition.BeforeTransport);
OpenAIClient client = new(openAIApiKey: "unused", clientOptions);

ChatCompletionsOptions options = new()
{
    DeploymentName = "Phi-4",
    Messages =
    {
        new ChatRequestSystemMessage("You are a helpful assistant. Be brief and succinct."),
        new ChatRequestUserMessage("What is the golden ratio?"),
    }
};

StreamingResponse<StreamingChatCompletionsUpdate> streamingChatResponse
    = await client.GetChatCompletionsStreamingAsync(options);

await foreach (StreamingChatCompletionsUpdate chatChunk in streamingChatResponse)
{
    Console.Write(chatChunk.ContentUpdate);
}
```
