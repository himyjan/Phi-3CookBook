# AGENTS.md

## Преглед пројекта

PhiCookBook је свеобухватан репозиторијум куварица који садржи практичне примере, туторијале и документацију за рад са Microsoft-овом Phi породицом малих језичких модела (SLM). Репозиторијум демонстрира разне случајеве употребе укључујући инференцију, фино подешавање, квантовање, имплементације RAG-а и мултимодалне апликације на различитим платформама и оквирима.

**Кључне технологије:**
- **Језици:** Python, C#/.NET, JavaScript/Node.js
- **Оквири:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **Платформе:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **Типови модела:** Phi-3, Phi-3.5, Phi-4 (текст, визија, мултимодалне, варијанте за разлоговање)

**Структура репозиторијума:**
- `/code/` - Радни код и пример имплементација
- `/md/` - Детаљна документација, туторијали и водичи  
- `/translations/` - Преводи на више језика (50+ језика преко аутоматизованог процеса)
- `/.devcontainer/` - Конфигурација дев контејнера (Python 3.12 са Ollama)

## Подешавање развојног окружења

### Коришћење GitHub Codespaces или Dev Containers (препоручено)

1. Отворите у GitHub Codespaces (најбрже):
   - Кликните на ознаку "Open in GitHub Codespaces" у README
   - Контейнер се аутоматски конфигурише са Python 3.12 и Ollama Phi-3

2. Отворите у VS Code Dev Containers:
   - Користите ознаку "Open in Dev Containers" из README
   - Контейнер захтева минимум 16GB меморије на хосту

### Локално подешавање

**Претходни услови:**
- Python 3.12 или новији
- .NET 8.0 SDK (за C# примере)
- Node.js 18+ и npm (за JavaScript примере)
- Препоручено минимум 16GB RAM

**Инсталација:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**За Python примере:**
Уђите у директоријуме са примерима и инсталирајте зависности:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # ако постоји requirements.txt
```

**За .NET примере:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**За JavaScript/Web примере:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # Покрени сервер за развој
npm run build  # Креирај за продукцију
```

## Тестирање

Овај репозиторијум садржи пример кодове и туторијале уместо традиционалног софтверског пројекта са јединичним тестовима. Валидација се обично врши кроз:

1. **Покретање примера** - Сваки пример треба да се изврши без грешака
2. **Провера излаза** - Проверите да одговори модела одговарају очекивањима
3. **Праћење туторијала** - Водичи треба да функционишу како је описано

**Уобичајени приступ валидацији:**
- Тестирање покретања примера у циљаном окружењу
- Верификација исправне инсталације зависности
- Провера да ли се модели успешно скидају/учитавају
- Потврда да очекивано понашање одговара документацији

## Стил кода и конвенције

### Општа упутства

- Примери треба да буду јасни, добро коментарисани и едукативни
- Пратите језичке конвенције (PEP 8 за Python, C# стандарде за .NET)
- Примери треба да буду фокусирани на демонстрирање специфичних могућности Phi модела
- Укључите коментаре који објашњавају кључне концепте и параметре специфичне за модел

### Стандарди за документацију

**Форматирање URL-ова:**
- Користите формат `[text](../../url)` без додатних размака
- Релативне везе: користите `./` за тренутни директоријум, `../` за родитељски
- Без специфичних локала у URL-овима (избегавајте `/en-us/`, `/en/`)

**Слике:**
- Све слике смештати у директоријум `/imgs/`
- Користити описна имена са енглеским знацима, бројевима и цртицама
- Пример: `phi-3-architecture.png`

**Markdown фајлови:**
- Реферисање стварних радних примера у директоријуму `/code/`
- Документација треба да буде синхронизована са променама кода
- Користите 📓 емоџи за означавање Jupyter notebook линкова у README

### Организација фајлова

- Код примери у `/code/` организовани по темама/функцијама
- Документација у `/md/` прати структуру кода кад год је применљиво
- Одржавајте повезане фајлове (ноутбукови, скрипте, конфигурације) заједно у поддиректоријумима

## Упутства за Pull Request

### Пре слања

1. **Форкујте репозиторијум** у свој налог
2. **Одвојите PR-ове по типу:**
   - Поправке грешака у једном PR-у
   - Ажурирања документације у другом
   - Нови примери у посебним PR-овима
   - Поправке типографских грешака могу бити комбиноване

3. **Решавање конфликата приликом спајања:**
   - Ажурирајте свој локални `main` пре измена
   - Учестало синхронизујте са upstream

4. **PR-ови за превод:**
   - Морају садржати преводе за СВЕ фајлове у фасцикли
   - Одржавајте структуру конзистентну са оригиналом

### Обавезне провере

PR-ови аутоматски покрећу GitHub радне токове за валидацију:

1. **Валидација релативних путева** - Све унутрашње везе морају радити
   - Тестирајте линкове локално: Ctrl+Click у VS Code
   - Користите предложене путеве из VS Code (`./` или `../`)

2. **Провера URL локала** - Web URL-ови не смеју садржати локалне ознаке земаља
   - Уклоните `/en-us/`, `/en/` или друге језичке шифре
   - Користите опште међународне URL-ове

3. **Провера нерадних URL-ова** - Сви URL-ови морају враћати статус 200
   - Проверите да су линкови доступни пре слања
   - Напомена: неки неуспеси могу бити узроковани мрежним ограничењима

### Формат наслова PR-а

```
[component] Brief description
```

Примери:
- `[docs] Add Phi-4 inference tutorial`
- `[code] Fix ONNX Runtime integration example`
- `[translation] Add Japanese translation for intro guides`

## Решавање проблема

### Уобичајени проблеми

**Проблеми са меморијом:**
- Phi модели захтевају значајан RAM (посебно визијски/мултимодални)
- Користите квантоване моделе у окружењима са ограниченим ресурсима
- Види `/md/01.Introduction/04/QuantifyingPhi.md`

**Конфликти зависности:**
- Python примери могу имати специфичне захтеве за верзије
- Користите виртуелна окружења за сваки пример
- Проверите појединачне фајлове `requirements.txt`

**Проблеми приликом скидања модела:**
- Велики модели могу имати временско ограничење на спорим конекцијама
- Размислите о коришћењу облачних окружења (Codespaces, Azure)
- Проверите Hugging Face кеш: `~/.cache/huggingface/`

**Проблеми са .NET пројектима:**
- Осигурајте да је инсталиран .NET 8.0 SDK
- Користите `dotnet restore` пре изградње
- Неки пројекти имају CUDA-специфичне конфигурације (Debug_Cuda)

**JavaScript/Web примери:**
- Користите Node.js 18+ ради компатибилности
- Обришите `node_modules` и поново инсталирајте ако проблеми трају
- Проверите конзолу прегледача због проблема са WebGPU
