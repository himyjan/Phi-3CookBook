# Поддръжка на хардуер Phi

Microsoft Phi е оптимизиран за ONNX Runtime и поддържа Windows DirectML. Работи добре с различни видове хардуер, включително GPU, CPU и дори мобилни устройства.

## Хардуер на устройството
По-конкретно, поддържаният хардуер включва:

- GPU SKU: RTX 4090 (DirectML)
- GPU SKU: 1 A100 80GB (CUDA)
- CPU SKU: Standard F64s v2 (64 vCPU, 128 GiB памет)

## Мобилен SKU

- Android - Samsung Galaxy S21
- Apple iPhone 14 или по-висок с процесор A16/A17

## Спецификация на хардуера Phi

- Минимална изисквана конфигурация.
- Windows: GPU с поддръжка на DirectX 12 и поне 4GB комбинирана RAM

CUDA: NVIDIA GPU с изчислителна способност >= 7.02

## Стартиране на onnxruntime на няколко GPU

В момента наличните Phi ONNX модели са само за 1 GPU. Възможно е да се поддържа multi-gpu за Phi модел, но ORT с 2 GPU не гарантира, че ще осигури по-голям капацитет в сравнение с 2 инстанции на ORT. Моля, вижте [ONNX Runtime](https://onnxruntime.ai/) за последните новости.

По време на [Build 2024 GenAI ONNX екипът](https://youtu.be/WLW4SE8M9i8?si=EtG04UwDvcjunyfC) обяви, че са активирали мулти-инстанция вместо multi-gpu за Phi модели.

В момента това ви позволява да стартирате една инстанция onnxruntime или onnxruntime-genai с променлива на средата CUDA_VISIBLE_DEVICES по следния начин.

```Python
CUDA_VISIBLE_DEVICES=0 python infer.py
CUDA_VISIBLE_DEVICES=1 python infer.py
```

Чувствайте се свободни да изследвате допълнително Phi в [Microsoft Foundry](https://ai.azure.com)
