# AGENTS.md

## Преглед на проекта

PhiCookBook е изчерпателно хранилище с готварски рецепти, съдържащо практически примери, уроци и документация за работа със семейството Phi на Microsoft от Малки Езикови Модели (SLMs). Хранилището демонстрира различни случаи на използване, включително извеждане на изводи, фино настройване, квантуване, реализации на RAG и мултимодални приложения на различни платформи и рамки.

**Ключови технологии:**
- **Езици:** Python, C#/.NET, JavaScript/Node.js
- **Рамки:** ONNX Runtime, PyTorch, Transformers, MLX, OpenVINO, Semantic Kernel
- **Платформи:** Microsoft Foundry, GitHub Models, Hugging Face, Ollama
- **Видове модели:** Phi-3, Phi-3.5, Phi-4 (текст, зрение, мултимодални, варианти за разсъждение)

**Структура на хранилището:**
- `/code/` - Работещи примери с код и примерни реализации
- `/md/` - Подробна документация, уроци и ръководства  
- `/translations/` - Преводи на много езици (50+ езика чрез автоматизиран работен процес)
- `/.devcontainer/` - Конфигурация на Dev контейнер (Python 3.12 с Ollama)

## Настройка на среда за развитие

### Използване на GitHub Codespaces или Dev контейнер (Препоръчително)

1. Отворете в GitHub Codespaces (най-бързо):
   - Кликнете на значката "Open in GitHub Codespaces" в README
   - Контейнерът се конфигурира автоматично с Python 3.12 и Ollama с Phi-3

2. Отваряне във VS Code Dev Containers:
   - Използвайте значката "Open in Dev Containers" от README
   - Контейнерът изисква минимум 16GB памет на хост машината

### Локална настройка

**Изисквания:**
- Python 3.12 или по-нова версия
- .NET 8.0 SDK (за C# примери)
- Node.js 18+ и npm (за JavaScript примери)
- Препоръчително минимум 16GB RAM

**Инсталиране:**
```bash
git clone https://github.com/microsoft/PhiCookBook.git
cd PhiCookBook
```

**За Python примери:**
Навигирайте до конкретните директории с примери и инсталирайте зависимостите:
```bash
cd code/<example-directory>
pip install -r requirements.txt  # ако съществува requirements.txt
```

**За .NET примери:**
```bash
cd md/04.HOL/dotnet/src
dotnet restore LabsPhi.sln
dotnet build LabsPhi.sln
```

**За JavaScript/уеб примери:**
```bash
cd code/08.RAG/rag_webgpu_chat
npm install
npm run dev  # Стартирай сървър за развитие
npm run build  # Създай за продукция
```

## Отстраняване на проблеми

### Чести проблеми

**Проблеми с паметта:**
- Phi моделите изискват значителна RAM (особено зрение/мултимодални варианти)
- Използвайте квантувани модели за среди с ограничени ресурси
- Вижте `/md/01.Introduction/04/QuantifyingPhi.md`

**Конфликти на зависимости:**
- Python примерите може да имат специфични изисквания за версии
- Използвайте виртуални среди за всеки пример
- Проверете отделните файлове `requirements.txt`

**Проблеми с изтегляне на модели:**
- Големите модели могат да изтекат при бавни връзки
- Помислете за използване на облачни среди (Codespaces, Azure)
- Проверете кеша на Hugging Face: `~/.cache/huggingface/`

**Проблеми с .NET проекти:**
- Уверете се, че .NET 8.0 SDK е инсталиран
- Използвайте `dotnet restore` преди компилация
- Някои проекти имат CUDA-специфични конфигурации (Debug_Cuda)

**JavaScript/уеб примери:**
- Използвайте Node.js 18+ за съвместимост
- Изчистете `node_modules` и инсталирайте наново при проблеми
- Проверете конзолата на браузъра за проблеми с WebGPU
